# База знаний по предметным областям: Java

Проект-база знаний для подготовки и развития Java-разработчика: систематизированные
предметные области, большая база вопросов для собеседований с проверенными ответами и
русский перевод официальной Java-документации. Весь контент — на русском, со ссылками на
достоверные источники. У каждого раздела есть собственный `README`-индекс.

## Структура

| Раздел | Что внутри |
|--------|-----------|
| [`knowledge-base/`](knowledge-base/README.md) | **Предметные области (25).** Темы, которые должен знать современный Java-программист. У каждой области — описание, ключевые подтемы и 3–4 достоверных источника. |
| [`interview-questions/`](interview-questions/README.md) | **Вопросы для собеседований (1500+).** База с проверенными ответами (секции «Оригинальный ответ из интернета» и «Ответ от Claude»), разбита по разделам, ≤ 20 вопросов в файле. |

## Быстрый запуск приложения

Откроется локальный сервер, браузер запустится автоматически (http://127.0.0.1:8000/).
Остановка — `Ctrl+C`. Нужен только Python 3, внешних зависимостей нет.

### Linux / macOS

Из терминала в корне проекта:

```bash
./start-app.sh        # или: cd webapp && python3 app.py
```

> Для запуска двойным кликом из файлового менеджера можно создать ярлык `.desktop`,
> прописав в нём абсолютные пути `Exec=` и `Path=` к этому каталогу (ярлык машинозависим
> и в репозиторий не входит).

### Windows

Двойной клик по `start-app.bat` в проводнике или из командной строки в корне проекта:

```bat
start-app.bat         REM или: cd webapp && python app.py
```

## Онлайн-версия (GitHub Pages)

Статическая версия просмотрщика публикуется автоматически из ветки `master`:

- Живая ссылка: https://kaivoid.github.io/knowledge_base/
- Сборка: `python3 webapp/build.py` (генерирует `dist/`); в CI это делает
  workflow `.github/workflows/pages.yml` при каждом push в `master`.
- **Разовая настройка (вручную в GitHub, обязательна один раз):**
  Settings → Pages → Build and deployment → Source = «GitHub Actions».
  Токен Actions не вправе создать Pages-сайт сам (`Resource not accessible by
  integration`), поэтому первый раз источник включается вручную. После этого
  каждый push в `master` деплоится автоматически.

## Принципы

- Всё на русском языке. Иностранные источники переводятся.
- Источники — реальные и проверенные (приоритет: официальная документация Oracle/Spring/Hibernate, Baeldung, признанные книги и авторы).
- Ответы на вопросы не выдумываются: ищутся в интернете, сверяются по нескольким источникам.
- Файлы перекрёстно ссылаются друг на друга: область знаний ↔ соответствующие вопросы.
