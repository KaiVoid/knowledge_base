# Правила формирования вопросов для собеседования

Этот файл — единый источник правды по тому, как оформляется **каждый** вопрос в базе.
Любой новый вопрос (в т.ч. добавленный вручную или через волны генерации) обязан
соответствовать шаблону ниже. Общие правила проекта — в [`../CLAUDE.md`](../CLAUDE.md).

## Обязательные пункты вопроса

У каждого вопроса должны присутствовать **все** перечисленные элементы:

| # | Пункт | Описание |
|---|-------|----------|
| 1 | **Уровень** | Один из трёх: **Junior** (джун), **Middle** (мидл), **Senior** (сеньор). |
| 2 | **Исходный вопрос (на русском)** | Текст вопроса на русском языке. Если найден на другом языке — переводится. |
| 3 | **Оригинальный ответ** | Ответ из реального интернет-источника, переведённый на русский. Ничего не выдумывать. |
| 4 | **Ответ от Claude** | Выверенная по нескольким источникам версия ответа; расхождения помечаются явно. |
| 5 | **Ссылка на источник** | Рабочий URL источника, из которого взят оригинальный ответ. |

Дополнительно: **категория** (раздел) и **сквозной номер** вопроса внутри файла раздела.

## Шаблон вопроса

```markdown
### Вопрос {N}. {исходный вопрос на русском}

**Категория:** {раздел} · **Уровень:** Junior | Middle | Senior

#### Оригинальный ответ из интернета
> Источник: [{название источника}]({рабочий URL})

{оригинальный ответ из источника, переведённый на русский}

#### Ответ от Claude

{выверенный по нескольким источникам ответ на русском; расхождения помечаются явно}

---
```

## Соответствие пунктов шаблону

- **Уровень** → поле `**Уровень:**` в строке после заголовка.
- **Исходный вопрос (на русском)** → текст в заголовке `### Вопрос {N}. …`.
- **Ссылка на источник** → строка `> Источник: [название](URL)` в секции оригинального ответа.
- **Оригинальный ответ** → текст секции `#### Оригинальный ответ из интернета`.
- **Ответ от Claude** → текст секции `#### Ответ от Claude`.

## Дополнительные правила

1. **Один файл = один раздел.** Все вопросы в одном файле хранить запрещено
   (разбивка по предметным областям: core-java, collections, concurrency, …).
   **Не более 20 вопросов в одном файле.** Если в разделе больше 20 вопросов, он разбивается
   на части вида `<раздел>-NN.md` (`collections-01.md`, `collections-02.md`, …), в каждой ≤ 20
   вопросов. Сквозная нумерация **продолжается** между частями (часть 02 начинается с номера,
   следующего за последним в части 01). Все части перечисляются в `README.md`.
2. **Источник — только реальный.** URL должен быть рабочим; приоритет достоверности:
   официальная документация (Oracle, Spring, Hibernate, Apache, Kubernetes, OWASP) →
   авторитетные ресурсы (Baeldung, Jenkov, refactoring.guru) → признанные книги.
3. **Несколько источников.** Если ответ собран из нескольких источников — каждый помечается
   своей ссылкой; противоречия между источниками указываются явно в «Ответе от Claude».
4. **Сквозная нумерация** вопросов внутри раздела, без пропусков.
5. **Без дубликатов** — ни внутри раздела, ни между разделами.
6. **Никаких управляющих/невидимых символов** в тексте (например, нулевой символ
   записывается как escape-последовательность \u0000 (в виде текста), а не как реальный байт).

## Пример заполненного вопроса

```markdown
### Вопрос 4. Что выведет `Integer a = 127, b = 127; System.out.println(a == b);` и почему?

**Категория:** Core Java — основы · **Уровень:** Middle

#### Оригинальный ответ из интернета
> Источник: [Java Language Specification, §5.1.7 — docs.oracle.com](https://docs.oracle.com/javase/specs/jls/se17/html/jls-5.html)

Boxing значений в диапазоне −128…127 кешируется, поэтому два `Integer` со значением 127
указывают на один объект, и `a == b` возвращает `true`. Для 128 кеш не действует.

#### Ответ от Claude

Выведет `true`. Автоупаковка вызывает `Integer.valueOf(127)`, а этот метод кеширует значения
в диапазоне −128…127 (`IntegerCache`), возвращая один и тот же объект. Для `128` объекты
разные → `==` дало бы `false`. Для сравнения значений используйте `equals()`.

---
```

## Под-раздел «Вопросы с HH»

Вопросы, перенесённые из репозитория
[Londeren/hh-skill-verifications-quizzes](https://github.com/Londeren/hh-skill-verifications-quizzes),
живут отдельным деревом `interview-questions/hh/<topic>/<level>/questions.md` и
отличаются от остальных только тремя пунктами:

- **Уровни англоязычные** — `Advanced` / `Intermediate` / `Basic` (в источнике так).
- **Источник** — файл темы/уровня в репозитории HH (ветка `main`).
- **Раскладка папками**: тема → уровень; имя файла `questions.md` (при разбиении —
  `questions-NN.md` со сквозной нумерацией).

В остальном — тот же шаблон: два блока (оригинал + ответ Claude), ≤20 вопросов в
файле, сквозная нумерация внутри раздела. Готовый шаблон —
[`../docs/templates/hh-question-template.md`](../docs/templates/hh-question-template.md).
