### Вопрос 1. LC 217. Содержит дубликат (Contains Duplicate)

**Категория:** Алгоритмы и структуры данных · Hash Table · **Уровень:** Easy

Дан целочисленный массив `nums`. Верните `true`, если какое-либо значение встречается
в массиве хотя бы дважды, и `false`, если все элементы различны.

**Пример 1.** Вход: `nums = [1,2,3,1]` → Выход: `true` (число 1 повторяется).
**Пример 2.** Вход: `nums = [1,2,3,4]` → Выход: `false` (все различны).
**Ограничения:** `1 <= nums.length <= 10^5`, `-10^9 <= nums[i] <= 10^9`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Contains Duplicate](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0217.Contains%20Duplicate/README.md)

Подход с хеш-множеством: проходим по массиву, добавляя элементы в `HashSet`. Если
очередной элемент уже присутствует — найден дубликат, возвращаем `true`. Сложность —
время O(n), память O(n).

```java
class Solution {
    public boolean containsDuplicate(int[] nums) {
        Set<Integer> s = new HashSet<>();
        for (int num : nums) {
            if (!s.add(num)) {
                return true;
            }
        }
        return false;
    }
}
```

#### Ответ от Claude

**Идея.** Дубликат — это коллизия по значению. Достаточно одного прохода с множеством
уже встреченных чисел: `HashSet.add` возвращает `false`, если элемент уже был.

```java
public boolean containsDuplicate(int[] nums) {
    Set<Integer> seen = new HashSet<>(nums.length * 4 / 3 + 1);
    for (int x : nums) {
        if (!seen.add(x)) return true;
    }
    return false;
}
```

**Разбор по шагам.** Идём слева направо; на каждом шаге пытаемся вставить `x`. Первая же
неудачная вставка означает повтор. Предразмер множества убирает рехеши.

**Сложность.** Время O(n), память O(n).

**Типичные ошибки / альтернативы.** Сортировка + проверка соседей даёт O(n log n) время и
O(1) доп. памяти (если разрешено менять вход) — медленнее по времени, но экономнее по
памяти. Подход «битовая маска» здесь неприменим из-за диапазона значений.

### Вопрос 2. LC 136. Одиночное число (Single Number)

**Категория:** Алгоритмы и структуры данных · Bit Manipulation · **Уровень:** Easy

Дан непустой целочисленный массив `nums`, в котором каждый элемент встречается дважды,
кроме одного. Найдите этот одиночный элемент.

**Пример 1.** Вход: `nums = [2,2,1]` → Выход: `1`.
**Пример 2.** Вход: `nums = [4,1,2,1,2]` → Выход: `4`.
**Ограничения:** `1 <= nums.length <= 3*10^4`, `-3*10^4 <= nums[i] <= 3*10^4`.
Алгоритм должен работать за O(n) времени и O(1) доп. памяти.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Single Number](https://github.com/doocs/leetcode/blob/main/solution/0100-0199/0136.Single%20Number/README.md)

Подход через XOR: каждое число, встреченное дважды, обнуляет себя (a XOR a = 0),
а одиночное число остаётся (a XOR 0 = a). Сложность — время O(n), память O(1).

```java
class Solution {
    public int singleNumber(int[] nums) {
        int ans = 0;
        for (int v : nums) {
            ans ^= v;
        }
        return ans;
    }
}
```

#### Ответ от Claude

**Идея.** XOR коммутативен и ассоциативен: порядок не важен. Все парные элементы
взаимно уничтожатся, останется только одиночный.

```java
public int singleNumber(int[] nums) {
    int result = 0;
    for (int x : nums) {
        result ^= x;
    }
    return result;
}
```

**Разбор по шагам.** Начинаем с `result = 0`. При каждой операции XOR либо устанавливаем
биты нового числа, либо снимаем их (если число встречается второй раз). В итоге в
`result` остаются только биты числа, встретившегося нечётное число раз.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Альтернатива — `HashMap` с подсчётом частот
(O(n) время, O(n) память) или сортировка (O(n log n) время). XOR-решение оптимально.
Важно: метод XOR работает только когда «лишнее» число встречается нечётное число раз,
а все остальные — чётное.

### Вопрос 3. LC 242. Анаграмма (Valid Anagram)

**Категория:** Алгоритмы и структуры данных · Hash Table / строки · **Уровень:** Easy

Даны две строки `s` и `t`. Верните `true`, если `t` является анаграммой строки `s`,
иначе `false`. Анаграмма — перестановка символов строки.

**Пример 1.** Вход: `s = "anagram"`, `t = "nagaram"` → Выход: `true`.
**Пример 2.** Вход: `s = "rat"`, `t = "car"` → Выход: `false`.
**Ограничения:** `1 <= s.length, t.length <= 5*10^4`. Строки состоят из строчных
латинских букв.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Valid Anagram](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0242.Valid%20Anagram/README.md)

Подход с массивом счётчиков: для каждого символа строки `s` увеличиваем счётчик,
для `t` уменьшаем. Если все счётчики равны нулю — анаграмма. Сложность — время O(n),
память O(1) (размер алфавита фиксирован).

```java
class Solution {
    public boolean isAnagram(String s, String t) {
        if (s.length() != t.length()) {
            return false;
        }
        int[] cnt = new int[26];
        for (int i = 0; i < s.length(); ++i) {
            ++cnt[s.charAt(i) - 'a'];
            --cnt[t.charAt(i) - 'a'];
        }
        for (int i = 0; i < 26; ++i) {
            if (cnt[i] != 0) {
                return false;
            }
        }
        return true;
    }
}
```

#### Ответ от Claude

**Идея.** Две строки являются анаграммами тогда и только тогда, когда имеют одинаковую
частоту каждого символа. Используем массив из 26 счётчиков (по одному на каждую букву).

```java
public boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) return false;
    int[] freq = new int[26];
    for (int i = 0; i < s.length(); i++) {
        freq[s.charAt(i) - 'a']++;
        freq[t.charAt(i) - 'a']--;
    }
    for (int count : freq) {
        if (count != 0) return false;
    }
    return true;
}
```

**Разбор по шагам.** Сначала проверяем длины — разные длины гарантируют не-анаграмму.
Затем в одном проходе одновременно считаем частоты обеих строк. Финальная проверка:
все счётчики должны быть нулевыми.

**Сложность.** Время O(n), память O(1) (алфавит фиксирован, массив из 26 элементов).

**Типичные ошибки / альтернативы.** Для Unicode-строк массив из 26 не подходит —
нужен `HashMap<Character, Integer>`. Ещё вариант: отсортировать обе строки и сравнить —
O(n log n) время.

### Вопрос 4. LC 349. Пересечение двух массивов (Intersection of Two Arrays)

**Категория:** Алгоритмы и структуры данных · Hash Set · **Уровень:** Easy

Даны два массива `nums1` и `nums2`. Верните массив их пересечения — уникальных
элементов, которые есть в обоих массивах. Порядок в ответе не важен.

**Пример 1.** Вход: `nums1 = [1,2,2,1]`, `nums2 = [2,2]` → Выход: `[2]`.
**Пример 2.** Вход: `nums1 = [4,9,5]`, `nums2 = [9,4,9,8,4]` → Выход: `[9,4]`.
**Ограничения:** `1 <= nums1.length, nums2.length <= 1000`, `0 <= nums[i] <= 1000`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Intersection of Two Arrays](https://github.com/doocs/leetcode/blob/main/solution/0300-0399/0349.Intersection%20of%20Two%20Arrays/README.md)

Подход с булевым массивом: отмечаем все элементы `nums1`, затем проходим `nums2`,
собирая совпадения и избегая дубликатов. Сложность — время O(n + m), память O(n).

```java
class Solution {
    public int[] intersection(int[] nums1, int[] nums2) {
        boolean[] s = new boolean[1001];
        for (int x : nums1) {
            s[x] = true;
        }
        List<Integer> ans = new ArrayList<>();
        for (int x : nums2) {
            if (s[x]) {
                ans.add(x);
                s[x] = false;
            }
        }
        return ans.stream().mapToInt(Integer::intValue).toArray();
    }
}
```

#### Ответ от Claude

**Идея.** Преобразуем `nums1` в `HashSet` для быстрой проверки принадлежности.
Затем проходим `nums2`, добавляя совпадающие элементы в результирующий набор
(чтобы избежать дублей).

```java
public int[] intersection(int[] nums1, int[] nums2) {
    Set<Integer> set1 = new HashSet<>();
    for (int x : nums1) set1.add(x);
    Set<Integer> result = new HashSet<>();
    for (int x : nums2) {
        if (set1.contains(x)) result.add(x);
    }
    int[] ans = new int[result.size()];
    int i = 0;
    for (int x : result) ans[i++] = x;
    return ans;
}
```

**Разбор по шагам.** Первый `HashSet` хранит уникальные значения `nums1`. При обходе
`nums2` каждый элемент проверяется за O(1). Второй `HashSet` для результата автоматически
исключает дубли из пересечения.

**Сложность.** Время O(n + m), память O(n + m).

**Типичные ошибки / альтернативы.** Решение из источника экономит память, используя
`boolean[]` вместо `HashSet` — возможно при гарантированном диапазоне значений 0–1000.
Также можно отсортировать оба массива и использовать два указателя: O((n+m) log(n+m))
время, O(1) доп. память.

### Вопрос 5. LC 169. Мажоритарный элемент (Majority Element)

**Категория:** Алгоритмы и структуры данных · Boyer–Moore · **Уровень:** Easy

Дан массив `nums` размером `n`. Найдите мажоритарный элемент — тот, что встречается
более `n/2` раз. Такой элемент всегда существует.

**Пример 1.** Вход: `nums = [3,2,3]` → Выход: `3`.
**Пример 2.** Вход: `nums = [2,2,1,1,1,2,2]` → Выход: `2`.
**Ограничения:** `n == nums.length`, `1 <= n <= 5*10^4`, `-10^9 <= nums[i] <= 10^9`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Majority Element](https://github.com/doocs/leetcode/blob/main/solution/0100-0199/0169.Majority%20Element/README.md)

Алгоритм голосования Бойера–Мура: поддерживаем кандидата и счётчик. Когда счётчик
достигает нуля — берём новый кандидат. Сложность — время O(n), память O(1).

```java
class Solution {
    public int majorityElement(int[] nums) {
        int cnt = 0, m = 0;
        for (int x : nums) {
            if (cnt == 0) {
                m = x;
                cnt = 1;
            } else {
                cnt += m == x ? 1 : -1;
            }
        }
        return m;
    }
}
```

#### Ответ от Claude

**Идея.** Алгоритм Бойера–Мура основан на инвариантe: при «вычёркивании» разнородных
пар элементов мажоритарный элемент обязательно выживет, так как он встречается чаще,
чем все остальные вместе взятые.

```java
public int majorityElement(int[] nums) {
    int candidate = nums[0];
    int count = 1;
    for (int i = 1; i < nums.length; i++) {
        if (count == 0) {
            candidate = nums[i];
            count = 1;
        } else if (nums[i] == candidate) {
            count++;
        } else {
            count--;
        }
    }
    return candidate;
}
```

**Разбор по шагам.** Первый элемент становится начальным кандидатом. При встрече
одинакового — увеличиваем счётчик, разного — уменьшаем. При достижении нуля смена
кандидата. Мажоритарный элемент «переживёт» все обнуления.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Если гарантии существования мажоритарного элемента
нет — требуется дополнительная верификационная проверка (второй проход с подсчётом).
Более простой вариант: HashMap с подсчётом частот — O(n) время, O(n) память.

### Вопрос 6. LC 88. Слияние отсортированных массивов (Merge Sorted Array)

**Категория:** Алгоритмы и структуры данных · Two Pointers · **Уровень:** Easy

Даны два отсортированных массива `nums1` (размер `m + n`, где первые `m` элементов
заполнены) и `nums2` (размер `n`). Слейте `nums2` в `nums1` на месте, сохраняя порядок.

**Пример 1.** Вход: `nums1 = [1,2,3,0,0,0]`, `m = 3`, `nums2 = [2,5,6]`, `n = 3`
→ Выход: `nums1 = [1,2,2,3,5,6]`.
**Ограничения:** `0 <= m, n <= 200`, `1 <= m + n <= 200`,
`-10^9 <= nums1[i], nums2[j] <= 10^9`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Merge Sorted Array](https://github.com/doocs/leetcode/blob/main/solution/0000-0099/0088.Merge%20Sorted%20Array/README.md)

Подход с двумя указателями с конца: заполняем `nums1` справа налево, выбирая наибольший
элемент из двух массивов. Сложность — время O(m + n), память O(1).

```java
class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        for (int i = m - 1, j = n - 1, k = m + n - 1; j >= 0; --k) {
            nums1[k] = i >= 0 && nums1[i] > nums2[j] ? nums1[i--] : nums2[j--];
        }
    }
}
```

#### Ответ от Claude

**Идея.** Если заполнять массив слева направо — нужно сдвигать элементы. Если идти
справа налево — заполняем гарантированно свободные ячейки в конце `nums1`.

```java
public void merge(int[] nums1, int m, int[] nums2, int n) {
    int i = m - 1;     // указатель на конец заполненной части nums1
    int j = n - 1;     // указатель на конец nums2
    int k = m + n - 1; // указатель на конец результата
    while (i >= 0 && j >= 0) {
        if (nums1[i] > nums2[j]) {
            nums1[k--] = nums1[i--];
        } else {
            nums1[k--] = nums2[j--];
        }
    }
    // если в nums2 ещё что-то осталось
    while (j >= 0) {
        nums1[k--] = nums2[j--];
    }
}
```

**Разбор по шагам.** Три указателя стартуют с конца своих областей. На каждом шаге
берём наибольший из двух «хвостов». Оставшиеся элементы `nums2` копируем напрямую
(остаток `nums1` уже на месте).

**Сложность.** Время O(m + n), память O(1).

**Типичные ошибки / альтернативы.** Частая ошибка — копирование слева направо, что
требует дополнительного буфера O(m). Решение из источника ещё компактнее: цикл
продолжается до `j >= 0`, что автоматически обрабатывает случай `i < 0`.

### Вопрос 7. LC 283. Перемещение нулей (Move Zeroes)

**Категория:** Алгоритмы и структуры данных · Two Pointers · **Уровень:** Easy

Дан целочисленный массив `nums`. Переместите все нули в конец, сохраняя относительный
порядок ненулевых элементов. Изменение должно быть выполнено in-place.

**Пример 1.** Вход: `nums = [0,1,0,3,12]` → Выход: `[1,3,12,0,0]`.
**Пример 2.** Вход: `nums = [0]` → Выход: `[0]`.
**Ограничения:** `1 <= nums.length <= 10^4`, `-2^31 <= nums[i] <= 2^31 - 1`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Move Zeroes](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0283.Move%20Zeroes/README.md)

Подход с двумя указателями: `k` отслеживает позицию для следующего ненулевого элемента.
При встрече ненулевого — меняем с позицией `k` и инкрементируем `k`. Сложность — O(n)
время, O(1) память.

```java
class Solution {
    public void moveZeroes(int[] nums) {
        int k = 0, n = nums.length;
        for (int i = 0; i < n; ++i) {
            if (nums[i] != 0) {
                int t = nums[i];
                nums[i] = nums[k];
                nums[k++] = t;
            }
        }
    }
}
```

#### Ответ от Claude

**Идея.** Указатель `k` делит массив на две части: `[0..k-1]` — уже обработанные
ненулевые элементы, `[k..i-1]` — нули. При встрече ненулевого — «проталкиваем» его
на позицию `k`.

```java
public void moveZeroes(int[] nums) {
    int k = 0; // позиция следующего ненулевого элемента
    for (int i = 0; i < nums.length; i++) {
        if (nums[i] != 0) {
            // swap nums[i] и nums[k]
            int tmp = nums[i];
            nums[i] = nums[k];
            nums[k] = tmp;
            k++;
        }
    }
}
```

**Разбор по шагам.** `k` начинается с 0. Когда `i` встречает ненулевой элемент —
он меняется местами с элементом на позиции `k`. Так все ненулевые элементы собираются
в начале в исходном порядке, нули сдвигаются в конец.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Если элемент на позиции `k` уже ненулевой (то
есть `i == k`), swap выполняется с самим собой — это безвредно, но можно добавить
проверку `if (i != k)` для экономии операций. Альтернатива: двухпроходный алгоритм —
сначала скопировать все ненулевые, затем дозаполнить нулями.

### Вопрос 8. LC 125. Палиндром (Valid Palindrome)

**Категория:** Алгоритмы и структуры данных · Two Pointers / строки · **Уровень:** Easy

Строка является палиндромом, если после удаления всех не-буквенно-цифровых символов
и приведения к нижнему регистру она читается одинаково в обоих направлениях.
Верните `true`, если строка `s` — палиндром.

**Пример 1.** Вход: `s = "A man, a plan, a canal: Panama"` → Выход: `true`.
**Пример 2.** Вход: `s = "race a car"` → Выход: `false`.
**Ограничения:** `1 <= s.length <= 2*10^5`. Строка состоит из ASCII-символов.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Valid Palindrome](https://github.com/doocs/leetcode/blob/main/solution/0100-0199/0125.Valid%20Palindrome/README.md)

Два указателя с концов строки, пропускающие не-буквенно-цифровые символы. При встрече
двух валидных символов сравниваем их без учёта регистра. Сложность — O(n) время, O(1)
память.

```java
class Solution {
    public boolean isPalindrome(String s) {
        int i = 0, j = s.length() - 1;
        while (i < j) {
            if (!Character.isLetterOrDigit(s.charAt(i))) {
                ++i;
            } else if (!Character.isLetterOrDigit(s.charAt(j))) {
                --j;
            } else if (Character.toLowerCase(s.charAt(i)) != Character.toLowerCase(s.charAt(j))) {
                return false;
            } else {
                ++i;
                --j;
            }
        }
        return true;
    }
}
```

#### Ответ от Claude

**Идея.** Два указателя движутся навстречу друг другу, пропуская «мусорные» символы.
Сравниваются только буквенно-цифровые символы в нижнем регистре.

```java
public boolean isPalindrome(String s) {
    int left = 0, right = s.length() - 1;
    while (left < right) {
        while (left < right && !Character.isLetterOrDigit(s.charAt(left))) {
            left++;
        }
        while (left < right && !Character.isLetterOrDigit(s.charAt(right))) {
            right--;
        }
        if (left < right) {
            if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) {
                return false;
            }
            left++;
            right--;
        }
    }
    return true;
}
```

**Разбор по шагам.** Внешний цикл продолжается, пока `left < right`. Оба внутренних
цикла перескакивают не-алфавитно-цифровые символы. Затем сравниваем символы в нижнем
регистре; несовпадение сразу возвращает `false`.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Не забыть проверку `left < right` во внутренних
циклах во избежание выхода за границы. Альтернатива: отфильтровать строку регулярным
выражением и сравнить с реверсом — O(n) время, O(n) память, но менее эффективно.

### Вопрос 9. LC 26. Удаление дубликатов из отсортированного массива (Remove Duplicates from Sorted Array)

**Категория:** Алгоритмы и структуры данных · Two Pointers · **Уровень:** Easy

Дан отсортированный массив `nums`. Удалите дубликаты in-place так, чтобы каждый
уникальный элемент встречался ровно один раз. Верните количество уникальных элементов `k`.

**Пример 1.** Вход: `nums = [1,1,2]` → Выход: `k = 2`, `nums = [1,2,_]`.
**Пример 2.** Вход: `nums = [0,0,1,1,1,2,2,3,3,4]` → Выход: `k = 5`, `nums = [0,1,2,3,4,_,_,_,_,_]`.
**Ограничения:** `1 <= nums.length <= 3*10^4`, `-100 <= nums[i] <= 100`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Remove Duplicates from Sorted Array](https://github.com/doocs/leetcode/blob/main/solution/0000-0099/0026.Remove%20Duplicates%20from%20Sorted%20Array/README.md)

Указатель `k` отслеживает позицию для записи следующего уникального элемента. Массив
отсортирован, поэтому дубликаты всегда рядом. Сложность — время O(n), память O(1).

```java
class Solution {
    public int removeDuplicates(int[] nums) {
        int k = 0;
        for (int x : nums) {
            if (k == 0 || x != nums[k - 1]) {
                nums[k++] = x;
            }
        }
        return k;
    }
}
```

#### Ответ от Claude

**Идея.** Поскольку массив отсортирован, дубликаты идут подряд. Медленный указатель `k`
отслеживает «конец» уникальной части. Быстрый указатель обходит весь массив.

```java
public int removeDuplicates(int[] nums) {
    int k = 0; // медленный указатель
    for (int i = 0; i < nums.length; i++) {
        // записываем элемент, если он первый или отличается от последнего уникального
        if (k == 0 || nums[i] != nums[k - 1]) {
            nums[k++] = nums[i];
        }
    }
    return k;
}
```

**Разбор по шагам.** `k` показывает, сколько уникальных элементов уже записано.
Условие `nums[i] != nums[k - 1]` работает только для отсортированного массива:
новый элемент добавляется лишь если он больше последнего записанного.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Для задачи «оставить не более двух дубликатов»
(LeetCode 80) условие меняется на `k < 2 || nums[i] != nums[k - 2]`. Следует помнить:
«удалённые» элементы в `nums[k..]` остаются в памяти, просто не учитываются.

### Вопрос 10. LC 27. Удаление элемента (Remove Element)

**Категория:** Алгоритмы и структуры данных · Two Pointers · **Уровень:** Easy

Дан массив `nums` и значение `val`. Удалите все вхождения `val` из `nums` in-place.
Верните количество элементов, не равных `val`.

**Пример 1.** Вход: `nums = [3,2,2,3]`, `val = 3` → Выход: `2`, `nums = [2,2,_,_]`.
**Пример 2.** Вход: `nums = [0,1,2,2,3,0,4,2]`, `val = 2` → Выход: `5`, `nums = [0,1,3,0,4,_,_,_]`.
**Ограничения:** `0 <= nums.length <= 100`, `0 <= nums[i] <= 50`, `0 <= val <= 100`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Remove Element](https://github.com/doocs/leetcode/blob/main/solution/0000-0099/0027.Remove%20Element/README.md)

Один проход: указатель `k` хранит число не-равных `val` элементов. Каждый
элемент, не равный `val`, записывается на позицию `k`. Сложность — O(n) время, O(1) память.

```java
class Solution {
    public int removeElement(int[] nums, int val) {
        int k = 0;
        for (int x : nums) {
            if (x != val) {
                nums[k++] = x;
            }
        }
        return k;
    }
}
```

#### Ответ от Claude

**Идея.** Когда значение `val` встречается редко, выгоднее не перезаписывать весь
массив слева направо, а заменять найденные вхождения элементами с конца — тогда число
записей пропорционально количеству удаляемых элементов, а не длине массива.
Используем два указателя: `left` движется слева в поисках `val`, `right` движется
справа в поисках не-`val`.

```java
public int removeElement(int[] nums, int val) {
    int left = 0, right = nums.length;
    // right — эксклюзивная правая граница «живой» части массива
    while (left < right) {
        if (nums[left] == val) {
            // заменяем найденное вхождение последним «живым» элементом
            nums[left] = nums[--right];
            // не двигаем left: только что скопированный элемент тоже надо проверить
        } else {
            left++;
        }
    }
    return right; // right == количество оставшихся элементов
}
```

**Разбор по шагам.** `right` изначально равно `nums.length` и служит эксклюзивной
правой границей. Когда `nums[left] == val`, копируем `nums[right - 1]` на позицию
`left` и уменьшаем `right`. `left` не продвигается, потому что скопированный элемент
тоже нужно проверить. Когда `nums[left] != val`, просто продвигаем `left`. Цикл
завершается при `left == right`; `right` и есть ответ.

**Сложность.** Время O(n), память O(1). Число операций записи равно числу удалённых
элементов — оптимально, если `val` редок. Порядок оставшихся элементов не сохраняется,
что допускается условием задачи.

**Типичные ошибки / альтернативы.** Стандартный однопроходный вариант (как в
«Оригинальном ответе») проще и сохраняет порядок, но делает столько записей, сколько
элементов не равно `val`. Если `val` встречается часто — это оптимальнее;
если редко — вариант swap-from-end экономит записи. Частая ошибка в swap-варианте:
после замены продвинуть `left` — тогда не-`val` элемент, только что скопированный
с хвоста, будет пропущен.

### Вопрос 11. LC 344. Разворот строки (Reverse String)

**Категория:** Алгоритмы и структуры данных · Two Pointers · **Уровень:** Easy

Дан массив символов `s`. Разверните его in-place без выделения дополнительной памяти.

**Пример 1.** Вход: `s = ['h','e','l','l','o']` → Выход: `['o','l','l','e','h']`.
**Пример 2.** Вход: `s = ['H','a','n','n','a','h']` → Выход: `['h','a','n','n','a','H']`.
**Ограничения:** `1 <= s.length <= 10^5`. `s[i]` — печатный ASCII-символ.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Reverse String](https://github.com/doocs/leetcode/blob/main/solution/0300-0399/0344.Reverse%20String/README.md)

Два указателя от концов к центру, обмен элементами. Сложность — O(n) время, O(1) память.

```java
class Solution {
    public void reverseString(char[] s) {
        for (int i = 0, j = s.length - 1; i < j; ++i, --j) {
            char t = s[i];
            s[i] = s[j];
            s[j] = t;
        }
    }
}
```

#### Ответ от Claude

**Идея.** Классическая задача на два указателя. Обмениваем крайние элементы и движемся
к центру.

```java
public void reverseString(char[] s) {
    int left = 0, right = s.length - 1;
    while (left < right) {
        char tmp = s[left];
        s[left] = s[right];
        s[right] = tmp;
        left++;
        right--;
    }
}
```

**Разбор по шагам.** На каждой итерации меняем местами `s[left]` и `s[right]`, затем
сужаем окно. Когда указатели встречаются или пересекаются — строка перевёрнута.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Попытка создать новый `String` из `char[]` и
использовать `StringBuilder.reverse()` нарушает требование O(1) памяти. XOR-обмен
(`s[i]^=s[j]; s[j]^=s[i]; s[i]^=s[j]`) для `char` синтаксически работает — `char`
является целочисленным типом в Java и поддерживает `^=`. Однако применять его не стоит:
он менее читаем, а при `i == j` обнуляет элемент (результат `x^x^x = x`, но
`x^=x` даёт `0`, после чего `x^=0` остаётся `0`). Обычный обмен через временную
переменную надёжнее и понятнее.

Итеративный подход с фиктивным узлом: сравниваем головы двух списков, присоединяем
меньший узел к результату. Сложность — O(m + n) время, O(1) память.

```java
class Solution {
    public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
        ListNode dummy = new ListNode();
        ListNode curr = dummy;
        while (list1 != null && list2 != null) {
            if (list1.val <= list2.val) {
                curr.next = list1;
                list1 = list1.next;
            } else {
                curr.next = list2;
                list2 = list2.next;
            }
            curr = curr.next;
        }
        curr.next = list1 == null ? list2 : list1;
        return dummy.next;
    }
}
```

#### Ответ от Claude

**Идея.** Фиктивный (dummy) узел упрощает обработку: не нужно проверять, пуст ли

```java
public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
    ListNode dummy = new ListNode(0);
    ListNode cur = dummy;
    while (list1 != null && list2 != null) {
        if (list1.val <= list2.val) {
            cur.next = list1;
            list1 = list1.next;
        } else {
            cur.next = list2;
            list2 = list2.next;

        cur = cur.next;

```

**Разбор по шагам.** Пока оба списка непусты — берём меньший элемент. После выхода из
цикла один список исчерпан; оставшийся хвост другого списка целиком присоединяем к

### Вопрос 13. LC 206. Разворот связного списка (Reverse Linked List)

**Категория:** Алгоритмы и структуры данных · Связные списки · **Уровень:** Easy

Дан односвязный список. Разверните его и верните новую голову.

**Пример 1.** Вход: `head = [1,2,3,4,5]` → Выход: `[5,4,3,2,1]`.
**Пример 2.** Вход: `head = [1,2]` → Выход: `[2,1]`.
**Ограничения:** Число узлов: `[0, 5000]`. `-5000 <= Node.val <= 5000`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Reverse Linked List](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0206.Reverse%20Linked%20List/README.md)

Итеративный подход с фиктивным узлом: вставка в начало (head insertion).
Сложность — O(n) время, O(1) память.

```java
class Solution {
    public ListNode reverseList(ListNode head) {
        ListNode dummy = new ListNode();
        ListNode curr = head;
        while (curr != null) {
            ListNode next = curr.next;
            curr.next = dummy.next;
            dummy.next = curr;
            curr = next;
        }
        return dummy.next;
    }
}
```

#### Ответ от Claude

**Идея.** При итеративном развороте нужно перенаправить стрелку каждого узла на
предыдущий. Храним три ссылки: `prev`, `curr`, `next`.

```java
public ListNode reverseList(ListNode head) {
    ListNode prev = null;
    ListNode curr = head;
    while (curr != null) {
        ListNode next = curr.next; // сохраняем следующий
        curr.next = prev;          // разворачиваем стрелку
        prev = curr;               // сдвигаем prev
        curr = next;               // сдвигаем curr
    }
    return prev; // prev — новая голова
}
```

**Разбор по шагам.** На каждой итерации: сохраняем `curr.next`, меняем направление
ссылки `curr.next = prev`, сдвигаем `prev` и `curr` вперёд. Когда `curr == null`,
`prev` указывает на бывший хвост — новую голову.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Ключевая ошибка — не сохранить `curr.next` до
перенаправления ссылки (потеря хвоста). Рекурсивный вариант (`reverseList(head.next)`)
элегантен, но тратит O(n) стека.

Один проход: если следующий узел имеет то же значение — перескакиваем через него.
Сложность — O(n) время, O(1) память.

```java
class Solution {
    public ListNode deleteDuplicates(ListNode head) {

        while (cur != null && cur.next != null) {
            if (cur.val == cur.next.val) {
                cur.next = cur.next.next;
            } else {
                cur = cur.next;
            }
```

```java
public ListNode deleteDuplicates(ListNode head) {
```

### Вопрос 15. LC 234. Палиндром-список (Palindrome Linked List)

**Категория:** Алгоритмы и структуры данных · Связные списки / Two Pointers · **Уровень:** Easy

Дан связный список. Верните `true`, если он является палиндромом.

**Пример 1.** Вход: `head = [1,2,2,1]` → Выход: `true`.
**Пример 2.** Вход: `head = [1,2]` → Выход: `false`.
**Ограничения:** Число узлов: `[1, 10^5]`. `0 <= Node.val <= 9`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Palindrome Linked List](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0234.Palindrome%20Linked%20List/README.md)

Подход: найти середину (fast/slow указатели), развернуть вторую половину, сравнить
с первой. Сложность — O(n) время, O(1) память.

```java
class Solution {
    public boolean isPalindrome(ListNode head) {
        ListNode slow = head, fast = head.next;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
        }
        ListNode cur = slow.next;
        slow.next = null;
        ListNode pre = null;
        while (cur != null) {
            ListNode t = cur.next;
            cur.next = pre;
            pre = cur;
            cur = t;
        }
        while (pre != null) {
            if (pre.val != head.val) return false;
            pre = pre.next;
            head = head.next;
        }
        return true;
    }
}
```

#### Ответ от Claude

**Идея.** Три шага: (1) найти середину с помощью быстрого и медленного указателей;
(2) развернуть вторую половину списка; (3) сравнить первую и вторую половины поэлементно.

```java
public boolean isPalindrome(ListNode head) {
    // шаг 1: найти конец первой половины
    ListNode slow = head, fast = head;
    while (fast.next != null && fast.next.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    // шаг 2: развернуть вторую половину
    ListNode prev = null, cur = slow.next;
    while (cur != null) {
        ListNode next = cur.next;
        cur.next = prev;
        prev = cur;
        cur = next;
    }
    // шаг 3: сравнить половины
    ListNode left = head, right = prev;
    while (right != null) {
        if (left.val != right.val) return false;
        left = left.next;
        right = right.next;
    }
    return true;
}
```

**Разбор по шагам.** `slow` останавливается на конце первой половины. `prev` после
разворота указывает на бывший хвост (начало развёрнутой второй половины). Сравниваем
две части пока `right != null`.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Простое решение — скопировать значения в массив
и проверить обычный палиндром: O(n) время, O(n) память. Ключевая ошибка in-place решения:
неверная остановка slow-указателя при чётном/нечётном числе элементов.

### Вопрос 16. LC 203. Удаление элементов списка (Remove Linked List Elements)

**Категория:** Алгоритмы и структуры данных · Связные списки · **Уровень:** Easy

Дан связный список и целое число `val`. Удалите все узлы со значением `val`,
верните новую голову списка.

**Пример 1.** Вход: `head = [1,2,6,3,4,5,6]`, `val = 6` → Выход: `[1,2,3,4,5]`.
**Пример 2.** Вход: `head = [7,7,7,7]`, `val = 7` → Выход: `[]`.
**Ограничения:** Число узлов: `[0, 10^4]`. `1 <= Node.val <= 50`. `0 <= val <= 50`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Remove Linked List Elements](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0203.Remove%20Linked%20List%20Elements/README.md)

Фиктивный узел (dummy) перед головой — элегантно обрабатывает удаление головы.
Сложность — O(n) время, O(1) память.

```java
class Solution {
    public ListNode removeElements(ListNode head, int val) {
        ListNode dummy = new ListNode(-1, head);
        ListNode pre = dummy;
        while (pre.next != null) {
            if (pre.next.val != val)
                pre = pre.next;
            else
                pre.next = pre.next.next;
        }
        return dummy.next;
    }
}
```

#### Ответ от Claude

**Идея.** Фиктивный узел перед `head` снимает необходимость специальной обработки
удаления первого узла. `pre` указывает на узел перед кандидатом на удаление.

```java
public ListNode removeElements(ListNode head, int val) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode prev = dummy;
    while (prev.next != null) {
        if (prev.next.val == val) {
            prev.next = prev.next.next; // удаляем узел
        } else {
            prev = prev.next; // двигаемся только если не удалили
        }
    }
    return dummy.next;
}
```

**Разбор по шагам.** `prev` не продвигается при удалении — это позволяет удалять
несколько подряд идущих совпадающих узлов. Только при «пропуске» перемещаемся вперёд.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Без фиктивного узла нужно отдельно обрабатывать
начало списка: `while (head != null && head.val == val) head = head.next;`. Фиктивный
узел устраняет этот специальный случай.

### Вопрос 17. LC 35. Позиция вставки (Search Insert Position)

**Категория:** Алгоритмы и структуры данных · Бинарный поиск · **Уровень:** Easy

Дан отсортированный массив `nums` и целевое значение `target`. Найдите позицию `target`
в массиве, или позицию, на которую его нужно вставить для сохранения порядка.

**Пример 1.** Вход: `nums = [1,3,5,6]`, `target = 5` → Выход: `2`.
**Пример 2.** Вход: `nums = [1,3,5,6]`, `target = 2` → Выход: `1`.
**Пример 3.** Вход: `nums = [1,3,5,6]`, `target = 7` → Выход: `4`.
**Ограничения:** `1 <= nums.length <= 10^4`. `-10^4 <= nums[i], target <= 10^4`. Без дубликатов.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Search Insert Position](https://github.com/doocs/leetcode/blob/main/solution/0000-0099/0035.Search%20Insert%20Position/README.md)

Классический бинарный поиск левой границы: ищем наименьший индекс `i` такой, что
`nums[i] >= target`. Сложность — O(log n) время, O(1) память.

```java
class Solution {
    public int searchInsert(int[] nums, int target) {
        int l = 0, r = nums.length;
        while (l < r) {
            int mid = (l + r) >>> 1;
            if (nums[mid] >= target) {
                r = mid;
            } else {
                l = mid + 1;
            }
        }
        return l;
    }
}
```

#### Ответ от Claude

**Идея.** Задача сводится к поиску левой границы: первого индекса с `nums[i] >= target`.
Если `target` не найден — этот индекс и есть позиция вставки.

```java
public int searchInsert(int[] nums, int target) {
    int lo = 0, hi = nums.length; // hi = length (не length-1!)
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] < target) {
            lo = mid + 1;
        } else {
            hi = mid;
        }
    }
    return lo;
}
```

**Разбор по шагам.** Правая граница `hi = nums.length` (за пределами массива)
учитывает случай, когда `target` больше всех элементов. Инвариант: ответ всегда в
`[lo, hi]`. Когда `lo == hi` — нашли позицию.

**Сложность.** Время O(log n), память O(1).

**Типичные ошибки / альтернативы.** Частая ошибка: `hi = nums.length - 1` — тогда
пропускается случай вставки в конец. Использование `(l + r) / 2` может вызвать переполнение
для больших значений, безопаснее `l + (r - l) / 2` или `(l + r) >>> 1`.

### Вопрос 18. LC 278. Первая плохая версия (First Bad Version)

**Категория:** Алгоритмы и структуры данных · Бинарный поиск · **Уровень:** Easy

У продукта `n` версий с номерами `1..n`. Начиная с некоторой версии `k`, все
последующие плохие. Дан API `isBadVersion(version)`. Найдите первую плохую версию
с минимальным числом вызовов API.

**Пример 1.** `n = 5`, первая плохая = 4 → вызовы: `isBadVersion(3)=false`, `isBadVersion(5)=true`, `isBadVersion(4)=true` → Выход: `4`.
**Ограничения:** `1 <= bad <= n <= 2^31 - 1`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — First Bad Version](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0278.First%20Bad%20Version/README.md)

Бинарный поиск левой границы: если версия `mid` плохая — ищем левее (может быть ещё
более ранняя плохая). Сложность — O(log n) время, O(1) память.

```java
public class Solution extends VersionControl {
    public int firstBadVersion(int n) {
        int l = 1, r = n;
        while (l < r) {
            int mid = (l + r) >>> 1;
            if (isBadVersion(mid)) {
                r = mid;
            } else {
                l = mid + 1;
            }
        }
        return l;
    }
}
```

#### Ответ от Claude

**Идея.** Версии образуют монотонную последовательность: `[good, good, ..., bad, bad, ..., bad]`.
Задача — найти первый `bad`. Бинарный поиск находит левую границу перехода.

```java
public int firstBadVersion(int n) {
    int lo = 1, hi = n;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2; // защита от переполнения
        if (isBadVersion(mid)) {
            hi = mid; // mid может быть первой плохой
        } else {
            lo = mid + 1; // mid точно хорошая
        }
    }
    return lo;
}
```

**Разбор по шагам.** Диапазон `[lo, hi]` всегда содержит ответ. При `isBadVersion(mid) == true`
сужаем к левой половине, включая `mid`. При `false` — исключаем `mid` из диапазона.

**Сложность.** Время O(log n), память O(1).

**Типичные ошибки / альтернативы.** Вычисление `mid = (l + r) / 2` опасно: при `n = 2^31 - 1`
сумма `l + r` переполняет `int`. Безопасные варианты: `l + (r - l) / 2` или беззнаковый
сдвиг `(l + r) >>> 1`.

### Вопрос 19. LC 367. Полный квадрат (Valid Perfect Square)

**Категория:** Алгоритмы и структуры данных · Бинарный поиск / Math · **Уровень:** Easy

Дано положительное целое число `num`. Верните `true`, если `num` является полным
квадратом (то есть квадратом какого-либо целого числа). Нельзя использовать встроенные
функции типа `sqrt`.

**Пример 1.** Вход: `num = 16` → Выход: `true` (4*4 = 16).
**Пример 2.** Вход: `num = 14` → Выход: `false`.
**Ограничения:** `1 <= num <= 2^31 - 1`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Valid Perfect Square](https://github.com/doocs/leetcode/blob/main/solution/0300-0399/0367.Valid%20Perfect%20Square/README.md)

Бинарный поиск в диапазоне `[1, num]`: ищем `mid` такое, что `mid * mid == num`.
Для предотвращения переполнения используется тип `long`. Сложность — O(log n) время,
O(1) память.

```java
class Solution {
    public boolean isPerfectSquare(int num) {
        int l = 1, r = num;
        while (l < r) {
            int mid = (l + r) >>> 1;
            if (1L * mid * mid >= num) {
                r = mid;
            } else {
                l = mid + 1;
            }
        }
        return l * l == num;
    }
}
```

#### Ответ от Claude

**Идея.** Ищем целое `x` такое, что `x*x == num`. Бинарным поиском сужаем диапазон
от `1` до `num`. Важно использовать `long` для промежуточных вычислений.

```java
public boolean isPerfectSquare(int num) {
    long lo = 1, hi = num;
    while (lo <= hi) {
        long mid = lo + (hi - lo) / 2;
        long sq = mid * mid;
        if (sq == num) {
            return true;
        } else if (sq < num) {
            lo = mid + 1;
        } else {
            hi = mid - 1;
        }
    }
    return false;
}
```

**Разбор по шагам.** Переменные `long` предотвращают переполнение при `num ~ 2^31`.
Классический бинарный поиск: при `mid*mid < num` уходим вправо, при `mid*mid > num`
влево, при совпадении возвращаем `true`.

**Сложность.** Время O(log n), память O(1).

**Типичные ошибки / альтернативы.** Ключевая ошибка — `int mid * int mid` переполняется
при больших значениях: `46341 * 46341 = 2147488281 > Integer.MAX_VALUE`. Альтернатива:
математический подход — полные квадраты — это суммы последовательных нечётных чисел
(1, 1+3=4, 4+5=9, ...): вычитаем `1, 3, 5, ...` пока `num > 0`, проверяем `num == 0`.

### Вопрос 20. LC 104. Максимальная глубина бинарного дерева (Maximum Depth of Binary Tree)

**Категория:** Алгоритмы и структуры данных · Деревья / DFS · **Уровень:** Easy

Дано бинарное дерево. Найдите его максимальную глубину — число узлов вдоль
наидлиннейшего пути от корня до листа.

**Пример 1.** Дерево `[3,9,20,null,null,15,7]` → Выход: `3`.
**Пример 2.** Дерево `[1,null,2]` → Выход: `2`.
**Ограничения:** Число узлов: `[0, 10^4]`. `-100 <= Node.val <= 100`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Maximum Depth of Binary Tree](https://github.com/doocs/leetcode/blob/main/solution/0100-0199/0104.Maximum%20Depth%20of%20Binary%20Tree/README.md)

Рекурсивный DFS: глубина дерева = 1 + max(глубина левого поддерева, глубина правого
поддерева). База рекурсии: `null` → 0. Сложность — O(n) время, O(h) память (h — высота).

```java
class Solution {
    public int maxDepth(TreeNode root) {
        if (root == null) {
            return 0;
        }
        int l = maxDepth(root.left);
        int r = maxDepth(root.right);
        return 1 + Math.max(l, r);
    }
}
```

#### Ответ от Claude

**Идея.** DFS (обход в глубину) естественно вычисляет глубину: глубина узла = 1 +
наибольшая глубина его поддеревьев. Рекурсия отражает структуру задачи напрямую.

```java
public int maxDepth(TreeNode root) {
    if (root == null) return 0;
    int leftDepth = maxDepth(root.left);
    int rightDepth = maxDepth(root.right);
    return 1 + Math.max(leftDepth, rightDepth);
}
```

**Разбор по шагам.** Базовый случай: `null` — глубина 0. Для каждого узла рекурсивно
находим глубины левого и правого поддеревьев. Добавляем 1 (текущий узел) к
максимальному из них.

**Сложность.** Время O(n) — каждый узел посещается один раз. Память O(h) — стек
рекурсии, где h — высота дерева (O(log n) для сбалансированного, O(n) для вырожденного).

**Типичные ошибки / альтернативы.** Итеративный BFS (обход в ширину) с подсчётом
уровней: O(n) время, O(w) память (w — максимальная ширина уровня). При очень глубоких
деревьях рекурсия может вызвать `StackOverflowError` — тогда лучше BFS. Итеративный DFS
через явный стек тоже возможен: O(n) время, O(h) память.
