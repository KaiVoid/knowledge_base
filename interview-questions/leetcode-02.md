### Вопрос 21. LC 226. Инвертировать дерево (Invert Binary Tree)

**Категория:** Алгоритмы и структуры данных · Деревья · **Уровень:** Easy

Дано бинарное дерево. Инвертируйте его (зеркально отразите) и верните корень.

**Пример 1.** Дерево `[4,2,7,1,3,6,9]` → Выход: `[4,7,2,9,6,3,1]`.
**Пример 2.** Дерево `[2,1,3]` → Выход: `[2,3,1]`.
**Ограничения:** Число узлов: `[0, 100]`. `-100 <= Node.val <= 100`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Invert Binary Tree](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0226.Invert%20Binary%20Tree/README.md)

Рекурсивный подход: инвертируем левое и правое поддеревья, затем меняем их местами
у текущего узла. Сложность — время O(n), память O(h) (h — высота дерева).

```java
class Solution {
    public TreeNode invertTree(TreeNode root) {
        if (root == null) {
            return null;
        }
        TreeNode l = invertTree(root.left);
        TreeNode r = invertTree(root.right);
        root.left = r;
        root.right = l;
        return root;
    }
}
```

#### Ответ от Claude

**Идея.** Инверсия дерева — это рекурсивный обмен: для каждого узла меняем левого и
правого детей местами, затем рекурсивно инвертируем каждое поддерево. Можно идти
«сверху вниз» (сначала swap, потом рекурсия) или «снизу вверх» (как в источнике).

```java
public TreeNode invertTree(TreeNode root) {
    if (root == null) return null;
    // сначала меняем детей, потом рекурсивно опускаемся
    TreeNode tmp = root.left;
    root.left = root.right;
    root.right = tmp;
    invertTree(root.left);
    invertTree(root.right);
    return root;
}
```

**Разбор по шагам.** Базовый случай: `null` — возвращаем `null`. Для каждого узла
меняем `left` и `right` ссылки (swap через временную переменную), затем рекурсивно
спускаемся в оба поддерева.

**Сложность.** Время O(n) — каждый узел посещается один раз. Память O(h) — стек
рекурсии, где h — высота дерева (O(log n) для сбалансированного, O(n) для вырожденного).

**Типичные ошибки / альтернативы.** Решение из источника идёт «снизу вверх»: сначала
рекурсивно инвертирует поддеревья, потом присваивает результаты. Оба подхода корректны.
Итеративный вариант: BFS с очередью — берём каждый узел, меняем его детей местами,
добавляем детей в очередь. Сложность та же: O(n) время, O(w) память (w — ширина уровня).

### Вопрос 22. LC 100. Одинаковые деревья (Same Tree)

**Категория:** Алгоритмы и структуры данных · Деревья · **Уровень:** Easy

Даны корни двух бинарных деревьев `p` и `q`. Верните `true`, если деревья структурно
идентичны и все соответствующие узлы имеют одинаковые значения.

**Пример 1.** Деревья `[1,2,3]` и `[1,2,3]` → Выход: `true`.
**Пример 2.** Деревья `[1,2]` и `[1,null,2]` → Выход: `false`.
**Ограничения:** Число узлов в каждом дереве: `[0, 100]`. `-10^4 <= Node.val <= 10^4`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Same Tree](https://github.com/doocs/leetcode/blob/main/solution/0100-0199/0100.Same%20Tree/README.md)

Рекурсивный DFS: если оба `null` — одинаковые; если один `null` или значения различны —
нет. Иначе рекурсивно проверяем оба поддерева. Сложность — O(n) время, O(h) память.

```java
class Solution {
    public boolean isSameTree(TreeNode p, TreeNode q) {
        if (p == q) return true;
        if (p == null || q == null || p.val != q.val) return false;
        return isSameTree(p.left, q.left) && isSameTree(p.right, q.right);
    }
}
```

#### Ответ от Claude

**Идея.** Два дерева одинаковы, если: (1) оба пусты, или (2) оба непусты, их корни
имеют равные значения, и их левые поддеревья одинаковы, и их правые поддеревья одинаковы.

```java
public boolean isSameTree(TreeNode p, TreeNode q) {
    // оба null — идентичны
    if (p == null && q == null) return true;
    // один null, другой нет — различны
    if (p == null || q == null) return false;
    // значения различаются — различны
    if (p.val != q.val) return false;
    // рекурсивно проверяем поддеревья
    return isSameTree(p.left, q.left) && isSameTree(p.right, q.right);
}
```

**Разбор по шагам.** Три условия завершения рекурсии: (1) оба `null` → `true`;
(2) один `null` → `false`; (3) значения различны → `false`. Оператор `&&` обеспечивает
ранний выход: если левые поддеревья различны, правые не проверяются.

**Сложность.** Время O(min(n, m)) — останавливаемся при первом расхождении. Память
O(min(h1, h2)) — глубина рекурсии ограничена меньшим деревом.

**Типичные ошибки / альтернативы.** Решение из источника компактнее: `if (p == q) return true`
сразу обрабатывает случай двух `null` (оба — `null`-ссылки равны). Итеративный BFS:
сравниваем узлы попарно уровень за уровнем — O(n) время, O(w) память.

### Вопрос 23. LC 572. Поддерево другого дерева (Subtree of Another Tree)

**Категория:** Алгоритмы и структуры данных · Деревья · **Уровень:** Easy

Даны корни двух бинарных деревьев `root` и `subRoot`. Верните `true`, если в дереве
`root` существует поддерево, структурно идентичное `subRoot`.

**Пример 1.** `root = [3,4,5,1,2]`, `subRoot = [4,1,2]` → Выход: `true`.
**Пример 2.** `root = [3,4,5,1,2,null,null,null,null,0]`, `subRoot = [4,1,2]` → Выход: `false`.
**Ограничения:** Узлов в `root`: `[1, 2000]`. Узлов в `subRoot`: `[1, 1000]`.
`-10^4 <= Node.val <= 10^4`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Subtree of Another Tree](https://github.com/doocs/leetcode/blob/main/solution/0500-0599/0572.Subtree%20of%20Another%20Tree/README.md)

DFS + вспомогательная функция проверки идентичности деревьев. Для каждого узла `root`
проверяем, совпадает ли поддерево с `subRoot`. Сложность — O(n * m) время, O(h) память.

```java
class Solution {
    public boolean isSubtree(TreeNode root, TreeNode subRoot) {
        if (root == null) {
            return false;
        }
        return same(root, subRoot) || isSubtree(root.left, subRoot)
            || isSubtree(root.right, subRoot);
    }

    private boolean same(TreeNode p, TreeNode q) {
        if (p == null || q == null) {
            return p == q;
        }
        return p.val == q.val && same(p.left, q.left) && same(p.right, q.right);
    }
}
```

#### Ответ от Claude

**Идея.** Задача строится на задаче «одинаковые деревья» (LC 100). Для каждого узла
главного дерева проверяем: не является ли поддерево с этим корнем идентичным `subRoot`.
Обход DFS гарантирует проверку всех узлов.

```java
public boolean isSubtree(TreeNode root, TreeNode subRoot) {
    if (root == null) return false;
    // проверяем текущий узел как возможный корень совпадения
    if (isSameTree(root, subRoot)) return true;
    // рекурсивно проверяем левое и правое поддеревья
    return isSubtree(root.left, subRoot) || isSubtree(root.right, subRoot);
}

private boolean isSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) return true;
    if (p == null || q == null) return false;
    return p.val == q.val && isSameTree(p.left, q.left) && isSameTree(p.right, q.right);
}
```

**Разбор по шагам.** Для каждого узла `root` вызываем `isSameTree`. Если совпадение
найдено — сразу `true` (оператор `||` обеспечивает ранний выход). Если `root` исчерпан
(`null`) — `false`.

**Сложность.** Время O(n * m), где n — узлов в `root`, m — узлов в `subRoot` (в худшем
случае). Память O(h) — глубина стека рекурсии по основному дереву.

**Типичные ошибки / альтернативы.** Более эффективная альтернатива — хэширование
поддеревьев или алгоритм KMP на сериализованных строках деревьев: O(n + m) время.
Частая ошибка: не обрабатывать случай `subRoot == null` — по условию задачи `subRoot`
всегда непустой, но стоит помнить об этом при обобщении.

### Вопрос 24. LC 112. Сумма пути (Path Sum)

**Категория:** Алгоритмы и структуры данных · Деревья / DFS · **Уровень:** Easy

Дано бинарное дерево и целевая сумма `targetSum`. Верните `true`, если существует
путь от корня до листа, сумма узлов которого равна `targetSum`.

**Пример 1.** Дерево `[5,4,8,11,null,13,4,7,2,null,null,null,1]`, `targetSum = 22`
→ Выход: `true` (путь 5→4→11→2).
**Пример 2.** Дерево `[1,2,3]`, `targetSum = 5` → Выход: `false`.
**Ограничения:** Число узлов: `[0, 5000]`. `-1000 <= Node.val <= 1000`.
`-1000 <= targetSum <= 1000`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Path Sum](https://github.com/doocs/leetcode/blob/main/solution/0100-0199/0112.Path%20Sum/README.md)

DFS с вычитанием текущего значения из целевой суммы. На листовом узле проверяем,
стала ли оставшаяся сумма нулём. Сложность — O(n) время, O(h) память.

```java
class Solution {
    public boolean hasPathSum(TreeNode root, int targetSum) {
        return dfs(root, targetSum);
    }

    private boolean dfs(TreeNode root, int s) {
        if (root == null) {
            return false;
        }
        s -= root.val;
        if (root.left == null && root.right == null && s == 0) {
            return true;
        }
        return dfs(root.left, s) || dfs(root.right, s);
    }
}
```

#### Ответ от Claude

**Идея.** На каждом шаге вычитаем значение текущего узла из оставшейся суммы. Когда
достигаем листа (оба ребёнка `null`), проверяем, равна ли оставшаяся сумма нулю.

```java
public boolean hasPathSum(TreeNode root, int targetSum) {
    if (root == null) return false;
    // лист: проверяем остаток
    if (root.left == null && root.right == null) {
        return root.val == targetSum;
    }
    int remaining = targetSum - root.val;
    return hasPathSum(root.left, remaining) || hasPathSum(root.right, remaining);
}
```

**Разбор по шагам.** Базовый случай 1: пустой узел — пути нет (`false`). Базовый случай 2:
лист — проверяем `root.val == targetSum`. Рекурсивный случай: вычитаем `root.val` и
передаём остаток дальше. Оператор `||` обеспечивает ранний выход при нахождении пути.

**Сложность.** Время O(n) — в худшем случае посещаем все узлы. Память O(h) — стек
рекурсии. Для сбалансированного дерева h = O(log n), для вырожденного h = O(n).

**Типичные ошибки / альтернативы.** Решение из источника передаёт уменьшаемую сумму `s`
в параметре — стиль немного отличается, но суть та же. Важно: «путь» — только
корень-до-листа, а не любой путь в дереве. Итеративный вариант: DFS через явный стек
с парами `(узел, оставшаяся сумма)`.

### Вопрос 25. LC 20. Корректные скобки (Valid Parentheses)

**Категория:** Алгоритмы и структуры данных · Стек · **Уровень:** Easy

Дана строка `s`, содержащая символы `(`, `)`, `{`, `}`, `[`, `]`. Верните `true`,
если строка корректно скобочная: каждая открывающая скобка закрывается соответствующей
закрывающей в правильном порядке.

**Пример 1.** Вход: `s = "()"` → Выход: `true`.
**Пример 2.** Вход: `s = "()[]{}"` → Выход: `true`.
**Пример 3.** Вход: `s = "(]"` → Выход: `false`.
**Ограничения:** `1 <= s.length <= 10^4`. `s` содержит только `()[]{}`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Valid Parentheses](https://github.com/doocs/leetcode/blob/main/solution/0000-0099/0020.Valid%20Parentheses/README.md)

Стек: открывающие скобки кладём на стек, для закрывающих проверяем, совпадает ли
верхушка стека с соответствующей открывающей. В конце стек должен быть пуст.
Сложность — O(n) время, O(n) память.

```java
class Solution {
    public boolean isValid(String s) {
        Deque<Character> stk = new ArrayDeque<>();
        for (char c : s.toCharArray()) {
            if (c == '(' || c == '{' || c == '[') {
                stk.push(c);
            } else if (stk.isEmpty() || !match(stk.pop(), c)) {
                return false;
            }
        }
        return stk.isEmpty();
    }

    private boolean match(char l, char r) {
        return (l == '(' && r == ')') || (l == '{' && r == '}') || (l == '[' && r == ']');
    }
}
```

#### Ответ от Claude

**Идея.** Стек хранит незакрытые открывающие скобки. При встрече закрывающей —
снимаем вершину стека и проверяем совпадение. Используем `Map` для хранения пар
скобок — код становится чище, чем цепочка `||`.

```java
public boolean isValid(String s) {
    Map<Character, Character> pairs = new HashMap<>();
    pairs.put(')', '(');
    pairs.put(']', '[');
    pairs.put('}', '{');

    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (!pairs.containsKey(c)) {
            // открывающая скобка — на стек
            stack.push(c);
        } else {
            // закрывающая: стек пуст или вершина не совпадает
            if (stack.isEmpty() || stack.pop() != pairs.get(c)) {
                return false;
            }
        }
    }
    return stack.isEmpty();
}
```

**Разбор по шагам.** Словарь `pairs` отображает закрывающую скобку на соответствующую
открывающую. При встрече закрывающей извлекаем вершину стека: если стек пуст или
вершина не совпадает с ожидаемой — строка некорректна. В конце стек должен быть пуст
(все открытые скобки закрыты).

**Сложность.** Время O(n), память O(n) — в худшем случае (все открывающие) стек вмещает
всю строку.

**Типичные ошибки / альтернативы.** Нельзя пропускать проверку `stack.isEmpty()` перед
`stack.pop()` — иначе `EmptyDeque` при строке вида `"]"`. Альтернатива без `Map`: при
встрече открывающей скобки сразу кладём на стек соответствующую ей закрывающую, тогда
сравнение становится прямым `c != stack.pop()`.

### Вопрос 26. LC 232. Очередь на двух стеках (Implement Queue using Stacks)

**Категория:** Алгоритмы и структуры данных · Стек / очередь / дизайн · **Уровень:** Easy

Реализуйте очередь FIFO, используя только два стека. Реализуйте операции `push`, `pop`,
`peek` и `empty`. Операции `pop` и `peek` гарантированно вызываются на непустой очереди.

**Пример.** `MyQueue q = new MyQueue(); q.push(1); q.push(2); q.peek()` → `1`;
`q.pop()` → `1`; `q.empty()` → `false`.
**Ограничения:** `1 <= x <= 9`. Не более `100` вызовов. `pop`/`peek` — на непустой очереди.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Implement Queue using Stacks](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0232.Implement%20Queue%20using%20Stacks/README.md)

Два стека: `stk1` для `push`, `stk2` для `pop`/`peek`. Перекладываем из `stk1` в `stk2`
лениво (только когда `stk2` пуст). Сложность — `push` O(1), `pop`/`peek` амортизировано O(1).

```java
class MyQueue {
    private Deque<Integer> stk1 = new ArrayDeque<>();
    private Deque<Integer> stk2 = new ArrayDeque<>();

    public MyQueue() {
    }

    public void push(int x) {
        stk1.push(x);
    }

    public int pop() {
        move();
        return stk2.pop();
    }

    public int peek() {
        move();
        return stk2.peek();
    }

    public boolean empty() {
        return stk1.isEmpty() && stk2.isEmpty();
    }

    private void move() {
        while (stk2.isEmpty()) {
            while (!stk1.isEmpty()) {
                stk2.push(stk1.pop());
            }
        }
    }
}
```

#### Ответ от Claude

**Идея.** Стек инвертирует порядок: первый вошедший оказывается последним. Если
перелить стек в другой стек — порядок восстанавливается. Ленивый перенос (только когда
выходной стек пуст) даёт амортизированную O(1) сложность для `pop`/`peek`.

```java
class MyQueue {
    private final Deque<Integer> inbox  = new ArrayDeque<>(); // стек для push
    private final Deque<Integer> outbox = new ArrayDeque<>(); // стек для pop/peek

    public void push(int x) {
        inbox.push(x); // O(1)
    }

    public int pop() {
        transfer();
        return outbox.pop(); // амортизировано O(1)
    }

    public int peek() {
        transfer();
        return outbox.peek(); // амортизировано O(1)
    }

    public boolean empty() {
        return inbox.isEmpty() && outbox.isEmpty();
    }

    // перекладываем только когда outbox пуст
    private void transfer() {
        if (outbox.isEmpty()) {
            while (!inbox.isEmpty()) {
                outbox.push(inbox.pop());
            }
        }
    }
}
```

**Разбор по шагам.** Все новые элементы идут в `inbox`. При `pop`/`peek`: если `outbox`
пуст — переливаем весь `inbox` в `outbox` (порядок обращается дважды → FIFO).
Если `outbox` непуст — просто используем его верхушку.

**Сложность.** `push` O(1). `pop`/`peek` амортизировано O(1): каждый элемент
перекладывается ровно один раз за время жизни. Память O(n).

**Типичные ошибки / альтернативы.** Решение из источника отличается лишь именами
переменных и условием `while (stk2.isEmpty())` вместо `if (outbox.isEmpty())` — оба
эквивалентны, поскольку внутренний цикл перекладывает всё. Частая ошибка: пытаться
перекладывать при каждом вызове, не проверяя, пуст ли outbox — это ухудшает сложность.

### Вопрос 27. LC 13. Римское число в целое (Roman to Integer)

**Категория:** Алгоритмы и структуры данных · Строки / Hash Table · **Уровень:** Easy

Дана строка `s`, представляющая римское число в диапазоне `[1, 3999]`. Переведите
его в целое число. Римские цифры: `I=1, V=5, X=10, L=50, C=100, D=500, M=1000`.
Если меньший символ стоит перед большим — он вычитается (например, `IV=4`, `IX=9`).

**Пример 1.** Вход: `s = "III"` → Выход: `3`.
**Пример 2.** Вход: `s = "LVIII"` → Выход: `58` (`L=50, V=5, III=3`).
**Пример 3.** Вход: `s = "MCMXCIV"` → Выход: `1994`.
**Ограничения:** `1 <= s.length <= 15`. `s` содержит только `IVXLCDM`. Гарантированно корректное римское число.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Roman to Integer](https://github.com/doocs/leetcode/blob/main/solution/0000-0099/0013.Roman%20to%20Integer/README.md)

Проходим строку слева направо. Если текущий символ меньше следующего — вычитаем его,
иначе складываем. Сложность — O(n) время, O(1) память (размер алфавита фиксирован).

```java
class Solution {
    public int romanToInt(String s) {
        String cs = "IVXLCDM";
        int[] vs = {1, 5, 10, 50, 100, 500, 1000};
        Map<Character, Integer> d = new HashMap<>();
        for (int i = 0; i < vs.length; ++i) {
            d.put(cs.charAt(i), vs[i]);
        }
        int n = s.length();
        int ans = d.get(s.charAt(n - 1));
        for (int i = 0; i < n - 1; ++i) {
            int sign = d.get(s.charAt(i)) < d.get(s.charAt(i + 1)) ? -1 : 1;
            ans += sign * d.get(s.charAt(i));
        }
        return ans;
    }
}
```

#### Ответ от Claude

**Идея.** Правило вычитания: если символ меньше следующего за ним — он вычитается,
иначе прибавляется. Можно обойти строку один раз, сравнивая каждый символ со следующим.
Для хранения соответствий используем массив `switch` — O(1) без HashMap.

```java
public int romanToInt(String s) {
    int total = 0;
    int prev = 0;
    for (int i = s.length() - 1; i >= 0; i--) {
        int curr = valueOf(s.charAt(i));
        // если текущий меньше предыдущего (справа) — вычитаем
        if (curr < prev) {
            total -= curr;
        } else {
            total += curr;
        }
        prev = curr;
    }
    return total;
}

private int valueOf(char c) {
    switch (c) {
        case 'I': return 1;
        case 'V': return 5;
        case 'X': return 10;
        case 'L': return 50;
        case 'C': return 100;
        case 'D': return 500;
        case 'M': return 1000;
        default:  return 0;
    }
}
```

**Разбор по шагам.** Идём справа налево. Поддерживаем `prev` — значение предыдущего
(правее стоящего) символа. Если `curr < prev` — это случай вычитания (например, `I`
перед `V`): вычитаем. Иначе прибавляем.

**Сложность.** Время O(n), память O(1). `switch` работает за O(1).

**Типичные ошибки / альтернативы.** Решение из источника идёт слева направо и сравнивает
`s[i]` с `s[i+1]` — логически эквивалентно, но требует обработки последнего символа
отдельно. Вариант справа налево проще: не нужно особого случая для конца строки.

### Вопрос 28. LC 387. Первый уникальный символ (First Unique Character in a String)

**Категория:** Алгоритмы и структуры данных · Строки / Hash Table · **Уровень:** Easy

Дана строка `s`. Найдите первый неповторяющийся символ и верните его индекс.
Если такого нет — верните `-1`.

**Пример 1.** Вход: `s = "leetcode"` → Выход: `0` (символ `'l'`).
**Пример 2.** Вход: `s = "loveleetcode"` → Выход: `2` (символ `'v'`).
**Пример 3.** Вход: `s = "aabb"` → Выход: `-1`.
**Ограничения:** `1 <= s.length <= 10^5`. `s` содержит только строчные латинские буквы.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — First Unique Character in a String](https://github.com/doocs/leetcode/blob/main/solution/0300-0399/0387.First%20Unique%20Character%20in%20a%20String/README.md)

Два прохода: первый считает частоты символов, второй ищет первый с частотой 1.
Сложность — O(n) время, O(1) память (массив из 26 элементов).

```java
class Solution {
    public int firstUniqChar(String s) {
        int[] cnt = new int[26];
        int n = s.length();
        for (int i = 0; i < n; ++i) {
            ++cnt[s.charAt(i) - 'a'];
        }
        for (int i = 0; i < n; ++i) {
            if (cnt[s.charAt(i) - 'a'] == 1) {
                return i;
            }
        }
        return -1;
    }
}
```

#### Ответ от Claude

**Идея.** Уникальный символ — тот, чья частота в строке равна 1. Два прохода:
подсчёт частот, затем поиск первого символа с частотой 1. Второй проход идёт
по строке (а не по массиву частот), чтобы гарантировать минимальный индекс.

```java
public int firstUniqChar(String s) {
    int[] freq = new int[26]; // счётчики для 'a'..'z'
    // первый проход: считаем частоты
    for (char c : s.toCharArray()) {
        freq[c - 'a']++;
    }
    // второй проход: ищем первый символ с freq == 1
    for (int i = 0; i < s.length(); i++) {
        if (freq[s.charAt(i) - 'a'] == 1) {
            return i;
        }
    }
    return -1;
}
```

**Разбор по шагам.** Массив `freq[26]` хранит число вхождений каждой буквы. После
подсчёта частот проходим строку снова: первый символ с `freq == 1` — ответ. Проход
по строке (не по алфавиту) гарантирует, что вернётся именно наименьший индекс.

**Сложность.** Время O(n) — два линейных прохода. Память O(1) — массив из 26 элементов.

**Типичные ошибки / альтернативы.** Ошибка: во втором проходе итерировать по `freq` вместо
строки — тогда нельзя вернуть корректный индекс. Альтернатива с `LinkedHashMap`:
один проход для подсчёта, затем перебор ключей в порядке вставки — O(n) время,
O(1) память (не более 26 ключей), но медленнее из-за накладных расходов `HashMap`.

### Вопрос 29. LC 191. Число единичных бит (Number of 1 Bits)

**Категория:** Алгоритмы и структуры данных · Bit Manipulation · **Уровень:** Easy

Дано беззнаковое 32-битное целое `n`. Верните число единичных битов (число Хэмминга).

**Пример 1.** Вход: `n = 11` (двоичное `00000000000000000000000000001011`) → Выход: `3`.
**Пример 2.** Вход: `n = 128` (двоичное `00000000000000000000000010000000`) → Выход: `1`.
**Пример 3.** Вход: `n = 2147483645` → Выход: `30`.
**Ограничения:** `n` — беззнаковое 32-битное целое.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Number of 1 Bits](https://github.com/doocs/leetcode/blob/main/solution/0100-0199/0191.Number%20of%201%20Bits/README.md)

Трюк `n &= n - 1`: обнуляет самый правый единичный бит. Повторяем, пока `n != 0`.
Число итераций равно числу единичных битов. Сложность — O(k) время (k — число единиц),
O(1) память.

```java
public class Solution {
    public int hammingWeight(int n) {
        int ans = 0;
        while (n != 0) {
            n &= n - 1;
            ++ans;
        }
        return ans;
    }
}
```

#### Ответ от Claude

**Идея.** Операция `n & (n - 1)` убирает самый правый установленный бит числа `n`.
Считаем, сколько раз это можно сделать до `n == 0` — это и есть число единичных битов.
Альтернатива: `n & 1` даёт младший бит, затем беззнаковый сдвиг `n >>> 1`.

```java
public int hammingWeight(int n) {
    int count = 0;
    while (n != 0) {
        count += (n & 1); // проверяем младший бит
        n >>>= 1;         // беззнаковый сдвиг вправо (важно: не >>)
    }
    return count;
}
```

**Разбор по шагам.** На каждой итерации `n & 1` даёт 1, если младший бит установлен, 0
иначе. Беззнаковый сдвиг `>>>` необходим: в Java `int` знаковый, и `>>` заполняет
старший бит знаком — при `n < 0` получим бесконечный цикл. `>>>` всегда заполняет 0.

**Сложность.** Время O(32) = O(1) — ровно 32 итерации для любого 32-битного числа.
Память O(1).

**Типичные ошибки / альтернативы.** Самая распространённая ошибка: использовать `>>` вместо
`>>>` — бесконечный цикл для отрицательных чисел. Вариант из источника (`n &= n-1`)
быстрее для чисел с малым числом единиц: число итераций = числу единичных битов. В Java
есть встроенный `Integer.bitCount(n)` — O(1) через алгоритм Кернигана.

### Вопрос 30. LC 268. Пропущенное число (Missing Number)

**Категория:** Алгоритмы и структуры данных · Bit Manipulation / Math · **Уровень:** Easy

Дан массив `nums` из `n` различных чисел в диапазоне `[0, n]`. Найдите единственное
число из этого диапазона, которого нет в массиве.

**Пример 1.** Вход: `nums = [3,0,1]` → Выход: `2`.
**Пример 2.** Вход: `nums = [0,1]` → Выход: `2`.
**Пример 3.** Вход: `nums = [9,6,4,2,3,5,7,0,1]` → Выход: `8`.
**Ограничения:** `n == nums.length`, `1 <= n <= 10^4`, `0 <= nums[i] <= n`. Все числа различны.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Missing Number](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0268.Missing%20Number/README.md)

Математический подход: ожидаемая сумма `[0..n]` = `n*(n+1)/2`. Вычитаем фактическую
сумму — получаем пропущенное число. Сложность — O(n) время, O(1) память.

```java
class Solution {
    public int missingNumber(int[] nums) {
        int n = nums.length;
        int ans = n;
        for (int i = 0; i < n; ++i) {
            ans += i - nums[i];
        }
        return ans;
    }
}
```

#### Ответ от Claude

**Идея.** Подход через XOR: XOR всех чисел от `0` до `n` с XOR всех элементов массива.
Все пары числов `[0..n]`, которые есть в `nums`, взаимно уничтожатся (a^a=0), останется
только пропущенное число.

```java
public int missingNumber(int[] nums) {
    int xor = nums.length; // начинаем с n (оно всегда "ожидается")
    for (int i = 0; i < nums.length; i++) {
        xor ^= i;        // XOR с индексом [0..n-1]
        xor ^= nums[i];  // XOR с фактическим значением
    }
    return xor;
}
```

**Разбор по шагам.** Инициализируем `xor = n` (так как нам нужны числа `0..n`, а
индексы только `0..n-1`). В цикле XOR-им с каждым индексом `i` и с `nums[i]`. Все
числа, которые есть в массиве, встречаются дважды (как индекс и как значение) и
обнуляются. Пропущенное число встречается только один раз.

**Сложность.** Время O(n), память O(1).

**Типичные ошибки / альтернативы.** Решение из источника (математика) проще на первый
взгляд. Возможная проблема: при очень большом `n` сумма `n*(n+1)/2` может переполнить
`int` — в Java `int` вмещает до ~2*10^9, при `n = 10^4` переполнения нет, но при
расширении задачи лучше использовать `long`. XOR-подход переполнения не имеет.

### Вопрос 31. LC 231. Степень двойки (Power of Two)

**Категория:** Алгоритмы и структуры данных · Bit Manipulation / Math · **Уровень:** Easy

Дано целое число `n`. Верните `true`, если `n` является степенью двойки.

**Пример 1.** Вход: `n = 1` → Выход: `true` (2^0 = 1).
**Пример 2.** Вход: `n = 16` → Выход: `true` (2^4 = 16).
**Пример 3.** Вход: `n = 3` → Выход: `false`.
**Ограничения:** `-2^31 <= n <= 2^31 - 1`.

#### Оригинальный ответ из интернета
> Источник: [doocs/leetcode — Power of Two](https://github.com/doocs/leetcode/blob/main/solution/0200-0299/0231.Power%20of%20Two/README.md)

Битовый трюк: степень двойки в двоичном представлении имеет ровно один установленный бит.
`n & (n - 1)` обнуляет младший установленный бит; если результат 0 и `n > 0` — это
степень двойки. Сложность — O(1) время, O(1) память.

```java
class Solution {
    public boolean isPowerOfTwo(int n) {
        return n > 0 && (n & (n - 1)) == 0;
    }
}
```

#### Ответ от Claude

**Идея.** Степень двойки имеет в двоичном представлении ровно один установленный бит:
`1 = 0001`, `2 = 0010`, `4 = 0100`, `8 = 1000`. Операция `n & -n` (lowbit) изолирует
самый младший установленный бит числа. Если у числа ровно один установленный бит,
то его lowbit равен самому числу. Отсюда условие: `n > 0 && (n & -n) == n`.

```java
public boolean isPowerOfTwo(int n) {
    // n > 0 отсекает 0 и отрицательные числа — они не являются степенями двойки
    // n & -n выделяет младший установленный бит; для степени двойки он совпадает с n
    return n > 0 && (n & -n) == n;
}
```

**Разбор по шагам.** Проверка `n > 0` отсекает 0 и отрицательные числа (в
дополнительном коде старший бит у отрицательных всегда установлен, поэтому
они заведомо не являются степенями двойки). Далее: `-n` в дополнительном коде —
это побитовое отрицание `n` плюс 1, что гарантирует, что `n & -n` оставляет ровно
один бит — самый правый единичный. Для `n = 8 = 1000b`: `-n = 1000b` (в int),
`n & -n = 1000b = 8 = n`. Для `n = 6 = 110b`: `-n = ...11111010b`,
`n & -n = 010b = 2 != 6`.

**Сложность.** Время O(1), память O(1) — одна битовая операция.

**Типичные ошибки / альтернативы.** Проверка `n > 0` обязательна: при `n = 0`
`0 & 0 == 0`, условие `0 == 0` истинно, что дало бы ложный `true`. Альтернатива
из оригинального ответа: `n & (n - 1) == 0` — сбрасывает младший бит; оба подхода
дают O(1). Также корректен вариант `Integer.bitCount(n) == 1` (считает единичные биты),
однако он менее наглядно выражает идею единственного бита в сравнении с lowbit.
