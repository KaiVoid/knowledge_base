# Урок 1. Обзор API даты и времени

**Трейл:** Date-Time APIs · **Оригинал:** [Date-Time Overview](https://docs.oracle.com/javase/tutorial/datetime/overview/index.html)
**Связанные области:** [[09-modern-java-features]] · **Вопросы:** modern-java

> Перевод официального руководства Oracle (The Java Tutorials, JDK 8). Объединяет страницы
> *Date-Time Overview*, *Date-Time Design Principles*, *The Date-Time Packages* и
> *Method Naming Conventions*.

Время кажется простой темой: даже недорогие часы способны достаточно точно показывать дату
и время. Однако при более внимательном рассмотрении становятся видны тонкие сложности и
множество факторов, влияющих на наше понимание времени. Например, результат прибавления
одного месяца к 31 января различается для високосного и невисокосного года. Часовые пояса
добавляют ещё больше сложности. Так, страна может вводить и отменять летнее время с коротким
уведомлением, делать это более одного раза в год или вовсе пропустить летнее время в каком-то
конкретном году.

API даты и времени (Date-Time API) использует в качестве календаря по умолчанию календарную
систему, определённую в стандарте [ISO-8601](http://www.iso.org/iso/home/standards/iso8601.htm).
Этот календарь основан на григорианской календарной системе и используется по всему миру как
фактический стандарт представления даты и времени. Основные классы Date-Time API носят имена
вроде `LocalDateTime`, `ZonedDateTime` и `OffsetDateTime`. Все они используют календарную систему
ISO. Если вы хотите применить альтернативную календарную систему, например Hijrah (исламский
календарь) или тайский буддийский календарь, пакет `java.time.chrono` позволяет воспользоваться
одной из предопределённых календарных систем. Либо вы можете создать собственную.

API даты и времени использует [Unicode Common Locale Data Repository (CLDR)](http://cldr.unicode.org) —
репозиторий общих данных локалей. Он поддерживает языки всего мира и содержит крупнейшую
доступную коллекцию данных о локалях. Информация в этом репозитории локализована на сотни языков.
Кроме того, API даты и времени использует [базу часовых поясов (Time-Zone Database, TZDB)](http://www.iana.org/time-zones).
Эта база предоставляет сведения о каждом изменении часового пояса в мире начиная с 1970 года,
а для основных часовых поясов хранит историю с момента появления самой концепции часовых поясов.

## Принципы проектирования API даты и времени

API даты и времени был разработан с опорой на несколько принципов проектирования.

### Ясность (Clear)

Методы API чётко определены, а их поведение понятно и предсказуемо. Например, вызов метода
Date-Time с параметром, равным `null`, обычно приводит к исключению `NullPointerException`.

### Текучесть (Fluent)

API даты и времени предоставляет текучий интерфейс (fluent interface), благодаря чему код легко
читается. Поскольку большинство методов не принимают параметры со значением `null` и не возвращают
`null`, вызовы методов можно объединять в цепочки, а получающийся код быстро понятен. Например:

```java
LocalDate today = LocalDate.now();
LocalDate payday = today.with(TemporalAdjusters.lastDayOfMonth()).minusDays(2);
```

### Неизменяемость (Immutable)

Большинство классов API даты и времени создают неизменяемые (immutable) объекты: после создания
объект уже нельзя изменить. Чтобы поменять значение неизменяемого объекта, необходимо построить
новый объект как изменённую копию исходного. Это также означает, что API даты и времени по
определению потокобезопасен (thread-safe). На API это влияет так: большинство методов, создающих
объекты даты или времени, имеют префиксы `of`, `from` или `with` вместо использования конструкторов,
и методов `set` нет вовсе. Например:

```java
LocalDate dateOfBirth = LocalDate.of(2012, Month.MAY, 14);
LocalDate firstBirthday = dateOfBirth.plusYears(1);
```

### Расширяемость (Extensible)

API даты и времени расширяем везде, где это возможно. Например, вы можете определять собственные
корректировщики времени (time adjusters) и запросы (queries) либо построить собственную
календарную систему.

## Пакеты API даты и времени

API даты и времени состоит из основного пакета `java.time` и четырёх подпакетов.

**`java.time`**

Ядро API для представления даты и времени. Включает классы для даты, времени, объединённых даты
и времени, часовых поясов, моментов времени (instants), длительностей (duration) и часов (clocks).
Эти классы основаны на календарной системе, определённой в ISO-8601, они неизменяемы и
потокобезопасны.

**`java.time.chrono`**

API для представления календарных систем, отличных от используемой по умолчанию ISO-8601. Также
позволяет определить собственную календарную систему. Это руководство не рассматривает данный
пакет сколько-нибудь подробно.

**`java.time.format`**

Классы для форматирования и разбора (parsing) дат и времени.

**`java.time.temporal`**

Расширенный API, предназначенный главным образом для авторов фреймворков и библиотек. Обеспечивает
взаимодействие между классами даты и времени, выполнение запросов (querying) и корректировку
(adjustment). В этом пакете определяются поля (`TemporalField` и `ChronoField`) и единицы измерения
(`TemporalUnit` и `ChronoUnit`).

**`java.time.zone`**

Классы для поддержки часовых поясов, смещений относительно часовых поясов и правил часовых поясов.
При работе с часовыми поясами большинству разработчиков потребуется использовать только
`ZonedDateTime`, а также `ZoneId` или `ZoneOffset`.

## Соглашения об именовании методов

API даты и времени предлагает богатый набор методов в богатом наборе классов. Имена методов
сделаны согласованными между классами везде, где это возможно. Например, многие классы предлагают
метод `now`, который фиксирует относящиеся к данному классу значения даты или времени текущего
момента. Существуют методы `from`, позволяющие выполнить преобразование из одного класса в другой.

Также есть стандартизация в отношении префиксов имён методов. Поскольку большинство классов API
даты и времени неизменяемы, API не содержит методов `set`. (После создания значение неизменяемого
объекта изменить нельзя. Неизменяемым эквивалентом метода `set` является метод `with`.)
В таблице ниже перечислены часто используемые префиксы.

| Префикс | Тип метода | Назначение |
|---------|-----------|------------|
| `of` | статическая фабрика | Создаёт экземпляр, при этом фабрика прежде всего проверяет (валидирует) входные параметры, а не преобразует их. |
| `from` | статическая фабрика | Преобразует входные параметры в экземпляр целевого класса; при этом часть информации из исходных данных может быть потеряна. |
| `parse` | статическая фабрика | Разбирает входную строку и создаёт экземпляр целевого класса. |
| `format` | экземплярный | Использует указанный форматтер, чтобы отформатировать значения временнóго объекта и получить строку. |
| `get` | экземплярный | Возвращает часть состояния целевого объекта. |
| `is` | экземплярный | Запрашивает состояние целевого объекта. |
| `with` | экземплярный | Возвращает копию целевого объекта с одним изменённым элементом; это неизменяемый эквивалент метода `set` у JavaBean. |
| `plus` | экземплярный | Возвращает копию целевого объекта с прибавленным количеством времени. |
| `minus` | экземплярный | Возвращает копию целевого объекта с вычтенным количеством времени. |
| `to` | экземплярный | Преобразует данный объект в другой тип. |
| `at` | экземплярный | Объединяет данный объект с другим. |
