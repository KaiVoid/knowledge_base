# Урок 1. Быстрый обзор управления приложениями

**Связанные области:** [[23-security]] · **Вопросы:** security

> Перевод официального руководства Oracle (The Java Tutorials, JDK 8). Урок показывает, как
> с помощью менеджера безопасности (security manager) разрешать или запрещать приложениям
> доступ к системным ресурсам. Кроме того, урок демонстрирует, что доступ к ресурсам — например,
> чтение или запись файла — для приложений, запущенных с менеджером безопасности, не разрешён,
> пока он явно не предоставлен записью о разрешении (permission) в файле политики (policy file).

> **Примечание.** Руководство написано для JDK 8. Примеры и приёмы, описанные на этой странице,
> не используют улучшений, появившихся в более поздних выпусках, и могут опираться на технологии,
> которые больше недоступны (например, апплеты и `policytool` удалены в новых версиях JDK).

## Наблюдаем свободу приложения

Менеджер безопасности (security manager) **не** устанавливается автоматически при запуске
*приложения*. На следующем шаге вы увидите, как применить к приложению, найденному в локальной
файловой системе, ту же политику безопасности, что и к загруженным из сети «песочничным»
(sandbox) апплетам. Но сначала покажем, что для приложения менеджер безопасности по умолчанию
не установлен, а значит, приложение имеет полный доступ к ресурсам.

Создайте на своём компьютере файл с именем `GetProps.java`, скопировав или скачав исходный код
`GetProps.java`.

В примерах этого урока предполагается, что вы поместили `GetProps.java` в каталог `C:\Test`,
если работаете в Windows, или в каталог `~/test` в UNIX.

Как видно из исходного файла, эта программа пытается получить (прочитать) значения свойств
(properties) с именами `"os.name"`, `"java.version"`, `"user.home"` и `"java.home"`.

Теперь скомпилируйте и запустите `GetProps.java`. Вы должны увидеть примерно такой вывод:

```text
C:\TEST>java GetProps
    About to get os.name property value
      The name of your operating system is:
      Windows XP
    About to get java.version property value
      The version of the JVM you are running is:
      1.6.0
    About to get user.home property value
      Your user home directory is: C:\WINDOWS
    About to get java.home property value
      Your JRE installation directory is:
      C:\JDK7.0.0\JRE
```

Это показывает, что приложению был разрешён доступ ко всем значениям свойств.

## Как ограничить приложение

Как вы видели на предыдущем шаге, среда выполнения Java **не** устанавливает менеджер
безопасности автоматически при запуске *приложения*. Чтобы применить к приложению из локальной
файловой системы ту же политику безопасности, что и к загруженным из сети «песочничным» апплетам,
можно вызвать интерпретатор с аргументом командной строки `-Djava.security.manager`.

Чтобы выполнить приложение `GetProps` с менеджером безопасности по умолчанию, введите следующее:

```text
java -Djava.security.manager GetProps
```

Вот вывод программы:

```text
C:\TEST>java -Djava.security.manager GetProps
    About to get os.name property value
      The name of your operating system is: SunOS
    About to get java.version property value
      The version of the JVM you are running is: 1.7.0
    About to get user.home property value
    Caught exception java.security.AccessControlException:
        access denied ("java.util.PropertyPermission"
        "user.home" "read")
```

### Свойства, чувствительные к безопасности

Среда выполнения Java по умолчанию загружает стандартный файл политики и предоставляет всему
коду разрешение на доступ к некоторым общеполезным свойствам, таким как `"os.name"` и
`"java.version"`. Эти свойства не являются чувствительными к безопасности, поэтому предоставление
таких разрешений обычно не создаёт угрозы безопасности.

Остальные свойства, к которым пытается обратиться `GetProps` — `"user.home"` и `"java.home"` —
**не** входят в число свойств, для которых системный файл политики предоставляет разрешение на
чтение. Поэтому, как только `GetProps` пытается обратиться к первому из этих свойств
(`"user.home"`), менеджер безопасности предотвращает доступ и сообщает об исключении
`AccessControlException`. Это исключение указывает, что действующая в данный момент политика,
которая состоит из записей в одном или нескольких файлах политики, не разрешает чтение свойства
`"user.home"`.

> **Примечание.** Код всегда может прочитать файл из того же каталога, в котором находится сам
> (или из подкаталога этого каталога); явного разрешения для этого не требуется. Код также может
> получить путь к каталогу, из которого он запущен, и этот путь может содержать чувствительную
> информацию. Например, если код выполняется из домашнего каталога (или его подкаталога), путь
> может раскрыть имя текущего пользователя.

### Стандартный файл политики

Стандартный файл политики `java.policy` по умолчанию расположен по адресу:

- **Windows**: `<java.home>\lib\security\java.policy`
- **UNIX**: `<java.home>/lib/security/java.policy`

Здесь *java.home* обозначает значение свойства `"java.home"` — системного свойства, задающего
каталог, в который установлена среда JRE. Так, если JRE установлена в каталог `C:\jdk\jre`
в Windows и `/jdk/jre` в UNIX, системный файл политики находится по адресу:

- **Windows**: `C:\jdk\jre\lib\security\java.policy`
- **UNIX**: `/jdk/jre/lib/security/java.policy`

## Настройка файла политики для предоставления нужных разрешений

На этом шаге используется утилита Policy Tool, чтобы открыть файл политики с именем
`examplepolicy`, созданный в уроке *Создание файла политики* (Creating a Policy File). Вы добавите
новую запись политики, предоставляющую коду из каталога, где хранится `GetProps.class`, разрешение
на чтение значений свойств `"user.home"` и `"java.home"`.

Шаги таковы:

1. [Открыть файл политики](#открыть-файл-политики)
2. [Предоставить нужные разрешения](#предоставить-нужные-разрешения)
3. [Сохранить файл политики](#сохранить-файл-политики)

> **Примечание для пользователей UNIX.** Инструкции иллюстрируют создание файла политики для
> системы Windows. Шаги абсолютно те же и в UNIX, со следующими отличиями:
>
> - Файл `examplepolicy` вы берёте из каталога `test` в своём домашнем каталоге.
> - Для URL в поле **CodeBase** на шаге предоставления разрешений вместо `file:/C:/Test/` можно
>   подставить `file:${user.home}/test/`. Либо вы можете напрямую указать свой домашний каталог,
>   а не ссылаться на свойство `"user.home"`, например `file:/home/jones/test/`.

### Открыть файл политики

Запустите Policy Tool, введя в командной строке следующее:

```text
policytool
```

Откроется окно Policy Tool. Чтобы открыть файл политики `examplepolicy`, используйте команду
**Open** в меню **File**. Появится диалог открытия, с помощью которого вы перемещаетесь по
структуре каталогов, пока не доберётесь до каталога, содержащего файл политики (то есть
до каталога `C:\Test\`).

Выберите в этом каталоге файл `examplepolicy` и нажмите кнопку **Open**.

В результате окно Policy Tool заполнится информацией из файла политики `examplepolicy`, включая
имя файла политики и часть **CodeBase** записи политики, созданной в уроке *Создание файла политики*.

### Предоставить нужные разрешения

Чтобы предоставить приложению `GetProps` разрешение на чтение значений свойств `"user.home"`
и `"java.home"`, нужно создать запись политики, предоставляющую эти разрешения. Нажмите кнопку
**Add Policy Entry** в главном окне Policy Tool. Откроется диалоговое окно Policy Entry.

Введите в текстовое поле **CodeBase** следующий файловый URL, чтобы указать, что вы собираетесь
предоставить разрешение коду из заданного каталога — каталога, в котором хранится `GetProps.class`:

```text
file:/C:/Test/
```

(Учтите, что это URL и потому всегда должен содержать прямые слеши, а не обратные.)

Поле **SignedBy** оставьте пустым, так как вы не требуете, чтобы код был подписан.

Чтобы добавить разрешение на чтение значения свойства `"user.home"`, нажмите кнопку
**Add Permission**. Откроется диалоговое окно Permissions. Выполните следующее:

1. Выберите **Property Permission** из выпадающего списка Permission. Полное имя типа разрешения
   (`java.util.PropertyPermission`) теперь появится в текстовом поле справа от выпадающего списка.
2. Введите в текстовое поле справа от списка с меткой Target Name следующее, чтобы указать
   свойство `"user.home"`:

   ```text
   user.home
   ```

3. Задайте разрешение на чтение этого свойства, выбрав вариант **read** из выпадающего списка Actions.

Нажмите кнопку **OK**. Новое разрешение появится строкой в окне записи политики.

Чтобы добавить разрешение на чтение значения свойства `"java.home"`, снова нажмите кнопку
**Add Permission**. В диалоговом окне Permissions выполните следующее:

1. Выберите **Property Permission** из выпадающего списка Permission.
2. Введите в текстовое поле справа от списка Target Name следующее, чтобы указать свойство
   `"java.home"`:

   ```text
   java.home
   ```

3. Задайте разрешение на чтение этого свойства, выбрав вариант **read** из выпадающего списка Actions.

Нажмите кнопку **OK**. Новое разрешение и добавленное ранее разрешение появятся строками в окне
записи политики.

На этом задание записи политики завершено, поэтому нажмите кнопку **Done** в диалоге Policy Entry.
Окно Policy Tool теперь содержит строку, представляющую новую запись политики, с указанным
значением **CodeBase**.

### Сохранить файл политики

Чтобы сохранить файл политики, просто выберите команду **Save** в меню **File**.

Затем закройте Policy Tool, выбрав команду **Exit** из меню **File**.

## Эффект от файла политики

Теперь, когда вы добавили нужную запись политики в файл политики `examplepolicy`, вы должны
суметь прочитать указанные свойства при запуске приложения `GetProps` с менеджером безопасности.

Всякий раз, когда вы запускаете апплет или приложение с менеджером безопасности, по умолчанию
загружаются и используются те файлы политики, которые указаны в **«файле свойств безопасности»**
(security properties file). Он расположен в одном из следующих каталогов:

- **Windows**: `<java.home>\lib\security\java.security`
- **UNIX**: `<java.home>/lib/security/java.security`

> **Примечание.** Переменная `java.home` указывает на каталог, в который установлена среда JRE.

Расположения файлов политики задаются как значения свойств, имена которых имеют вид:

```text
policy.url.n
```

где переменная `n` обозначает число. Каждое значение свойства указывается в строке вида:

```text
policy.url.n=URL
```

где *URL* — спецификация URL. Например, стандартные файлы политики, иногда называемые системным
и пользовательским файлами политики соответственно, определены в файле свойств безопасности так:

```text
policy.url.1=file:${java.home}/lib/security/java.policy
policy.url.2=file:${user.home}/.java.policy
```

> **Примечание.** Запись вида `${propName}` в файле свойств безопасности — это способ указать
> значение свойства. Так, `${java.home}` будет заменено во время выполнения фактическим значением
> свойства `"java.home"` (каталог, в который установлена JRE), а `${user.home}` будет заменено
> значением свойства `"user.home"`, например `C:\Windows`.

Есть два возможных способа, чтобы файл `examplepolicy` учитывался как часть общей политики
в дополнение к файлам политики, указанным в файле свойств безопасности. Можно либо указать
дополнительный файл политики в свойстве, переданном среде выполнения (см. [Подход 1](#подход-1)),
либо добавить строку в файл свойств безопасности, задающую дополнительный файл политики
(см. [Подход 2](#подход-2)).

### Подход 1

Можно использовать аргумент командной строки интерпретатора `-Djava.security.policy`, чтобы
указать файл политики, который следует использовать в дополнение к указанным в файле свойств
безопасности.

Убедитесь, что вы находитесь в каталоге, содержащем `GetProps.class` и `examplepolicy`. Затем
вы можете запустить приложение `GetProps` и передать интерпретатору файл политики `examplepolicy`,
набрав следующую команду в одну строку:

```text
java -Djava.security.manager -Djava.security.policy=examplepolicy GetProps
```

> **Примечание.** Помните, что аргумент `-Djava.security.manager` обязателен для запуска
> приложения с менеджером безопасности, как показано на шаге *Как ограничить приложение*.

Программа сообщит значения свойств `"user.home"` и `"java.home"`.

Если приложение по-прежнему сообщает об ошибке, что-то не так в файле политики. Используйте Policy
Tool, чтобы проверить запись политики, которую вы только что создали на шаге *Настройка файла
политики для предоставления нужных разрешений*.

### Подход 2

В файле свойств безопасности можно указать несколько URL в свойствах `policy.url.n`, и все
обозначенные файлы политики будут загружены. Поэтому один из способов заставить интерпретатор
`java` учитывать записи политики из вашего файла `examplepolicy` — добавить запись, указывающую
этот файл политики, в файл свойств безопасности.

> **Важно.** Если вы используете собственную копию JDK, вы легко можете отредактировать свой файл
> свойств безопасности. Если же вы работаете с версией, разделяемой с другими пользователями,
> вы сможете изменить общесистемный файл свойств безопасности только при наличии прав на запись
> в него или попросив системного администратора изменить файл при необходимости. Однако вносить
> изменения в общесистемный файл политики ради этого тестового урока, вероятно, неуместно. Мы
> советуем просто прочитать изложенное ниже, чтобы увидеть, как это делается, либо установить
> собственную приватную версию JDK для прохождения уроков руководства.

Чтобы изменить файл свойств безопасности, откройте его в редакторе, подходящем для правки
текстового файла ASCII. Затем добавьте следующую строку после строки, содержащей `policy.url.2`.
Если вы в системе Windows, добавьте:

```text
policy.url.3=file:/C:/Test/examplepolicy
```

Если вы в системе UNIX, добавьте:

```text
policy.url.3=file:${user.home}/test/examplepolicy
```

В системе UNIX можно как альтернатива явно указать свой домашний каталог:

```text
policy.url.3=file:/home/jones/test/examplepolicy
```

### Запуск приложения

Теперь вы должны успешно запустить следующее:

```text
java -Djava.security.manager GetProps
```

Как и в подходе 1, если вы по-прежнему получаете исключение безопасности, что-то не так в файле
политики. Используйте Policy Tool, чтобы проверить запись политики, созданную на шаге *Настройка
файла политики для предоставления нужных разрешений*, и исправьте опечатки и другие ошибки.

> **Важно.** Включать файл `examplepolicy` нужно, только если вы проходите этот урок руководства.
> Чтобы исключить этот файл, откройте файл свойств безопасности и удалите строку, которую вы
> только что добавили.
>
> Прежде чем продолжить, возможно, вы захотите удалить добавленную строку из файла свойств
> безопасности (или закомментировать её), поскольку, скорее всего, вы не хотите, чтобы файл
> `examplepolicy` подключался, когда вы не проходите уроки руководства.
