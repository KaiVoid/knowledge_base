## Рисование геометрических примитивов

Java 2D API предоставляет несколько классов, которые описывают распространённые геометрические
объекты — точки (*points*), линии (*lines*), кривые (*curves*) и прямоугольники (*rectangles*).
Эти геометрические классы входят в пакет `java.awt.geom`.

Все примеры этого раздела создают геометрию средствами пакета `java.awt.geom`, а затем
отображают её с помощью класса `Graphics2D`. Чтобы начать, нужно получить объект `Graphics2D` —
например, приведением (*cast*) параметра `Graphics` метода `paint()`:

```java
public void paint (Graphics g) {
    Graphics2D g2 = (Graphics2D) g;
    ...
}
```

### Точка (Point)

Класс `Point` создаёт точку (*point*), представляющую местоположение в координатном
пространстве (x, y). Подклассы `Point2D.Float` и `Point2D.Double` обеспечивают соответственно
одинарную (*float*) и двойную (*double*) точность хранения координат точки.

Задать положение точки можно методом `setLocation`.

### Линия (Line)

Класс `Line2D` представляет отрезок прямой (*line segment*) в координатном пространстве (x, y).
Подклассы `Line2D.Float` и `Line2D.Double` задают линии в одинарной и двойной точности.

Концы линии можно указать через конструктор класса `Line2D.Float`:

- `Line2D.Float(float X1, float Y1, float X2, float Y2)`
- `Line2D.Float(Point2D p1, Point2D p2)`

### Прямоугольник (Rectangle)

Класс `Rectangle2D` представляет прямоугольник, заданный местоположением (x, y) и размером
(ширина × высота, *w × h*). Подклассы `Rectangle2D.Float` и `Rectangle2D.Double` задают
прямоугольник в одинарной и двойной точности.

### Прямоугольник со скруглёнными углами (RoundRectangle)

Класс `RoundRectangle2D` представляет прямоугольник со скруглёнными углами (*rounded corners*),
заданный местоположением (x, y), размером (ширина × высота), а также шириной и высотой дуги угла
(*corner arc*). Подклассы `RoundRectangle2D.Float` и `RoundRectangle2D.Double` задают такой
прямоугольник в одинарной и двойной точности.

Скруглённый прямоугольник задаётся следующими параметрами:

- местоположение (*location*);
- ширина (*width*);
- высота (*height*);
- ширина дуги угла (*width of the corner arc*);
- высота дуги угла (*height of the corner arc*).

### Эллипс (Ellipse)

Класс `Ellipse2D` представляет эллипс (*ellipse*), заданный ограничивающим прямоугольником
(*bounding rectangle*). Подклассы `Ellipse2D.Float` и `Ellipse2D.Double` задают эллипс в
одинарной и двойной точности. Эллипс полностью определяется местоположением, шириной и высотой.

### Дуга (Arc)

Чтобы нарисовать часть эллипса, используют класс `Arc2D`. Этот класс представляет дугу (*arc*),
заданную ограничивающим прямоугольником, начальным углом (*start angle*), угловой протяжённостью
(*angular extent*) и типом замыкания (*closure type*). Подклассы `Arc2D.Float` и `Arc2D.Double`
задают дугу в одинарной и двойной точности.

Класс `Arc2D` определяет три типа дуг, представленных соответствующими константами этого класса:

- `OPEN` — открытая дуга;
- `PIE` — дуга-«сектор» (как кусок пирога);
- `CHORD` — дуга, замкнутая хордой (*chord*).

Кроме того, методом `setArcByCenter` можно задать дугу от центральной точки, указав её
координаты и радиус.

### Квадратичная кривая (Quadratic curve)

Класс `QuadCurve2D` реализует интерфейс `Shape`. Этот класс представляет сегмент квадратичной
параметрической кривой (*quadratic parametric curve*) в координатном пространстве (x, y).
Подклассы `QuadCurve2D.Float` и `QuadCurve2D.Double` задают квадратичную кривую в одинарной и
двойной точности.

Для задания двух концевых точек (*endpoints*) и одной управляющей точки (*control point*) кривой
служат несколько методов `setCurve`.

### Кубическая кривая (Cubic curve)

Класс `CubicCurve2D` также реализует интерфейс `Shape`. Этот класс представляет сегмент
кубической параметрической кривой (*cubic parametric curve*) в координатном пространстве (x, y).
Подклассы `CubicCurve2D.Float` и `CubicCurve2D.Double` задают кубическую кривую в одинарной и
двойной точности.

Класс `CubicCurve2D` имеет методы задания кривой, аналогичные классу `QuadCurve2D`, но с
добавлением второй управляющей точки.

## Рисование произвольных фигур

Вы уже научились рисовать большинство фигур, представленных в пакете `java.awt.geom`. Для
создания более сложной геометрии — например, многоугольников (*polygons*), ломаных (*polylines*)
или звёзд — используется ещё один класс этого пакета: `GeneralPath`.

Этот класс реализует интерфейс `Shape` и представляет геометрический путь (*path*), построенный
из линий, а также квадратичных и кубических кривых. Три конструктора этого класса позволяют
создать объект `GeneralPath` с правилом заполнения по умолчанию (`WIND_NON_ZERO`), с заданным
правилом заполнения (`WIND_NON_ZERO` или `WIND_EVEN_ODD`) либо с указанной начальной ёмкостью
по координатам. Правило заполнения (*winding rule*) определяет, как вычисляется внутренняя
область пути.

```java
public void paint (Graphics g) {
    Graphics2D g2 = (Graphics2D) g;
    ...
}
```

Чтобы создать пустой экземпляр `GeneralPath`, вызовите `new GeneralPath()`, а затем добавляйте
к фигуре сегменты с помощью следующих методов:

- `moveTo(float x, float y)` — перемещает текущую точку пути в указанную точку;
- `lineTo(float x, float y)` — добавляет к текущему пути сегмент линии;
- `quadTo(float ctrlx, float ctrly, float x2, float y2)` — добавляет к текущему пути сегмент
  квадратичной кривой;
- `curveTo(float ctrlx1, float ctrly1, float ctrlx2, float ctrly2, float x3, float y3)` —
  добавляет к текущему пути сегмент кубической кривой;
- `closePath()` — замыкает текущий путь.

### Пример ломаной (polyline)

```java
// рисуем GeneralPath (ломаную)
int x2Points[] = {0, 100, 0, 100};
int y2Points[] = {0, 50, 50, 0};
GeneralPath polyline =
        new GeneralPath(GeneralPath.WIND_EVEN_ODD, x2Points.length);

polyline.moveTo (x2Points[0], y2Points[0]);

for (int index = 1; index < x2Points.length; index++) {
         polyline.lineTo(x2Points[index], y2Points[index]);
};

g2.draw(polyline);
```

### Пример многоугольника (polygon)

```java
// рисуем GeneralPath (многоугольник)
int x1Points[] = {0, 100, 0, 100};
int y1Points[] = {0, 50, 50, 0};
GeneralPath polygon =
        new GeneralPath(GeneralPath.WIND_EVEN_ODD,
                        x1Points.length);
polygon.moveTo(x1Points[0], y1Points[0]);

for (int index = 1; index < x1Points.length; index++) {
        polygon.lineTo(x1Points[index], y1Points[index]);
};

polygon.closePath();
g2.draw(polygon);
```

Обратите внимание, что единственное различие между двумя последними примерами кода — метод
`closePath()`. Этот метод превращает ломаную в многоугольник, дорисовывая прямую линию обратно
к координатам последнего `moveTo`.

Чтобы добавить конкретный путь в конец вашего объекта `GeneralPath`, используйте один из методов
`append()`. Пример кода `ShapesDemo2D.java` содержит дополнительные реализации произвольных фигур.

## Обводка и заливка графических примитивов

Вы уже умеете создавать различные геометрические примитивы и более сложные фигуры. Этот урок
учит добавлять к графике цвет и красивые контуры и описывает заливку (*filling*) и обводку
(*stroking*):

- **Заливка** (*filling*) — процесс закрашивания внутренней области фигуры сплошным цветом,
  цветовым градиентом или текстурным узором (*texture pattern*).
- **Обводка** (*stroking*) — процесс рисования контура фигуры с применением атрибутов ширины
  обводки (*stroke width*), стиля линии (*line style*) и цвета.

Чтобы применить замысловатые стили линий и узоры заливки к геометрическим примитивам, измените
атрибуты обводки (*stroke*) и заливки (*paint*) в контексте `Graphics2D` перед отрисовкой.
Например, чтобы нарисовать пунктирную линию, создайте подходящий объект `Stroke`. Чтобы добавить
эту обводку в контекст `Graphics2D` перед отрисовкой линии, вызовите метод `setStroke`. Точно
так же градиентную заливку к объекту `Shape` применяют, создав объект `GradientPaint` и добавив
его в контекст `Graphics2D`.

Следующие строки кода обогащают геометрические примитивы заливкой и обводкой:

```java
// рисуем RoundRectangle2D.Double

final static float dash1[] = {10.0f};
    final static BasicStroke dashed =
        new BasicStroke(1.0f,
                        BasicStroke.CAP_BUTT,
                        BasicStroke.JOIN_MITER,
                        10.0f, dash1, 0.0f);
g2.setStroke(dashed);
g2.draw(new RoundRectangle2D.Double(x, y,
                                   rectWidth,
                                   rectHeight,
                                   10, 10));
```

```java
// заливаем Ellipse2D.Double
redtowhite = new GradientPaint(0,0,color.RED,100, 0,color.WHITE);
g2.setPaint(redtowhite);
g2.fill (new Ellipse2D.Double(0, 0, 100, 50));
```

### Задание замысловатых стилей линий и узоров заливки

С помощью классов Java 2D `Stroke` и `Paint` можно определять замысловатые стили линий и узоры
заливки.

#### Стили линий

Стили линий задаются атрибутом обводки (*stroke*) в контексте отрисовки `Graphics2D`. Чтобы
установить этот атрибут, создайте объект `BasicStroke` и передайте его в метод `setStroke`
класса `Graphics2D`.

Объект `BasicStroke` хранит сведения о ширине линии (*line width*), стиле соединения
(*join style*), стиле концов (*end-cap style*) и стиле пунктира (*dash style*). Эта информация
используется, когда фигура `Shape` отрисовывается методом `draw`.

**Ширина линии** (*line width*) — это толщина линии, измеренная перпендикулярно её траектории.
Ширина задаётся значением типа `float` в единицах пользовательских координат (*user coordinate
units*), которые при преобразовании по умолчанию примерно равны 1/72 дюйма.

**Стиль соединения** (*join style*) — это оформление, применяемое в месте, где встречаются два
сегмента линии. `BasicStroke` поддерживает три стиля соединения:

- `JOIN_BEVEL`
- `JOIN_MITER`
- `JOIN_ROUND`

**Стиль концов** (*end-cap style*) — это оформление, применяемое там, где сегмент линии
заканчивается. `BasicStroke` поддерживает три стиля концов:

- `CAP_BUTT`
- `CAP_ROUND`
- `CAP_SQUARE`

**Стиль пунктира** (*dash style*) определяет узор из непрозрачных и прозрачных участков,
применяемый вдоль длины линии. Стиль пунктира задаётся массивом пунктира (*dash array*) и фазой
пунктира (*dash phase*). **Массив пунктира** определяет рисунок пунктира: чередующиеся элементы
массива представляют длину штриха и длину промежутка между штрихами в единицах пользовательских
координат. Элемент 0 представляет первый штрих, элемент 1 — первый промежуток и так далее.
**Фаза пунктира** — это смещение внутри узора пунктира, также заданное в единицах
пользовательских координат. Фаза пунктира указывает, какая часть узора применяется к началу линии.

#### Узоры заливки

Узоры заливки задаются атрибутом заливки (*paint*) в контексте отрисовки `Graphics2D`. Чтобы
установить этот атрибут, создайте экземпляр объекта, реализующего интерфейс `Paint`, и передайте
его в метод `setPaint` класса `Graphics2D`.

Интерфейс `Paint` реализуют следующие три класса: `Color`, `GradientPaint` и `TexturePaint`.

Чтобы создать `GradientPaint`, нужно указать начальную позицию и цвет, а также конечную позицию
и цвет. Градиент изменяется пропорционально от одного цвета к другому вдоль линии, соединяющей
эти две позиции.

Узор для класса `TexturePaint` определяется классом `BufferedImage`. Чтобы создать объект
`TexturePaint`, нужно указать изображение с узором и прямоугольник, который используется для
размножения (*replicate*) и привязки (*anchor*) узора.
