# Core Java: синтаксис и ООП

> **Уровень:** Junior / Middle
> **Связанные вопросы:** [Вопросы по Core Java →](../../../interview-questions/core-java/README.md)
> **Связанные области:** [[05-generics]], [[06-exceptions]], [[09-modern-java-features]]

## Что это и зачем

Фундамент языка: типы данных (примитивы и ссылочные), переменные, операторы, управляющие
конструкции, массивы, строки. Объектно-ориентированное программирование в Java — классы и объекты,
четыре столпа (инкапсуляция, наследование, полиморфизм, абстракция), интерфейсы и абстрактные классы,
модификаторы доступа, ключевые слова `static`/`final`. Без уверенного владения этим материалом
невозможно понимать ни код приложений, ни более сложные темы (коллекции, многопоточность, фреймворки).

Java — статически типизированный язык: каждая переменная должна быть объявлена до использования,
а тип определяется на этапе компиляции. Все классы в Java неявно наследуют `java.lang.Object`,
что обеспечивает единый корень иерархии типов и общий набор методов.

## Ключевые подтемы

### Примитивные типы данных

Java определяет восемь примитивных типов. Они не являются объектами и хранятся непосредственно
в стеке (для локальных переменных) или в области объекта в куче (для полей).
Источник: [Primitive Data Types — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/nutsandbolts/datatypes.html).

| Тип       | Размер   | Диапазон / значения              | Значение по умолчанию |
|-----------|----------|----------------------------------|-----------------------|
| `byte`    | 8 бит    | -128 … 127                       | `0`                   |
| `short`   | 16 бит   | -32 768 … 32 767                 | `0`                   |
| `int`     | 32 бита  | -2³¹ … 2³¹-1                     | `0`                   |
| `long`    | 64 бита  | -2⁶³ … 2⁶³-1                     | `0L`                  |
| `float`   | 32 бита  | IEEE 754 одинарная точность      | `0.0f`                |
| `double`  | 64 бита  | IEEE 754 двойная точность        | `0.0d`                |
| `char`    | 16 бит   | '' (0) … '￿' (65 535) | `''`            |
| `boolean` | ~1 бит   | `true` / `false`                 | `false`               |

**Важные нюансы:**
- Значения по умолчанию применяются только к **полям** класса. Локальные переменные
  не инициализируются автоматически — компилятор выдаст ошибку при попытке прочитать
  неинициализированную локальную переменную.
- `int` и `long` начиная с Java 8 можно интерпретировать как беззнаковые через методы
  `Integer.toUnsignedString()`, `Long.compareUnsigned()` и т.д.
- Для денежных вычислений использовать `float`/`double` **нельзя** — они не дают точных результатов.
  Следует использовать `java.math.BigDecimal`.

**Литералы и запись числовых значений (Java 7+):**

```java
int decimal  = 26;
int hex      = 0x1a;          // шестнадцатеричный
int binary   = 0b11010;       // двоичный
long card    = 1234_5678_9012_3456L;  // подчёркивания для читаемости
double pi    = 3.14_15;
float radius = 1.234e2f;      // научная нотация
```

Подчёркивания разрешены **только между цифрами** (не у начала/конца, не рядом с `.`, `L`, `F`).

### Автоупаковка (autoboxing) и классы-обёртки

Для каждого примитивного типа существует соответствующий ссылочный тип — класс-обёртка.
Источник: [Autoboxing and Unboxing — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/data/autoboxing.html).

| Примитив  | Класс-обёртка |
|-----------|---------------|
| `boolean` | `Boolean`     |
| `byte`    | `Byte`        |
| `char`    | `Character`   |
| `short`   | `Short`       |
| `int`     | `Integer`     |
| `long`    | `Long`        |
| `float`   | `Float`       |
| `double`  | `Double`      |

**Autoboxing** — автоматическое преобразование примитива в обёртку, которое компилятор
выполняет при присваивании или передаче в метод, ожидающий ссылочный тип:

```java
List<Integer> list = new ArrayList<>();
for (int i = 0; i < 5; i++) {
    list.add(i);  // компилятор заменяет на Integer.valueOf(i)
}
```

**Unboxing** — обратное: компилятор добавляет вызов `intValue()`, `doubleValue()` и т.п.
при присваивании обёртки в примитивную переменную.

**Ловушки:**
- Unboxing из `null` вызывает `NullPointerException` в рантайме.
- Автоупаковка в цикле с частой операцией порождает множество временных объектов — это
  негативно сказывается на производительности. Предпочтительнее использовать примитивный тип.
- `Integer.valueOf()` кэширует значения в диапазоне -128…127 (детали зависят от JVM),
  поэтому `Integer a = 127; Integer b = 127; a == b` даёт `true`, а `Integer a = 128; ...`
  уже `false`. Для сравнения по значению всегда используйте `equals()`.

### Классы, объекты, конструкторы

Класс — шаблон, объект — его экземпляр, созданный оператором `new`.
Источник: [Classes — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/javaOO/classes.html).

```java
public class Bicycle {
    private int cadence;
    private int gear;
    private int speed;

    public Bicycle(int cadence, int speed, int gear) {
        this.cadence = cadence;
        this.speed   = speed;
        this.gear    = gear;
    }

    public void applyBrake(int decrement) {
        speed -= decrement;
    }
}
```

**Конструкторы:**
- Имя совпадает с именем класса, нет возвращаемого типа.
- Если конструктор не объявлен, компилятор добавляет **конструктор по умолчанию** (без параметров).
  Как только объявлен хотя бы один конструктор — конструктор по умолчанию не генерируется.
- **Перегрузка конструкторов** — несколько конструкторов с разными сигнатурами.
- `this(...)` вызывает другой конструктор того же класса (должен быть первым оператором).
- `super(...)` вызывает конструктор суперкласса.

**Ключевые слова:**
- `this` — ссылка на текущий объект; используется для разрешения конфликта имён между
  полем и параметром и для вызова другого конструктора.
- `static` — поле или метод принадлежит классу, а не экземпляру. Доступны без создания объекта.
- `final` применительно к полю — значение нельзя изменить после инициализации; применительно
  к методу — нельзя переопределить; применительно к классу — нельзя наследовать.

### Модификаторы доступа

Источник: [Controlling Access to Members of a Class — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/javaOO/accesscontrol.html).

| Модификатор      | Свой класс | Пакет | Подкласс (другой пакет) | Все |
|------------------|:----------:|:-----:|:-----------------------:|:---:|
| `public`         | да         | да    | да                      | да  |
| `protected`      | да         | да    | да                      | нет |
| package-private  | да         | да    | нет                     | нет |
| `private`        | да         | нет   | нет                     | нет |

**Рекомендации из официальной документации:**
- По умолчанию используйте самый ограниченный уровень (`private`).
- Избегайте `public` полей, кроме констант.
- `protected` членами пользуются подклассы, даже находящиеся в других пакетах.
- Package-private (без модификатора) — хороший выбор для служебных классов внутри одного пакета.

### Наследование и полиморфизм

Источник: [Inheritance — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/IandI/subclasses.html).

Java поддерживает **одиночное наследование классов**: класс может расширять только один суперкласс.
Ключевое слово `extends`. Все классы неявно расширяют `Object`, если `extends` не указан.

```java
public class MountainBike extends Bicycle {
    private int seatHeight;

    public MountainBike(int h, int cadence, int speed, int gear) {
        super(cadence, speed, gear);  // конструктор суперкласса
        seatHeight = h;
    }
}
```

**Переопределение методов (overriding):**
- Подкласс предоставляет собственную реализацию метода суперкласса с той же сигнатурой.
- Аннотация `@Override` помогает компилятору проверить корректность переопределения.
- Доступ к реализации суперкласса: `super.methodName()`.
- **Полиморфизм:** при вызове метода через ссылку суперкласса JVM в рантайме диспетчеризует
  вызов к фактическому типу объекта (динамическое связывание).

```java
Bicycle b = new MountainBike(10, 60, 30, 5);
b.applyBrake(5);  // вызовется метод реального класса MountainBike
```

**Перегрузка (overloading)** отличается от переопределения: методы с одним именем, но разными
параметрами в одном классе. Разрешается на этапе компиляции (статическое связывание).

**Скрытие полей (hiding fields):** если в подклассе объявлено поле с тем же именем, что
и у суперкласса, оно скрывает поле суперкласса, но не переопределяет его. Это нежелательная
практика, ведущая к запутанному коду.

### Абстрактные классы

Источник: [Abstract Methods and Classes — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/IandI/abstract.html).

Абстрактный класс объявляется с `abstract` и не может быть инстанциирован напрямую.
Абстрактный метод объявляется без тела.

```java
public abstract class GraphicObject {
    int x, y;

    void moveTo(int newX, int newY) { /* реализация */ }

    abstract void draw();
    abstract void resize();
}

class Circle extends GraphicObject {
    void draw()   { /* реализация */ }
    void resize() { /* реализация */ }
}
```

**Правила:**
- Если класс содержит хотя бы один абстрактный метод — он **обязан** быть `abstract`.
- Подкласс обязан реализовать все абстрактные методы; если не реализует — сам должен быть `abstract`.
- Абстрактный класс может содержать поля любого типа (`private`, `protected`, нестатические, нефинальные),
  конкретные методы, конструкторы.
- Абстрактный класс может частично реализовывать интерфейс; оставшиеся методы реализуют подклассы.

**Когда выбирать абстрактный класс вместо интерфейса:**
- Нужно разделить код между тесно связанными классами.
- Нужны нестатические, нефинальные поля или методы с `protected`/`private` доступом.

### Интерфейсы

Источник: [Interfaces — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/IandI/createinterface.html).

Интерфейс — это контракт: набор методов (и констант), которые обязан реализовать имплементирующий
класс. Класс может реализовывать несколько интерфейсов (`implements A, B, C`).

**Состав интерфейса:**
- Абстрактные методы (неявно `public abstract`).
- Константы (неявно `public static final`).
- `default`-методы с реализацией (Java 8+).
- `static`-методы (Java 8+).
- `private`-методы (Java 9+) — вспомогательные, не видны имплементаторам.

```java
public interface TimeClient {
    void setTime(int hour, int minute, int second);

    default ZonedDateTime getZonedDateTime(String zoneString) {
        return ZonedDateTime.of(getLocalDateTime(), getZoneId(zoneString));
    }

    static ZoneId getZoneId(String zoneString) {
        try {
            return ZoneId.of(zoneString);
        } catch (DateTimeException e) {
            return ZoneId.systemDefault();
        }
    }
}
```

**Default-методы** (Java 8+):
Источник: [Default Methods — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/IandI/defaultmethods.html).

- Позволяют добавлять новую функциональность в интерфейс, не ломая существующие реализации
  (бинарная совместимость).
- Имплементирующий класс может: унаследовать, переопределить или объявить метод абстрактным.
- Конфликт дефолтных методов из нескольких интерфейсов необходимо разрешить явным переопределением
  в классе.

**Абстрактный класс vs. интерфейс:**

| Критерий               | Абстрактный класс          | Интерфейс                      |
|------------------------|----------------------------|--------------------------------|
| Инстанциирование       | Нет                        | Нет                            |
| Поля                   | Нестатические, нефинальные | Только `public static final`   |
| Методы                 | Любые модификаторы доступа | `public` (или `private` Java 9+)|
| Наследование           | Только одно                | Множественное (`implements`)   |
| Когда применять        | Тесно связанные классы     | Несвязанные классы, типы-роли  |

### Класс `Object` и его ключевые методы

Источник: [Methods of the Object class — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/IandI/objectclass.html).

Все Java-классы неявно наследуют `java.lang.Object`. Три наиболее важных метода, которые
принято переопределять:

**`equals(Object obj)`**
- По умолчанию сравнивает ссылки (`==`), то есть проверяет идентичность объектов.
- Переопределяется для семантического сравнения (по содержимому).
- Контракт: рефлексивность, симметричность, транзитивность, постоянство, `equals(null) == false`.

```java
@Override
public boolean equals(Object obj) {
    if (this == obj) return true;
    if (!(obj instanceof Book)) return false;
    Book book = (Book) obj;
    return ISBN.equals(book.ISBN);
}
```

**`hashCode()`**
- Возвращает целочисленный хэш объекта.
- **Контракт:** если `a.equals(b) == true`, то `a.hashCode() == b.hashCode()`.
- Нарушение контракта ломает работу `HashMap`, `HashSet` и аналогичных коллекций.
- При переопределении `equals()` **обязательно** переопределять `hashCode()`.

**`toString()`**
- Возвращает строковое представление объекта; дефолтная реализация — `ClassName@hashCode`.
- Рекомендуется переопределять для удобной отладки и логирования.
- Автоматически вызывается при конкатенации через `+` и в `System.out.println()`.

**`getClass()`** — `final`, не переопределяется; возвращает объект `Class<?>` с рефлексивной
информацией о типе.

### Класс `String` и пул строк

Источник: [Strings — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/data/strings.html).

`String` — неизменяемый (`immutable`) ссылочный тип со специальной языковой поддержкой.

```java
String s1 = "Hello";          // литерал — помещается в String Pool
String s2 = new String("Hi"); // явное создание объекта в куче
```

**String Pool** (пул строк):
- При использовании строковых литералов JVM помещает объект в пул. Повторные литералы
  с одинаковым значением указывают на один и тот же объект.
- `String.intern()` помещает строку в пул вручную.
- `s1 == s2` сравнивает ссылки (не содержимое!); всегда используйте `s1.equals(s2)`.

**Неизменяемость:** любая операция, «изменяющая» строку, возвращает новый объект.
Это потокобезопасно, но неэффективно при большом количестве конкатенаций.

**Ключевые методы:** `length()`, `charAt(int)`, `substring(int, int)`, `indexOf(String)`,
`contains(String)`, `toUpperCase()`, `toLowerCase()`, `trim()`, `strip()` (Java 11+),
`startsWith()`, `endsWith()`, `replace()`, `split()`, `format()`.

**`StringBuilder` vs `String`:**
- `StringBuilder` — изменяемый (mutable) аналог. Использовать, когда строка формируется
  в цикле или из многих частей.
- `StringBuffer` — потокобезопасный вариант (методы синхронизированы), но медленнее;
  нужен только при конкурентном доступе к одному буферу.

```java
StringBuilder sb = new StringBuilder();
for (String word : words) {
    sb.append(word).append(" ");
}
String result = sb.toString();
```

### Перечисления (`enum`)

Источник: [Enum Types — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/javaOO/enum.html).

`enum` — специальный класс, неявно расширяющий `java.lang.Enum`. Задаёт фиксированный
набор именованных констант.

```java
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}
```

**Возможности:**
- Можно добавлять поля, конструкторы (только `private` или package-private) и методы.
- `values()` — возвращает массив всех констант в порядке объявления.
- `ordinal()` — порядковый номер константы (с нуля); использовать с осторожностью.
- `name()` / `toString()` — строковое имя константы.
- Удобны в `switch`-выражениях.

```java
public enum Planet {
    MERCURY(3.303e+23, 2.4397e6),
    EARTH  (5.976e+24, 6.37814e6);

    private final double mass;
    private final double radius;
    static final double G = 6.67300E-11;

    Planet(double mass, double radius) {
        this.mass   = mass;
        this.radius = radius;
    }

    double surfaceGravity() {
        return G * mass / (radius * radius);
    }
}
```

**Когда использовать:** любой фиксированный набор значений, известных на этапе компиляции
(стороны света, планеты, статусы заказа, опции командной строки).

### Вложенные и анонимные классы

Источник: [Nested Classes — The Java Tutorials](https://docs.oracle.com/javase/tutorial/java/javaOO/nested.html).

| Вид                  | Объявление                       | Доступ к внешнему классу        | Статические члены |
|----------------------|----------------------------------|----------------------------------|-------------------|
| Статический вложенный | `static class Inner { }`         | Только через ссылку              | Разрешены         |
| Внутренний (нестатический) | `class Inner { }`           | Прямой (включая `private`)       | Запрещены         |
| Локальный            | Внутри метода                    | Прямой (через effectively final) | Запрещены         |
| Анонимный            | `new Interface() { ... }`        | Прямой (через effectively final) | Запрещены         |

**Внутренний класс** требует экземпляра внешнего класса для своего создания:
```java
OuterClass outer = new OuterClass();
OuterClass.InnerClass inner = outer.new InnerClass();
```

**Статический вложенный класс** создаётся как обычный класс верхнего уровня:
```java
OuterClass.StaticNested nested = new OuterClass.StaticNested();
```

**Shadowing (затенение):** если вложенный класс объявляет переменную с тем же именем,
что и внешний, внешняя переменная скрывается. Для доступа к ней: `OuterClass.this.field`.

**Сериализация** внутренних, локальных и анонимных классов настоятельно не рекомендуется
из-за зависящих от компилятора синтетических конструкций.

### Ключевые слова `static` и `final`

**`static`:**
- Поле `static` — одно на весь класс, разделяется всеми экземплярами.
- Метод `static` — вызывается через имя класса, не имеет доступа к `this` и нестатическим членам.
- Статический инициализатор `static { ... }` — выполняется один раз при загрузке класса.

**`final`:**
- Поле `final` — инициализируется один раз (в объявлении или конструкторе) и более не меняется.
- Метод `final` — не может быть переопределён в подклассе.
- Класс `final` — не может иметь подклассов (например, `String`, `Integer`).
- Параметр `final` — значение параметра нельзя изменить внутри метода.

```java
public static final double PI = 3.14159; // константа: static + final
```

## Достоверные источники

1. **[The Java Tutorials — Learning the Java Language](https://docs.oracle.com/javase/tutorial/java/index.html)**
   Официальный учебник Oracle: классы, объекты, интерфейсы, наследование, перечисления, вложенные классы.
   Первоисточник по синтаксису и базовым концепциям ООП.

2. **[Java Language Specification (JLS)](https://docs.oracle.com/javase/specs/)**
   Официальная спецификация языка. Авторитетный источник для тонких вопросов семантики:
   порядок инициализации, видимость, правила приведения типов, формальное определение boxing/unboxing.

3. **[Primitive Data Types — Oracle Tutorials](https://docs.oracle.com/javase/tutorial/java/nutsandbolts/datatypes.html)**
   Подробная страница по всем восьми примитивным типам, литералам и числовым форматам.
   Официальный источник — первичный по диапазонам и значениям по умолчанию.

4. **[Default Methods — Oracle Tutorials](https://docs.oracle.com/javase/tutorial/java/IandI/defaultmethods.html)**
   Официальная документация по default- и static-методам в интерфейсах (Java 8+). Объясняет
   правила разрешения конфликтов и цель добавления функциональности для обратной совместимости.

5. **[dev.java — Learn Java](https://dev.java/learn/)**
   Современный ресурс Oracle (обновляется для Java 17+), охватывает языковые изменения, не
   вошедшие в классические туториалы JDK 8. Достоверен как официальный обучающий портал Oracle.

6. **[Baeldung — Java Basics](https://www.baeldung.com/java-tutorial)**
   Признанный в сообществе ресурс: практические статьи с примерами по основам языка и ООП.
   Не является официальной документацией, но регулярно проверяется на актуальность и точность.
