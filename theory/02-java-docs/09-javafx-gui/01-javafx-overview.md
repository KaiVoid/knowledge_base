## Приложения JavaFX

Поскольку библиотека JavaFX написана как Java API, код приложения JavaFX может обращаться к API
любой Java-библиотеки. Например, приложения JavaFX могут использовать библиотеки Java API для
доступа к нативным возможностям системы (*native system capabilities*) и подключения к
серверным middleware-приложениям.

Внешний вид и поведение (*look and feel*) приложений JavaFX можно настраивать. Каскадные таблицы
стилей (Cascading Style Sheets, CSS) отделяют оформление и стиль от реализации, чтобы разработчики
могли сосредоточиться на написании кода. Графические дизайнеры могут легко настраивать внешний
вид и стиль приложения через CSS. Если у вас есть опыт веб-дизайна или если вы хотите отделить
пользовательский интерфейс (user interface, UI) от внутренней логики (*back-end logic*), то вы
можете разрабатывать презентационные аспекты UI на языке разметки FXML, а логику приложения
писать на Java-коде. Если вы предпочитаете проектировать пользовательские интерфейсы без
написания кода, используйте JavaFX Scene Builder. По мере проектирования UI Scene Builder
создаёт разметку FXML, которую можно перенести в интегрированную среду разработки (Integrated
Development Environment, IDE), чтобы разработчики могли добавить бизнес-логику.

## Доступность

API JavaFX доступны как полностью интегрированная возможность среды выполнения Java SE (Java SE
Runtime Environment, JRE) и комплекта разработчика Java (Java Development Kit, JDK). Поскольку JDK
доступен для всех основных настольных платформ (Windows, Mac OS X и Linux), приложения JavaFX,
скомпилированные под JDK 7 и более поздние версии, также работают на всех основных настольных
платформах. С выпуском JavaFX 8 появилась поддержка платформ ARM. JDK для ARM включает базовый
компонент (*base*), графику (*graphics*) и компоненты управления (*controls*) JavaFX.

Кроссплатформенная совместимость обеспечивает единообразную работу приложений JavaFX как для
разработчиков, так и для пользователей. Oracle обеспечивает синхронизированные выпуски и
обновления на всех платформах и предлагает обширную программу поддержки для компаний, которые
запускают критически важные приложения (*mission-critical applications*).

На странице загрузки JDK (*JDK download page*) можно получить zip-файл с примерами приложений
JavaFX. Эти примеры содержат множество образцов кода и фрагментов, которые на примерах
показывают, как писать приложения JavaFX. Подробнее см. [«Как запустить пример приложения?»](#как-запустить-пример-приложения).

## Ключевые возможности

Перечисленные ниже возможности входят в состав JavaFX 8 и более поздних выпусков. Элементы,
которые были введены в выпуске JavaFX 8, отмечены соответствующим образом:

- **Java API.** JavaFX — это Java-библиотека, состоящая из классов и интерфейсов, написанных на
  Java-коде. API спроектированы как удобная альтернатива для языков виртуальной машины Java
  (Java Virtual Machine, Java VM), таких как JRuby и Scala.

- **FXML и Scene Builder.** FXML — это декларативный язык разметки на основе XML
  (*XML-based declarative markup language*) для построения пользовательского интерфейса
  приложения JavaFX. Дизайнер может писать на FXML или использовать JavaFX Scene Builder для
  интерактивного проектирования графического пользовательского интерфейса (graphical user
  interface, GUI). Scene Builder генерирует разметку FXML, которую можно перенести в IDE, где
  разработчик добавит бизнес-логику.

- **WebView.** Веб-компонент, использующий технологию WebKitHTML, что позволяет встраивать
  веб-страницы в приложение JavaFX. JavaScript, выполняющийся в WebView, может вызывать Java API,
  а Java API могут вызывать JavaScript, выполняющийся в WebView. В JavaFX 8 добавлена поддержка
  дополнительных возможностей HTML5, включая веб-сокеты (*Web Sockets*), веб-воркеры
  (*Web Workers*) и веб-шрифты (*Web Fonts*), а также возможности печати. См. *Adding HTML Content
  to JavaFX Applications*.

- **Совместимость со Swing (*Swing interoperability*).** Существующие приложения Swing можно
  дополнить возможностями JavaFX, такими как насыщенная графика, воспроизведение мультимедиа и
  встроенное веб-содержимое. В JavaFX 8 добавлен класс `SwingNode`, который позволяет встраивать
  содержимое Swing в приложения JavaFX. Подробнее см. документацию API `SwingNode` (javadoc) и
  *Embedding Swing Content in JavaFX Applications*.

- **Встроенные элементы управления UI и CSS (*Built-in UI controls and CSS*).** JavaFX
  предоставляет все основные элементы управления UI, необходимые для разработки полнофункционального
  приложения. Компонентам можно задавать оформление (*skinned*) с помощью стандартных
  веб-технологий, таких как CSS. В выпуске JavaFX 8 стали доступны элементы управления UI
  DatePicker и TreeTableView. Подробнее см. *Using JavaFX UI Controls*. Кроме того, классы
  Styleable\* в CSS стали публичным API, что позволяет стилизовать объекты с помощью CSS.

- **Тема Modena (*Modena theme*).** Тема Modena заменяет тему Caspian в качестве темы по
  умолчанию для приложений JavaFX 8. Тема Caspian по-прежнему доступна для использования: для
  этого добавьте строку `setUserAgentStylesheet(STYLESHEET_CASPIAN)` в метод `start()` вашего
  приложения. Подробнее см. блог о Modena на сайте fxexperience.com.

- **Возможности трёхмерной графики (*3D Graphics Features*).** В библиотеку трёхмерной графики
  JavaFX 8 добавлены новые классы API: `Shape3D` (подклассы `Box`, `Cylinder`, `MeshView` и
  `Sphere`), `SubScene`, `Material`, `PickResult`, `LightBase` (подклассы `AmbientLight` и
  `PointLight`), а также `SceneAntialiasing`. Класс API `Camera` в этом выпуске также обновлён.
  Подробнее см. документ *Getting Started with JavaFX 3D Graphics* и соответствующую документацию
  API (javadoc) для `javafx.scene.shape.Shape3D`, `javafx.scene.SubScene`,
  `javafx.scene.paint.Material`, `javafx.scene.input.PickResult` и `javafx.scene.SceneAntialiasing`.

- **Canvas API.** API Canvas позволяет рисовать непосредственно внутри области сцены JavaFX,
  состоящей из одного графического элемента (узла, *node*).

- **API печати (*Printing API*).** В выпуск Java SE 8 добавлен пакет `javafx.print`, который
  предоставляет публичные классы для API печати JavaFX (*JavaFX Printing API*).

- **Поддержка форматированного текста (*Rich Text Support*).** JavaFX 8 расширяет поддержку
  текста в JavaFX, включая двунаправленный текст (*bi-directional text*) и сложные текстовые
  системы письма (*complex text scripts*), такие как тайская и хинди, в элементах управления, а
  также многострочный текст с несколькими стилями в текстовых узлах.

- **Поддержка мультитач (*Multitouch Support*).** JavaFX обеспечивает поддержку
  мультисенсорных операций (*multitouch operations*) на основе возможностей нижележащей
  платформы.

- **Поддержка Hi-DPI (*Hi-DPI support*).** JavaFX 8 теперь поддерживает дисплеи с высоким
  разрешением (Hi-DPI).

- **Аппаратно-ускоренный графический конвейер (*Hardware-accelerated graphics pipeline*).**
  Графика JavaFX основана на графическом конвейере отрисовки Prism. JavaFX обеспечивает плавную
  графику, которая быстро отрисовывается через Prism, когда он используется с поддерживаемой
  видеокартой или графическим процессором (graphics processing unit, GPU). Если в системе нет
  ни одного из рекомендуемых GPU, поддерживаемых JavaFX, то Prism по умолчанию переходит на
  программный стек отрисовки (*software rendering stack*).

- **Высокопроизводительный мультимедийный движок (*High-performance media engine*).**
  Мультимедийный конвейер (*media pipeline*) поддерживает воспроизведение веб-мультимедиа. Он
  предоставляет стабильный мультимедийный фреймворк с малой задержкой (*low-latency*), основанный
  на мультимедийном фреймворке GStreamer.

- **Модель развёртывания самодостаточных приложений (*Self-contained application deployment
  model*).** Пакеты самодостаточных приложений содержат все ресурсы приложения и собственную
  копию сред выполнения Java и JavaFX. Они распространяются как нативные устанавливаемые пакеты и
  обеспечивают такой же опыт установки и запуска, как и нативные приложения для данной
  операционной системы.

## Что я могу создать с помощью JavaFX?

С помощью JavaFX можно создавать множество типов приложений. Как правило, это сетевые приложения
(*network-aware applications*), которые развёртываются на нескольких платформах и отображают
информацию в высокопроизводительном современном пользовательском интерфейсе с аудио, видео,
графикой и анимацией.

В таблице 1-1 показаны изображения нескольких примеров приложений JavaFX, входящих в состав
выпуска JavaFX 8.n.

### Таблица 1-1. Примеры приложений JavaFX

| Пример приложения | Описание |
|---|---|
| **JavaFX Ensemble8** | Ensemble8 — это галерея примеров приложений, демонстрирующих большое разнообразие возможностей JavaFX, включая анимацию, диаграммы и элементы управления. Вы можете просматривать каждый работающий пример и взаимодействовать с ним на ВСЕХ платформах, а также читать его описания. На настольных платформах вы можете копировать исходный код каждого примера, настраивать свойства компонентов примера, используемых в нескольких образцах, и переходить по ссылкам на соответствующую документацию API при наличии подключения к Интернету. Ensemble8 также работает с JavaFX для ARM. |
| **Modena** | Modena — это пример приложения, демонстрирующий внешний вид и поведение UI-компонентов с использованием темы Modena. Он позволяет сравнить темы Modena и Caspian и изучить различные аспекты этих тем. |
| **3D Viewer** | 3DViewer — это пример приложения, который позволяет перемещаться по трёхмерной сцене и изучать её с помощью мыши или трекпада. 3DViewer имеет импортёры для подмножества возможностей файлов OBJ и Maya. Для файлов Maya также предусмотрена возможность импорта анимации. (Обратите внимание, что в случае файлов Maya при сохранении в формате Maya следует удалить историю построения (*construction history*) для всех объектов.) 3DViewer также способен экспортировать содержимое сцены в виде файлов Java или FXML. |

## Как запустить пример приложения?

Шаги в этом разделе объясняют, как загрузить и запустить примеры приложений, доступные как
отдельная загрузка вместе с платформой Java (JDK 8).

> **Примечание.** Прежде чем запускать пример приложения JavaFX, на вашей машине должны быть
> установлены библиотеки среды выполнения JavaFX (*JavaFX runtime libraries*). Перед выполнением
> этих шагов установите либо последнюю версию JDK 8, либо последнюю версию JRE.

Чтобы загрузить и запустить примеры приложений:

1. Перейдите на страницу загрузки Java SE (Java SE Downloads):
   `https://www.oracle.com/technetwork/java/javase/downloads/`.
2. Прокрутите страницу вниз до раздела JDK 8 and JavaFX Demos and Samples.
3. Нажмите кнопку Demos and Samples Download, чтобы перейти на страницу загрузок.
4. На странице Java SE Development Kit 8 Downloads прокрутите вниз до раздела JavaFX Demos and
   Samples Downloads.
5. Загрузите zip-файл для нужной операционной системы и распакуйте файлы.

   Будет создан каталог `javafx-samples-8.x`, содержащий файлы доступных примеров. Проекты
   NetBeans для примеров находятся в каталоге `javafx-samples-8.x\src`.
6. Дважды щёлкните исполняемый файл нужного примера.

   Например, чтобы запустить готовый пример приложения Ensemble8, дважды щёлкните файл
   `Ensemble8.jar`.

## Как запустить пример в IDE?

Для разработки приложений JavaFX можно использовать несколько Java-IDE. Шаги ниже объясняют, как
просмотреть и запустить исходный код в среде NetBeans IDE.

Чтобы просмотреть и запустить исходный код примера в NetBeans IDE:

1. Загрузите примеры, как описано выше, и распакуйте файлы.
2. В среде NetBeans 7.4 или более поздней версии загрузите проект примера, который хотите
   просмотреть.
   1. В меню File выберите Open Project.
   2. В диалоговом окне Open Project перейдите в каталог со списком примеров. Путь навигации
      выглядит примерно так:

      `..\javafx_samples-8.x-<platform>\javafx-samples-8.x\src`
   3. Выберите пример, который хотите просмотреть.
   4. Нажмите кнопку Open Project.
3. В окне Projects щёлкните правой кнопкой мыши только что открытый проект и выберите Run.
   Обратите внимание: окно Output обновляется, проект-пример запускается и развёртывается.

## Как создать приложение JavaFX?

Поскольку приложения JavaFX пишутся на языке Java, для их создания можно использовать любой
любимый редактор или любую интегрированную среду разработки (IDE), поддерживающую язык Java
(такую как NetBeans, Eclipse или IntelliJ IDEA).

Чтобы создавать приложения JavaFX:

1. Перейдите на страницу загрузки Java SE (Java SE Downloads):
   `https://www.oracle.com/technetwork/java/javase/downloads/`, чтобы загрузить Oracle® JDK 8 с
   поддержкой JavaFX 8.n. На этой же странице доступны ссылки на сертифицированные конфигурации
   систем и примечания к выпуску.
2. Воспользуйтесь руководством *Getting Started with JavaFX Sample Applications*, чтобы создать
   простые приложения, демонстрирующие работу с макетами (*layouts*), таблицами стилей и
   визуальными эффектами.
3. Используйте JavaFX Scene Builder для проектирования UI вашего приложения JavaFX без написания
   кода. Вы можете перетаскивать UI-компоненты в рабочую область, изменять их свойства,
   применять таблицы стилей и интегрировать полученный код с логикой приложения.
   1. Загрузите JavaFX Scene Builder со страницы загрузок JavaFX:
      `https://www.oracle.com/technetwork/java/javase/downloads/`.
   2. Следуйте руководству *Getting Started with JavaFX Scene Builder*, чтобы узнать больше.
