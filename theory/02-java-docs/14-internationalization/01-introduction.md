# Урок 1. Введение в интернационализацию

**Трейл:** Internationalization · **Оригинал:** [Introduction](https://docs.oracle.com/javase/tutorial/i18n/intro/index.html)
**Связанные области:** [[01-core-java-syntax-oop]] · **Вопросы:** core-java

> Перевод официального руководства Oracle (The Java Tutorials, JDK 8). Объединяет страницы
> *Lesson: Introduction*, *A Quick Example* (*Before Internationalization*, *After Internationalization*)
> и *Checklist*.

**Интернационализация** (*internationalization*) — это процесс проектирования приложения
таким образом, чтобы его можно было адаптировать к различным языкам и регионам без изменений
в исходном коде. Иногда термин «интернационализация» сокращают до **i18n**, потому что между
первой буквой «i» и последней буквой «n» в английском слове *internationalization* находится
18 букв.

Интернационализированная программа обладает следующими свойствами:

- При добавлении локализованных данных один и тот же исполняемый файл может работать по всему миру.
- Текстовые элементы — такие как сообщения о состоянии и подписи компонентов графического
  интерфейса (GUI) — не «зашиты» (*hardcoded*) в программе. Вместо этого они хранятся вне
  исходного кода и извлекаются динамически.
- Поддержка новых языков не требует перекомпиляции.
- Зависящие от культуры данные — например, даты и денежные суммы — отображаются в форматах,
  соответствующих региону и языку конечного пользователя.
- Её можно быстро локализовать.

**Локализация** (*localization*) — это процесс адаптации программного обеспечения для
конкретного региона или языка путём добавления зависящих от локали компонентов и перевода
текста. Термин «локализация» часто сокращают до **l10n**, потому что между буквами «l» и «n»
в английском слове *localization* находится 10 букв.

Основная задача локализации — перевод элементов пользовательского интерфейса и документации.
Локализация включает не только изменение языка взаимодействия, но и другие значимые изменения,
такие как отображение чисел, дат, валют и т. п. Другие типы данных, например звуки и изображения,
тоже могут требовать локализации, если они культурно чувствительны. Чем лучше интернационализировано
приложение, тем легче локализовать его для конкретного языка и схемы кодирования символов.

Поначалу интернационализация может показаться немного пугающей. Чтение следующих разделов
поможет вам легче войти в эту тему.

## Краткий пример (A Quick Example)

Этот раздел шаг за шагом показывает, как интернационализировать простую программу, чтобы она
отображала текстовые сообщения на подходящем языке. Вы узнаете, как объекты `Locale` и
`ResourceBundle` работают вместе и как использовать файлы свойств (*properties files*).

### До интернационализации (Before Internationalization)

Первая версия исходного кода содержала «зашитые» (*hardcoded*) английские варианты сообщений,
которые мы хотим отобразить. Так писать интернационализированное программное обеспечение
**не следует**.

Предположим, вы написали программу, которая выводит три сообщения:

```java
public class NotI18N {

    static public void main(String[] args) {

        System.out.println("Hello.");
        System.out.println("How are you?");
        System.out.println("Goodbye.");
    }
}
```

Вы решили, что эта программа должна выводить те же самые сообщения для людей, живущих во
Франции и Германии. К сожалению, ваш штат программистов не владеет несколькими языками,
поэтому вам понадобится помощь в переводе сообщений на французский и немецкий. Поскольку
переводчики — не программисты, вам придётся вынести сообщения из исходного кода в текстовые
файлы, которые переводчики смогут редактировать. Кроме того, программа должна быть достаточно
гибкой, чтобы отображать сообщения и на других языках, но прямо сейчас никто не знает, какими
будут эти языки.

Похоже, программу нужно интернационализировать.

### После интернационализации (After Internationalization)

Это предварительный обзор того, как будет выглядеть наш исходный код после интернационализации.
Обратите внимание, что текст сообщений не включён в код программы.

```java
import java.util.*;

public class I18NSample {

    static public void main(String[] args) {

        String language;
        String country;

        if (args.length != 2) {
            language = new String("en");
            country = new String("US");
        } else {
            language = new String(args[0]);
            country = new String(args[1]);
        }

        Locale currentLocale;
        ResourceBundle messages;

        currentLocale = new Locale(language, country);

        messages = ResourceBundle.getBundle("MessagesBundle", currentLocale);
        System.out.println(messages.getString("greetings"));
        System.out.println(messages.getString("inquiry"));
        System.out.println(messages.getString("farewell"));
    }
}
```

Чтобы скомпилировать и запустить эту программу, понадобятся следующие исходные файлы:

- `I18NSample.java`
- `MessagesBundle.properties`
- `MessagesBundle_de_DE.properties`
- `MessagesBundle_en_US.properties`
- `MessagesBundle_fr_FR.properties`

> **Примечание.** В оригинальном руководстве за этим разделом следуют отдельные уроки
> *Running the Sample Program* (запуск примера) и *Internationalizing the Sample Program*
> (интернационализация примера), которые будут переведены отдельно.

## Контрольный список (Checklist)

Итак, вам достался для интернационализации унаследованный код, или вы планируете определить
требования к недавно разработанному программному обеспечению. Вероятно, вы не знаете, с чего
начать. Изучите этот контрольный список: он обобщает необходимые задачи интернационализации
и даёт ссылки на соответствующие уроки этой главы.

Многие программы не были интернационализированы при первоначальном написании. Такие программы
могли начинаться как прототипы, или, возможно, они не предназначались для международного
распространения. Если вам нужно интернационализировать существующую программу, выполните
следующие шаги.

### Определите данные, зависящие от культуры

Текстовые сообщения — самая очевидная форма данных, которые меняются в зависимости от культуры.
Однако и другие типы данных могут изменяться в зависимости от региона или языка. В следующем
списке приведены примеры данных, зависящих от культуры:

- сообщения (*messages*);
- подписи на компонентах графического интерфейса (GUI);
- онлайн-справка;
- звуки;
- цвета;
- графика;
- значки (*icons*);
- даты;
- время;
- числа;
- денежные суммы (валюты);
- меры (единицы измерения);
- телефонные номера;
- почётные обращения и личные титулы (*honorifics and personal titles*);
- почтовые адреса;
- макеты страниц (*page layouts*).

### Изолируйте переводимый текст в ресурсных пакетах

Перевод стоит дорого. Вы можете помочь снизить затраты, изолировав текст, требующий перевода,
в объектах `ResourceBundle` (ресурсных пакетах). К переводимому тексту относятся сообщения о
состоянии, сообщения об ошибках, записи в файлах журнала и подписи компонентов GUI. Такой текст
встречается в программах, которые не были интернационализированы. Вам нужно найти все случаи
встроенного текста, отображаемого конечным пользователям. Например, следует исправить код вроде
такого:

```java
String buttonLabel = "OK";
// ...
JButton okButton = new JButton(buttonLabel);
```

Подробности см. в разделе *Isolating Locale-Specific Data* ([Изоляция данных, зависящих от локали](https://docs.oracle.com/javase/tutorial/i18n/resbundle/index.html)).

### Разберитесь со составными сообщениями

Составные сообщения (*compound messages*) содержат переменные данные. В сообщении
«The disk contains 1100 files.» («На диске 1100 файлов.») целое число 1100 может меняться.
Такое сообщение трудно переводить, потому что позиция числа в предложении не одинакова во всех
языках. Следующее сообщение непереводимо, поскольку порядок элементов предложения зафиксирован
конкатенацией:

```java
Integer fileCount;
// ...
String diskStatus = "The disk contains " + fileCount.toString() + " files";
```

Когда это возможно, следует избегать построения составных сообщений, потому что их трудно
переводить. Однако если вашему приложению нужны составные сообщения, вы можете обрабатывать
их с помощью приёмов, описанных в разделе *Messages*
([Сообщения](https://docs.oracle.com/javase/tutorial/i18n/format/messageintro.html)).

### Форматируйте числа и денежные суммы

Если ваше приложение отображает числа и денежные суммы, вы должны форматировать их
независимым от локали способом. Следующий код ещё не интернационализирован, потому что он
не отобразит число правильно во всех странах:

```java
Double amount;
TextField amountField;
// ...
String displayAmount = amount.toString();
amountField.setText(displayAmount);
```

Вам следует заменить приведённый выше код процедурой, которая форматирует число корректно.
Язык программирования Java предоставляет несколько классов, форматирующих числа и денежные
суммы. Эти классы рассматриваются в разделе *Numbers and Currencies*
([Числа и денежные суммы](https://docs.oracle.com/javase/tutorial/i18n/format/numberintro.html)).

### Форматируйте даты и время

Форматы даты и времени различаются в зависимости от региона и языка. Если ваш код содержит
операторы вроде следующего, его нужно изменить:

```java
Date currentDate = new Date();
TextField dateField;
// ...
String dateString = currentDate.toString();
dateField.setText(dateString);
```

Если вы используете классы форматирования дат, ваше приложение сможет правильно отображать
даты и время по всему миру. Примеры и инструкции см. в разделе *Dates and Times*
([Даты и время](https://docs.oracle.com/javase/tutorial/i18n/format/dateintro.html)).

### Используйте свойства символов Unicode

Следующий код пытается проверить, является ли символ буквой:

```java
char ch;
// Этот код неверен
if ((ch >= 'a' && ch <= 'z') || (ch >= 'A' && ch <= 'Z'))
```

Остерегайтесь подобного кода, потому что он не будет работать с языками, отличными от
английского. Например, оператор `if` пропустит символ ü в немецком слове Grün.

Методы сравнения класса `Character` используют стандарт Unicode для определения свойств
символов. Поэтому вам следует заменить предыдущий код на следующий:

```java
char ch;
// ...
if (Character.isLetter(ch))
```

Дополнительные сведения о методах сравнения класса `Character` см. в разделе
*Checking Character Properties* ([Проверка свойств символов](https://docs.oracle.com/javase/tutorial/i18n/text/charintro.html)).

### Сравнивайте строки правильно

При сортировке текста вы часто сравниваете строки. Если текст отображается, не следует
использовать методы сравнения класса `String`. Программа, которая не была интернационализирована,
может сравнивать строки следующим образом:

```java
String target;
String candidate;
// ...
if (target.equals(candidate)) {
// ...
if (target.compareTo(candidate) < 0) {
// ...
```

Методы `String.equals` и `String.compareTo` выполняют побитовое сравнение, которое неэффективно
при сортировке в большинстве языков. Вместо этого вам следует использовать класс `Collator`,
описанный в разделе *Comparing Strings*
([Сравнение строк](https://docs.oracle.com/javase/tutorial/i18n/text/collationintro.html)).

### Преобразуйте не-Unicode текст

Символы в языке программирования Java кодируются в Unicode. Если ваше приложение обрабатывает
не-Unicode текст, вам может потребоваться преобразовать его в Unicode. Дополнительные сведения
см. в разделе *Converting Non-Unicode Text*
([Преобразование не-Unicode текста](https://docs.oracle.com/javase/tutorial/i18n/text/convertintro.html)).
