# Урок 4. Форматирование

**Трейл:** Internationalization · **Оригинал:** [Formatting](https://docs.oracle.com/javase/tutorial/i18n/format/index.html)
**Связанные области:** [[01-core-java-syntax-oop]] · [[09-modern-java-features]] · **Вопросы:** core-java

> Перевод официального руководства Oracle (The Java Tutorials, JDK 8). Урок объединяет страницы
> трейла *Internationalization → Formatting*: введение, *Numbers and Currencies* (Numbers,
> Currencies, Percentages, DecimalFormat), *Dates and Times* (Dates, Customizing Formats /
> SimpleDateFormat, Changing Date Format Symbols) и *Messages* (Dealing with Compound Messages,
> Handling Plurals).

Этот урок объясняет, как форматировать числа, валюты, даты, время и текстовые сообщения.
Поскольку конечные пользователи видят эти данные, их формат должен соответствовать различным
культурным соглашениям. Изучив примеры этого урока, вы научитесь:

- форматировать элементы данных с учётом локали (*locale-sensitive*);
- сохранять код независимым от локали (*locale-independent*);
- избегать необходимости писать процедуры форматирования под конкретные локали.

Урок состоит из трёх частей:

- **Числа и валюты (Numbers and Currencies)** — классы `NumberFormat`, `DecimalFormat`
  и `DecimalFormatSymbols`.
- **Даты и время (Dates and Times)** — классы `DateFormat`, `SimpleDateFormat`
  и `DateFormatSymbols`.
- **Сообщения (Messages)** — классы `MessageFormat` и `ChoiceFormat`.

## Числа и валюты (Numbers and Currencies)

Программы хранят числа и работают с ними независимо от локали. Прежде чем вывести или напечатать
число, программа должна преобразовать его в строку (`String`) в формате, учитывающем локаль.
Например, во Франции число 123456.78 должно быть отформатировано как `123 456,78`, а в Германии
оно будет выглядеть как `123.456,78`. В этом разделе вы научитесь делать свои программы
независимыми от локальных соглашений о десятичном разделителе, разделителе тысяч и других
свойствах форматирования.

Раздел состоит из двух частей:

- **Использование предопределённых форматов (Using Predefined Formats).** С помощью фабричных
  методов класса `NumberFormat` можно получить специфичные для локали форматы чисел, валют
  и процентов.
- **Форматирование по шаблонам (Customizing Formats).** Класс `DecimalFormat` задаёт формат числа
  строкой-шаблоном (`String`). Класс `DecimalFormatSymbols` позволяет менять символы
  форматирования — например, десятичные разделители и знаки «минус».

### Числа (Numbers)

Методы класса [`NumberFormat`](https://docs.oracle.com/javase/8/docs/api/java/text/NumberFormat.html)
можно использовать для форматирования примитивных числовых типов, таких как `double`, и
соответствующих им объектов-обёрток, таких как `Double`.

Следующий пример форматирует объект `Double` в соответствии с локалью (`Locale`). Вызов метода
`getNumberInstance` возвращает специфичный для локали экземпляр `NumberFormat`. Метод `format`
принимает `Double` в качестве аргумента и возвращает отформатированное число в виде строки
`String`.

```java
static public void displayNumber(Locale currentLocale) {

    Integer quantity = new Integer(123456);
    Double amount = new Double(345987.246);
    NumberFormat numberFormatter;
    String quantityOut;
    String amountOut;

    numberFormatter = NumberFormat.getNumberInstance(currentLocale);
    quantityOut = numberFormatter.format(quantity);
    amountOut = numberFormatter.format(amount);
    System.out.println(quantityOut + "   " + currentLocale.toString());
    System.out.println(amountOut + "   " + currentLocale.toString());
}
```

Этот пример печатает следующее; видно, как формат одного и того же числа меняется в зависимости
от локали (`Locale`):

```
123 456   fr_FR
345 987,246   fr_FR
123.456   de_DE
345.987,246   de_DE
123,456   en_US
345,987.246   en_US
```

#### Использование начертаний цифр, отличных от арабских

По умолчанию, когда текст содержит числовые значения, они отображаются арабскими цифрами. Если
предпочтительны другие начертания цифр из Unicode, используйте класс
[`java.awt.font.NumericShaper`](https://docs.oracle.com/javase/8/docs/api/java/awt/font/NumericShaper.html).
API `NumericShaper` позволяет отобразить числовое значение, внутренне представленное как
ASCII-значение, любым начертанием цифр Unicode. Подробнее см. *Converting Latin Digits to Other
Unicode Digits*.

Кроме того, у некоторых локалей есть коды-варианты (*variant codes*), указывающие, что вместо
арабских цифр следует использовать unicode-начертания, — например, локаль для тайского языка.
Подробнее см. раздел *Variant Code* в *Creating a Locale*.

### Валюты (Currencies)

Если вы пишете коммерческое приложение, скорее всего, вам понадобится форматировать и выводить
денежные суммы. Подобно числам, денежные суммы форматируются с учётом локали с помощью класса
`NumberFormat`. Однако вместо метода `getNumberInstance` для получения форматтера валют
вызывается фабричный метод `getCurrencyInstance`. Возвращённый объект `NumberFormat`
форматирует число с учётом локали, добавляя соответствующий символ валюты и используя принятые
для данной локали разделители групп и десятичной части.

```java
static public void displayCurrency(Locale currentLocale) {

    Double currencyAmount = new Double(9876543.21);
    Currency currentCurrency = Currency.getInstance(currentLocale);
    NumberFormat currencyFormatter =
        NumberFormat.getCurrencyInstance(currentLocale);

    System.out.println(
        currentLocale.getDisplayName() + ", "
        + currentCurrency.getDisplayName() + ": "
        + currencyFormatter.format(currencyAmount));
}
```

Метод `format` форматирует переданную сумму, добавляя символ валюты и применяя соглашения локали
о разделителях. При вызове этого кода для разных значений `Locale` одна и та же сумма выводится
с разным символом валюты и разными разделителями: для французской и немецкой локалей в качестве
символа валюты используется знак евро, для локали США — знак доллара, при этом разделители групп
и десятичной части соответствуют каждой локали.

> **Примечание.** Точные строки вывода (с символами валют и разделителями) для страницы
> *Currencies* на момент перевода были недоступны: оригинальная страница Oracle временно
> перенаправляет на корневую страницу документации JDK. Описание поведения приведено по
> вводной странице *Numbers and Currencies* и документации `NumberFormat.getCurrencyInstance`;
> ничего не выдумано. См. примечание в секции «Источник».

### Проценты (Percentages)

Числа можно также форматировать как проценты с учётом локали. Чтобы отформатировать процент,
сначала получите экземпляр форматтера, вызвав фабричный метод `getPercentInstance`. С его помощью
дробное значение, например `0.75`, можно отобразить как `75 %`. Перед отображением форматтер
умножает значение на 100 и добавляет принятый в данной локали знак процента.

```java
static public void displayPercent(Locale currentLocale) {

    Double percent = new Double(0.75);
    NumberFormat percentFormatter =
        NumberFormat.getPercentInstance(currentLocale);
    String percentOut = percentFormatter.format(percent);
    System.out.println(percentOut + "   " + currentLocale.toString());
}
```

Метод `format` возвращает строку с числом, умноженным на 100, и знаком процента, оформленную
согласно соглашениям выбранной локали.

> **Примечание.** Точные строки вывода для страницы *Percentages* на момент перевода были
> недоступны (страница Oracle временно перенаправляет на корневую страницу документации JDK).
> Описание поведения приведено по вводной странице *Numbers and Currencies* и документации
> `NumberFormat.getPercentInstance`; ничего не выдумано. См. примечание в секции «Источник».

### Форматирование по шаблонам: DecimalFormat

Класс `DecimalFormat` позволяет форматировать десятичные числа в специфичные для локали строки.
Этот класс даёт контроль над отображением ведущих и завершающих нулей, префиксов и суффиксов,
разделителей групп (тысяч) и десятичного разделителя. Если нужно изменить сами символы
форматирования — например, десятичный разделитель, — используйте класс `DecimalFormatSymbols`
вместе с `DecimalFormat`. Эти классы дают большую гибкость в форматировании чисел, но могут
усложнить код.

#### Построение шаблонов

Свойства форматирования `DecimalFormat` задаются строкой-шаблоном (`String`). Шаблон определяет,
как будет выглядеть отформатированное число.

В примере ниже форматтер создаётся передачей строки-шаблона в конструктор `DecimalFormat`. Метод
`format` принимает значение типа `double` и возвращает отформатированное число в виде строки:

```java
DecimalFormat myFormatter = new DecimalFormat(pattern);
String output = myFormatter.format(value);
System.out.println(value + " " + pattern + " " + output);
```

Вывод программы `DecimalFormatDemo`:

| `value` | `pattern` | `output` | Пояснение |
|---------|-----------|----------|-----------|
| 123456.789 | `###,###.###` | 123,456.789 | Решётка (`#`) обозначает цифру, запятая — заполнитель разделителя групп, точка — заполнитель десятичного разделителя. |
| 123456.789 | `###.##` | 123456.79 | У значения три цифры справа от десятичной точки, а в шаблоне их только две. Метод `format` округляет в большую сторону. |
| 123.78 | `000000.000` | 000123.780 | Шаблон задаёт ведущие и завершающие нули, так как вместо решётки (`#`) используется символ `0`. |
| 12345.67 | `$###,###.###` | $12,345.67 | Первый символ шаблона — знак доллара (`$`). Обратите внимание: он стоит непосредственно перед крайней левой цифрой отформатированного вывода. |
| 12345.67 | `¥###,###.###` | ¥12,345.67 | Шаблон задаёт символ валюты японской иены (`¥`) через unicode-значение `00A5`. |

#### Форматирование с учётом локали

Предыдущий пример создавал объект `DecimalFormat` для локали по умолчанию. Чтобы получить объект
`DecimalFormat` для локали, отличной от умолчательной, создают `NumberFormat`, а затем приводят
его к типу `DecimalFormat`:

```java
NumberFormat nf = NumberFormat.getNumberInstance(loc);
DecimalFormat df = (DecimalFormat)nf;
df.applyPattern(pattern);
String output = df.format(value);
System.out.println(pattern + " " + output + " " + loc.toString());
```

Запуск этого кода даёт следующий вывод. Отформатированное число (во втором столбце) меняется
в зависимости от локали (`Locale`):

```
###,###.###      123,456.789     en_US
###,###.###      123.456,789     de_DE
###,###.###      123 456,789     fr_FR
```

До сих пор рассмотренные шаблоны форматирования следовали соглашениям английского языка (США).
Например, в шаблоне `###,###.##` запятая — это разделитель тысяч, а точка обозначает десятичный
разделитель. Это нормально, пока конечный пользователь с такой записью не сталкивается. Однако
некоторые приложения — например, электронные таблицы и генераторы отчётов — позволяют конечным
пользователям задавать собственные шаблоны форматирования. Для таких приложений шаблоны,
заданные пользователем, должны использовать локализованную нотацию. В этих случаях у объекта
`DecimalFormat` нужно вызывать метод `applyLocalizedPattern`.

#### Изменение символов форматирования

Класс `DecimalFormatSymbols` позволяет изменить символы, появляющиеся в отформатированных числах,
которые возвращает метод `format`. К таким символам относятся, среди прочего, десятичный
разделитель, разделитель групп, знак «минус» и знак процента.

Следующий пример демонстрирует класс `DecimalFormatSymbols`, применяя к числу необычный формат.
Странный формат — результат вызовов методов `setDecimalSeparator`, `setGroupingSeparator`
и `setGroupingSize`.

```java
DecimalFormatSymbols unusualSymbols = new DecimalFormatSymbols(currentLocale);
unusualSymbols.setDecimalSeparator('|');
unusualSymbols.setGroupingSeparator('^');

String strange = "#,##0.###";
DecimalFormat weirdFormatter = new DecimalFormat(strange, unusualSymbols);
weirdFormatter.setGroupingSize(4);

String bizarre = weirdFormatter.format(12345.678);
System.out.println(bizarre);
```

При запуске пример печатает число в причудливом формате:

```
1^2345|678
```

#### Синтаксис шаблонов числовых форматов

Собственные шаблоны форматов для чисел можно проектировать по правилам, заданным следующей
BNF-диаграммой:

```
pattern    := subpattern{;subpattern}
subpattern := {prefix}integer{.fraction}{suffix}
prefix     := ''..'�' - specialCharacters
suffix     := ''..'�' - specialCharacters
integer    := '#'* '0'* '0'
fraction   := '0'* '#'*
```

Нотация, использованная в диаграмме, поясняется в таблице:

| Нотация | Описание |
|---------|----------|
| `X*` | 0 или более экземпляров X |
| `(X \| Y)` | либо X, либо Y |
| `X..Y` | любой символ от X до Y включительно |
| `S - T` | символы из S, кроме входящих в T |
| `{X}` | X необязателен |

В приведённой BNF-диаграмме первый подшаблон (*subpattern*) задаёт формат для положительных
чисел. Второй подшаблон, необязательный, задаёт формат для отрицательных чисел.

Хотя в BNF-диаграмме это не отмечено, запятая может встречаться внутри целой части (*integer*).

Внутри подшаблонов форматирование задаётся специальными символами. Они описаны в следующей
таблице:

| Символ | Описание |
|--------|----------|
| `0` | цифра |
| `#` | цифра; ноль показывается как отсутствующий |
| `.` | заполнитель десятичного разделителя |
| `,` | заполнитель разделителя групп |
| `E` | разделяет мантиссу и экспоненту в экспоненциальных форматах |
| `;` | разделяет форматы |
| `-` | префикс отрицательного числа по умолчанию |
| `%` | умножить на 100 и показать как проценты |
| `‰` (`‰`) | умножить на 1000 и показать как промилле |
| `¤` (`¤`) | знак валюты; заменяется символом валюты; если удвоен — заменяется международным символом валюты; если присутствует в шаблоне, вместо десятичного разделителя используется денежный десятичный разделитель |
| `X` | в префиксе или суффиксе можно использовать любые другие символы |
| `'` | используется для экранирования специальных символов в префиксе или суффиксе |

## Даты и время (Dates and Times)

> **Примечание о версии.** Этот раздел про дату и время использует API даты и времени из пакета
> `java.util`. API `java.time`, доступный начиная с выпуска JDK 8, предоставляет всеобъемлющую
> модель даты и времени, существенно превосходящую классы `java.util`. API `java.time` описан в
> трейле *Date Time*. Особенно полезной может оказаться страница *Legacy Date-Time Code*.

Объекты `Date` представляют даты и время. Нельзя отобразить или напечатать объект `Date`, не
преобразовав его сначала в строку (`String`) надлежащего формата. Что такое «надлежащий» формат?
Во-первых, формат должен соответствовать соглашениям локали (`Locale`) конечного пользователя.
Например, немцы распознают `20.4.09` как корректную дату, тогда как американцы ожидают, что та же
дата будет выглядеть как `4/20/09`. Во-вторых, формат должен включать необходимую информацию.
Например, программа, измеряющая производительность сети, может отчитываться в прошедших
миллисекундах. Онлайн-календарь встреч, вероятно, миллисекунды показывать не будет, но покажет
дни недели.

Этот раздел объясняет, как форматировать даты и время по-разному и с учётом локали. Следуя этим
приёмам, ваши программы будут отображать даты и время в соответствующей локали (`Locale`), при
этом исходный код останется независимым от какой-либо конкретной локали.

Раздел состоит из трёх частей:

- **Использование предопределённых форматов (Using Predefined Formats).** Класс `DateFormat`
  предоставляет предопределённые стили форматирования, специфичные для локали и простые в
  использовании.
- **Настройка форматов (Customizing Formats).** С помощью класса `SimpleDateFormat` можно
  создавать настраиваемые, специфичные для локали форматы.
- **Изменение символов формата даты (Changing Date Format Symbols).** С помощью класса
  `DateFormatSymbols` можно изменить символы, представляющие названия месяцев, дней недели
  и другие элементы форматирования.

### Даты (Dates)

Форматирование дат классом `DateFormat` состоит из двух шагов. Сначала создаётся форматтер
методом `getDateInstance`. Затем вызывается метод `format`, который возвращает строку `String` с
отформатированной датой. Следующий пример форматирует сегодняшнюю дату, вызывая эти два метода:

```java
Date today;
String dateOut;
DateFormat dateFormatter;

dateFormatter = DateFormat.getDateInstance(DateFormat.DEFAULT, currentLocale);
today = new Date();
dateOut = dateFormatter.format(today);

System.out.println(dateOut + " " + currentLocale.toString());
```

Вывод этого кода приведён ниже. Обратите внимание, что форматы дат меняются в зависимости от
локали (`Locale`). Поскольку `DateFormat` учитывает локаль, он берёт на себя детали
форматирования для каждой локали:

```
30 juin 2009     fr_FR
30.06.2009       de_DE
Jun 30, 2009     en_US
```

В предыдущем примере был указан стиль форматирования `DEFAULT`. Стиль `DEFAULT` — лишь один из
предопределённых стилей форматирования, которые предоставляет класс `DateFormat`:

- `DEFAULT`
- `SHORT`
- `MEDIUM`
- `LONG`
- `FULL`

Следующая таблица показывает, как даты форматируются для каждого стиля в локалях США и Франции:

| Стиль | Локаль США | Локаль Франции |
|-------|------------|----------------|
| `DEFAULT` | Jun 30, 2009 | 30 juin 2009 |
| `SHORT` | 6/30/09 | 30/06/09 |
| `MEDIUM` | Jun 30, 2009 | 30 juin 2009 |
| `LONG` | June 30, 2009 | 30 juin 2009 |
| `FULL` | Tuesday, June 30, 2009 | mardi 30 juin 2009 |

### Настройка форматов: SimpleDateFormat

Предыдущий раздел *Использование предопределённых форматов* описал стили форматирования,
предоставляемые классом `DateFormat`. В большинстве случаев этих предопределённых форматов
достаточно. Однако, если требуется создать собственные настраиваемые форматы, можно использовать
класс
[`SimpleDateFormat`](https://docs.oracle.com/javase/8/docs/api/java/text/SimpleDateFormat.html).

Приведённые ниже примеры демонстрируют методы класса `SimpleDateFormat`. Полный исходный код
примеров — в файле `SimpleDateFormatDemo`.

#### О шаблонах

При создании объекта `SimpleDateFormat` задаётся строка-шаблон (`String`). Содержимое шаблона
определяет формат даты и времени. Полное описание синтаксиса шаблона см. в таблицах раздела
«Синтаксис шаблонов формата даты».

Следующий код форматирует дату и время согласно строке-шаблону, переданной в конструктор
`SimpleDateFormat`. Строка `String`, возвращённая методом `format`, содержит отформатированные
дату и время для отображения:

```java
Date today;
String output;
SimpleDateFormat formatter;

formatter = new SimpleDateFormat(pattern, currentLocale);
today = new Date();
output = formatter.format(today);
System.out.println(pattern + " " + output);
```

Следующая таблица показывает вывод предыдущего кода при указании локали США:

| Шаблон | Вывод |
|--------|-------|
| `dd.MM.yy` | 30.06.09 |
| `yyyy.MM.dd G 'at' hh:mm:ss z` | 2009.06.30 AD at 08:29:36 PDT |
| `EEE, MMM d, ''yy` | Tue, Jun 30, '09 |
| `h:mm a` | 8:29 PM |
| `H:mm` | 8:29 |
| `H:mm:ss:SSS` | 8:28:36:249 |
| `K:mm a,z` | 8:29 AM,PDT |
| `yyyy.MMMMM.dd GGG hh:mm aaa` | 2009.June.30 AD 08:29 AM |

#### Шаблоны и локаль

Класс `SimpleDateFormat` учитывает локаль. Если создать `SimpleDateFormat` без параметра `Locale`,
он отформатирует дату и время согласно локали по умолчанию. И шаблон, и локаль определяют формат.
Для одного и того же шаблона `SimpleDateFormat` может форматировать дату и время по-разному, если
локаль различается.

В примере ниже шаблон жёстко зашит в операторе, создающем объект `SimpleDateFormat`:

```java
Date today;
String result;
SimpleDateFormat formatter;

formatter = new SimpleDateFormat("EEE d MMM yy", currentLocale);
today = new Date();
result = formatter.format(today);
System.out.println("Locale: " + currentLocale.toString());
System.out.println("Result: " + result);
```

При установке `currentLocale` в разные значения предыдущий пример генерирует такой вывод:

```
Locale: fr_FR
Result: mar. 30 juin 09
Locale: de_DE
Result: Di 30 Jun 09
Locale: en_US
Result: Tue 30 Jun 09
```

#### Синтаксис шаблонов формата даты

Собственные шаблоны форматов для дат и времени можно проектировать из списка символов в следующей
таблице. (Символ `z` представляет часовой пояс (*time zone*); число повторений буквы влияет на
форму вывода — см. вторую таблицу ниже.)

| Символ | Значение | Представление | Пример |
|--------|----------|---------------|--------|
| `G` | обозначение эры (*era designator*) | Текст | AD |
| `y` | год | Число | 2009 |
| `M` | месяц в году | Текст и число | July и 07 |
| `d` | день в месяце | Число | 10 |
| `h` | час в формате am/pm (1–12) | Число | 12 |
| `H` | час в сутках (0–23) | Число | 0 |
| `m` | минута в часе | Число | 30 |
| `s` | секунда в минуте | Число | 55 |
| `S` | миллисекунда | Число | 978 |
| `E` | день недели | Текст | Tuesday |
| `D` | день в году | Число | 189 |
| `F` | день недели в месяце | Число | 2 (2-я среда в июле) |
| `w` | неделя в году | Число | 27 |
| `W` | неделя в месяце | Число | 2 |
| `a` | маркер am/pm | Текст | PM |
| `k` | час в сутках (1–24) | Число | 24 |
| `K` | час в формате am/pm (0–11) | Число | 0 |
| `z` | часовой пояс (*time zone*) | Текст | Pacific Standard Time |
| `'` | экранирование текста | Разделитель | (нет) |
| `''` | одинарная кавычка | Литерал | `'` |

Символы, не являющиеся буквами, рассматриваются как заключённый в кавычки текст. То есть они
появятся в отформатированном тексте, даже если не заключены в одинарные кавычки.

Количество указанных символов-букв также определяет формат. Например, если шаблон `zz` даёт «PDT»,
то шаблон `zzzz` генерирует «Pacific Daylight Time». Следующая таблица обобщает эти правила:

| Представление | Число символов | Результат |
|---------------|----------------|-----------|
| Текст | 1–3 | сокращённая форма, если она существует |
| Текст | ≥ 4 | полная форма |
| Число | (любое) | требуется минимальное число цифр; более короткие числа дополняются нулями (для года: если количество `y` равно 2, год усекается до 2 цифр) |
| Текст и число | 1–2 | числовая форма |
| Текст и число | 3 | текстовая форма |

> **О часовых поясах.** В трейле *Internationalization* часовые пояса (*time zones*) рассматриваются
> в рамках символа `z` шаблонов `SimpleDateFormat` (см. таблицы выше): чем больше повторений буквы
> `z`, тем полнее форма названия пояса (`zz` → «PDT», `zzzz` → «Pacific Daylight Time»).

### Изменение символов формата даты

Метод `format` класса `SimpleDateFormat` возвращает строку `String`, составленную из цифр и
символов. Например, в строке «Friday, April 10, 2009» символами являются «Friday» и «April». Если
символы, инкапсулированные в `SimpleDateFormat`, не отвечают вашим потребностям, их можно изменить
с помощью класса
[`DateFormatSymbols`](https://docs.oracle.com/javase/8/docs/api/java/text/DateFormatSymbols.html).
Можно изменить символы, представляющие названия месяцев, дней недели, часовых поясов и другие.
Следующая таблица перечисляет методы `DateFormatSymbols`, позволяющие изменять символы:

Методы `DateFormatSymbols`:

| Метод-сеттер | Пример символа, который изменяет метод |
|--------------|----------------------------------------|
| `setAmPmStrings` | PM |
| `setEras` | AD |
| `setMonths` | December |
| `setShortMonths` | Dec |
| `setShortWeekdays` | Tue |
| `setWeekdays` | Tuesday |
| `setZoneStrings` | PST |

Следующий пример вызывает `setShortWeekdays`, чтобы изменить короткие названия дней недели со
строчных на прописные буквы. Полный исходный код примера — в `DateFormatSymbolsDemo`. Первый
элемент массива-аргумента `setShortWeekdays` — это пустая строка (`null String`). Поэтому массив
начинается с единицы, а не с нуля. Конструктор `SimpleDateFormat` принимает изменённый объект
`DateFormatSymbols` в качестве аргумента. Вот исходный код:

```java
Date today;
String result;
SimpleDateFormat formatter;
DateFormatSymbols symbols;
String[] defaultDays;
String[] modifiedDays;

symbols = new DateFormatSymbols( new Locale("en", "US"));
defaultDays = symbols.getShortWeekdays();

for (int i = 0; i < defaultDays.length; i++) {
    System.out.print(defaultDays[i] + " ");
}
System.out.println();

String[] capitalDays = {
    "", "SUN", "MON",
    "TUE", "WED", "THU",
    "FRI", "SAT"
};
symbols.setShortWeekdays(capitalDays);

modifiedDays = symbols.getShortWeekdays();
for (int i = 0; i < modifiedDays.length; i++) {
    System.out.print(modifiedDays[i] + " ");
}
System.out.println();
System.out.println();

formatter = new SimpleDateFormat("E", symbols);
today = new Date();
result = formatter.format(today);
System.out.println("Today's day of the week: " + result);
```

Предыдущий код генерирует такой вывод:

```
 Sun Mon Tue Wed Thu Fri Sat 
 SUN MON TUE WED THU FRI SAT 

Today's day of the week: MON
```

## Сообщения (Messages)

Всем нам нравится пользоваться программами, которые сообщают, что происходит. Программы,
держащие нас в курсе, часто делают это, отображая сообщения о статусе и об ошибках. Конечно, эти
сообщения нужно переводить, чтобы их понимали пользователи по всему миру. Раздел *Isolating
Locale-Specific Data* обсуждает переводимые текстовые сообщения. Обычно, переместив строку
сообщения в `ResourceBundle`, вы закончили. Однако, если в сообщение встроены переменные данные,
придётся предпринять несколько дополнительных шагов, чтобы подготовить его к переводу.

**Составное сообщение (*compound message*)** содержит переменные данные. В следующем списке
составных сообщений переменные данные подчёркнуты (в оригинале):

- The disk named MyDisk contains 300 files.
- The current balance of account #34-09-222 is $2,745.72.
- 405,390 people have visited your website since January 1, 2009.
- Delete all files older than 120 days.

Может возникнуть искушение построить последнее сообщение из списка конкатенацией фраз и
переменных:

```java
double numDays;
ResourceBundle msgBundle;
// ...
String message = msgBundle.getString(
                     "deleteolder" +
                     numDays.toString() +
                     msgBundle.getString("days"));
```

Этот подход прекрасно работает в английском, но не сработает для языков, в которых глагол стоит в
конце предложения. Поскольку порядок слов этого сообщения жёстко зашит, ваши локализаторы не
смогут создать грамматически правильные переводы для всех языков.

Как сделать программу локализуемой, если нужны составные сообщения? Это можно сделать с помощью
класса `MessageFormat`, которому посвящён этот раздел.

> **Осторожно.** Составные сообщения трудно переводить, потому что текст сообщения фрагментирован.
> Если вы используете составные сообщения, локализация займёт больше времени и обойдётся дороже.
> Поэтому используйте составные сообщения только при необходимости.

Раздел состоит из двух частей:

- **Работа с составными сообщениями (Dealing with Compound Messages).** Составное сообщение может
  содержать переменные нескольких видов: даты, время, строки, числа, валюты и проценты. Чтобы
  отформатировать составное сообщение независимо от локали, строится шаблон, применяемый к объекту
  `MessageFormat`.
- **Обработка множественного числа (Handling Plurals).** Слова в сообщении обычно меняются, если
  возможны и единственная, и множественная формы. С помощью класса `ChoiceFormat` можно сопоставить
  число со словом или фразой, что позволяет строить грамматически правильные сообщения.

### Работа с составными сообщениями: MessageFormat

Составное сообщение может содержать переменные нескольких видов: даты, время, строки, числа,
валюты и проценты. Чтобы отформатировать составное сообщение независимым от локали способом, вы
строите шаблон, применяете его к объекту `MessageFormat` и сохраняете этот шаблон в `ResourceBundle`.

Шаг за шагом разбирая пример программы, этот раздел показывает, как интернационализировать
составное сообщение. Пример использует класс
[`MessageFormat`](https://docs.oracle.com/javase/8/docs/api/java/text/MessageFormat.html). Полный
исходный код — в файле `MessageFormatDemo.java`. Свойства немецкой локали — в файле
`MessageBundle_de_DE.properties`.

#### 1. Определите переменные в сообщении

Предположим, нужно интернационализировать следующее сообщение:

> At 1:15 on April 13, 1998, we detected 7 spaceships on the planet Mars.

Заметьте, что переменные данные подчёркнуты (в оригинале) и определено, какие объекты будут эти
данные представлять.

#### 2. Изолируйте шаблон сообщения в ResourceBundle

Сохраните сообщение в `ResourceBundle` с именем `MessageBundle`:

```java
ResourceBundle messages =
   ResourceBundle.getBundle("MessageBundle", currentLocale);
```

Этот `ResourceBundle` опирается на файл свойств для каждой локали (`Locale`). Поскольку
`ResourceBundle` называется `MessageBundle`, файл свойств для английского (США) называется
`MessageBundle_en_US.properties`. Содержимое этого файла:

```
template = At {2,time,short} on {2,date,long}, \
    we detected {1,number,integer} spaceships on \
    the planet {0}.
planet = Mars
```

Первая строка файла свойств содержит шаблон сообщения. Сравнив этот шаблон с текстом сообщения из
шага 1, вы увидите, что каждый переменный элемент текста заменён аргументом в фигурных скобках.
Каждый аргумент начинается с цифры — **номера аргумента**, — соответствующей индексу элемента в
массиве `Object`, который хранит значения аргументов. Заметьте, что в шаблоне номера аргументов
идут не в каком-то особом порядке. Аргументы можно размещать в шаблоне где угодно. Единственное
требование — чтобы номеру аргумента соответствовал элемент в массиве значений аргументов.

Следующий шаг обсуждает массив значений аргументов, но сначала рассмотрим каждый из аргументов
шаблона. Следующая таблица даёт некоторые подробности об аргументах:

Аргументы шаблона `template` в `MessageBundle_en_US.properties`:

| Аргумент | Описание |
|----------|----------|
| `{2,time,short}` | Часть времени объекта `Date`. Стиль `short` задаёт стиль форматирования `DateFormat.SHORT`. |
| `{2,date,long}` | Часть даты объекта `Date`. Тот же объект `Date` используется и для переменной даты, и для переменной времени. В массиве `Object` аргументов индекс элемента, содержащего объект `Date`, равен 2. (Описано на следующем шаге.) |
| `{1,number,integer}` | Объект `Number`, дополнительно уточнённый числовым стилем `integer`. |
| `{0}` | Строка `String` из `ResourceBundle`, соответствующая ключу `planet`. |

Полное описание синтаксиса аргументов см. в документации API класса `MessageFormat`.

#### 3. Задайте аргументы сообщения

Следующие строки кода присваивают значения каждому аргументу шаблона. Индексы элементов массива
`messageArguments` соответствуют номерам аргументов в шаблоне. Например, элемент `Integer` с
индексом 1 соответствует аргументу `{1,number,integer}` в шаблоне. Поскольку её необходимо
перевести, строка `String` в элементе 0 будет извлечена из `ResourceBundle` методом `getString`.
Вот код, определяющий массив аргументов сообщения:

```java
Object[] messageArguments = {
    messages.getString("planet"),
    new Integer(7),
    new Date()
};
```

#### 4. Создайте форматтер

Далее создайте объект `MessageFormat`. Локаль (`Locale`) задаётся, потому что сообщение содержит
объекты `Date` и `Number`, которые должны форматироваться с учётом локали.

```java
MessageFormat formatter = new MessageFormat("");
formatter.setLocale(currentLocale);
```

#### 5. Отформатируйте сообщение с помощью шаблона и аргументов

Этот шаг показывает, как шаблон, аргументы сообщения и форматтер работают вместе. Сначала извлеките
строку-шаблон из `ResourceBundle` методом `getString`. Ключ к шаблону — `template`. Передайте
строку-шаблон форматтеру методом `applyPattern`. Затем отформатируйте сообщение, используя массив
аргументов, вызвав метод `format`. Строка `String`, возвращённая методом `format`, готова к
отображению. Всё это достигается всего двумя строками кода:

```java
formatter.applyPattern(messages.getString("template"));
String output = formatter.format(messageArguments);
```

#### 6. Запустите демонстрационную программу

Демонстрационная программа печатает переведённые сообщения для английской и немецкой локалей и
правильно форматирует переменные даты и времени. Заметьте, что английский и немецкий глаголы
(«detected» и «entdeckt») находятся в разных местах относительно переменных:

```
currentLocale = en_US
At 10:16 AM on July 31, 2009, we detected 7
spaceships on the planet Mars.
currentLocale = de_DE
Um 10:16 am 31. Juli 2009 haben wir 7 Raumschiffe
auf dem Planeten Mars entdeckt.
```

### Обработка множественного числа: ChoiceFormat

Слова в сообщении могут меняться, если возможны и множественная, и единственная формы. С помощью
класса `ChoiceFormat` можно сопоставить число со словом или фразой, что позволяет строить
грамматически правильные сообщения.

В английском языке множественная и единственная формы слова обычно различаются. Это может стать
проблемой при построении сообщений, ссылающихся на количества. Например, если ваше сообщение
сообщает число файлов на диске, возможны такие варианты:

```
There are no files on XDisk.
There is one file on XDisk.
There are 2 files on XDisk.
```

Самый быстрый способ решить эту проблему — создать шаблон `MessageFormat` вроде такого:

```
There are {0,number} file(s) on {1}.
```

К сожалению, предыдущий шаблон приводит к неправильной грамматике:

```
There are 1 file(s) on XDisk.
```

Можно сделать лучше, если использовать класс
[`ChoiceFormat`](https://docs.oracle.com/javase/8/docs/api/java/text/ChoiceFormat.html). В этом
разделе вы научитесь работать с множественным числом в сообщении, шаг за шагом разбирая пример
программы `ChoiceFormatDemo`. Эта программа также использует класс `MessageFormat`, рассмотренный
в предыдущем разделе *Работа с составными сообщениями*.

#### 1. Определите шаблон сообщения

Сначала определите переменные в сообщении (в оригинале переменные в каждой из трёх строк
выделены). Затем замените переменные в сообщении аргументами, создав шаблон, который можно
применить к объекту `MessageFormat`:

```
There {0} on {1}.
```

Аргумент для имени диска, представленный `{1}`, обрабатывается достаточно просто. С ним
обращаются как с любой другой строковой переменной (`String`) в шаблоне `MessageFormat`. Этот
аргумент соответствует элементу с индексом 1 в массиве значений аргументов. (См. шаг 7.)

Работа с аргументом `{0}` сложнее по нескольким причинам:

- Фраза, которую заменяет этот аргумент, меняется в зависимости от числа файлов. Чтобы построить
  эту фразу во время выполнения, нужно сопоставить число файлов с конкретной строкой `String`.
  Например, число 1 сопоставится со строкой, содержащей фразу `is one file`. Класс `ChoiceFormat`
  позволяет выполнить необходимое сопоставление.
- Если диск содержит несколько файлов, фраза включает целое число. Класс `MessageFormat`
  позволяет вставить число во фразу.

#### 2. Создайте ResourceBundle

Поскольку текст сообщения должен быть переведён, изолируйте его в `ResourceBundle`:

```java
ResourceBundle bundle = ResourceBundle.getBundle(
    "ChoiceBundle", currentLocale);
```

Пример опирается на файлы свойств. Файл `ChoiceBundle_en_US.properties` содержит следующие строки:

```
pattern = There {0} on {1}.
noFiles = are no files
oneFile = is one file
multipleFiles = are {2} files
```

Содержимое этого файла свойств показывает, как сообщение будет построено и отформатировано. Первая
строка содержит шаблон для `MessageFormat`. (См. шаг 1.) Остальные строки содержат фразы, которые
заменят аргумент `{0}` в шаблоне. Фраза для ключа `multipleFiles` содержит аргумент `{2}`, который
будет заменён числом.

Вот французская версия файла свойств, `ChoiceBundle_fr_FR.properties`:

```
pattern = Il {0} sur {1}.
noFiles = n'y a pas de fichiers
oneFile = y a un fichier
multipleFiles = y a {2} fichiers
```

#### 3. Создайте форматтер сообщений

На этом шаге создаётся `MessageFormat` и устанавливается его локаль:

```java
MessageFormat messageForm = new MessageFormat("");
messageForm.setLocale(currentLocale);
```

#### 4. Создайте форматтер выбора

Объект `ChoiceFormat` позволяет выбрать конкретную строку `String` на основе числа `double`.
Диапазон чисел `double` и строки `String`, с которыми они сопоставляются, задаются массивами:

```java
double[] fileLimits = {0,1,2};
String [] fileStrings = {
    bundle.getString("noFiles"),
    bundle.getString("oneFile"),
    bundle.getString("multipleFiles")
};
```

`ChoiceFormat` сопоставляет каждый элемент массива `double` с элементом массива `String`, имеющим
тот же индекс. В примере кода 0 сопоставляется со строкой, возвращённой вызовом
`bundle.getString("noFiles")`. По совпадению индекс совпадает со значением в массиве `fileLimits`.
Если бы код установил `fileLimits[0]` равным семи, `ChoiceFormat` сопоставил бы число 7 с
`fileStrings[0]`.

Массивы `double` и `String` указываются при создании `ChoiceFormat`:

```java
ChoiceFormat choiceForm = new ChoiceFormat(fileLimits, fileStrings);
```

#### 5. Примените шаблон

Помните шаблон, построенный на шаге 1? Пора извлечь его из `ResourceBundle` и применить к объекту
`MessageFormat`:

```java
String pattern = bundle.getString("pattern");
messageForm.applyPattern(pattern);
```

#### 6. Назначьте форматы

На этом шаге объекту `MessageFormat` назначается объект `ChoiceFormat`, созданный на шаге 4:

```java
Format[] formats = {choiceForm, null, NumberFormat.getInstance()};
messageForm.setFormats(formats);
```

Метод `setFormats` назначает объекты `Format` аргументам в шаблоне сообщения. Метод `applyPattern`
нужно вызвать до вызова `setFormats`. Следующая таблица показывает, как элементы массива `Format`
соответствуют аргументам в шаблоне сообщения:

Массив `Format` программы `ChoiceFormatDemo`:

| Элемент массива | Аргумент шаблона |
|-----------------|------------------|
| `choiceForm` | `{0}` |
| `null` | `{1}` |
| `NumberFormat.getInstance()` | `{2}` |

#### 7. Задайте аргументы и отформатируйте сообщение

Во время выполнения программа присваивает переменные массиву аргументов, который передаёт объекту
`MessageFormat`. Элементы массива соответствуют аргументам в шаблоне. Например, `messageArgument[1]`
сопоставляется аргументу шаблона `{1}` — строке `String` с именем диска. На предыдущем шаге
программа назначила объект `ChoiceFormat` аргументу `{0}` шаблона. Поэтому число, присвоенное
`messageArgument[0]`, определяет, какую строку `String` выберет объект `ChoiceFormat`. Если
`messageArgument[0]` больше либо равно 2, аргумент `{0}` в шаблоне заменяется строкой с фразой
`are {2} files`. Число, присвоенное `messageArgument[2]`, будет подставлено вместо аргумента
шаблона `{2}`. Вот код, который это проверяет:

```java
Object[] messageArguments = {null, "XDisk", null};

for (int numFiles = 0; numFiles < 4; numFiles++) {
    messageArguments[0] = new Integer(numFiles);
    messageArguments[2] = new Integer(numFiles);
    String result = messageForm.format(messageArguments);
    System.out.println(result);
}
```

#### 8. Запустите демонстрационную программу

Сравните сообщения, отображаемые программой, с фразами в `ResourceBundle` из шага 2. Заметьте, что
объект `ChoiceFormat` выбирает правильную фразу, которую объект `MessageFormat` использует для
построения нужного сообщения. Вывод программы `ChoiceFormatDemo`:

```

There is one file on XDisk.
There are 2 files on XDisk.
There are 3 files on XDisk.

currentLocale = fr_FR

Il y a un fichier sur XDisk.
Il y a 2 fichiers sur XDisk.
Il y a 3 fichiers sur XDisk.
```
