# Урок 1. Быстрый старт

**Трейл:** JavaBeans · **Оригинал:** [Quick Start](https://docs.oracle.com/javase/tutorial/javabeans/quick/index.html)
**Связанные области:** [[01-core-java-syntax-oop]] · **Вопросы:** core-java

> Перевод официального руководства Oracle (The Java Tutorials, JDK 8). Объединяет страницы
> *Quick Start* (индекс трейла), *Creating a Project*, *A Button is a Bean*,
> *Wiring the Application* и *Using a Third-Party Bean*.
>
> Руководство Java Tutorials написано для JDK 8. Примеры и приёмы, описанные на этих

Этот урок показывает, как с помощью NetBeans собрать простое приложение. Хороший инструмент
вроде NetBeans позволяет собирать приложение из компонентов JavaBeans, не написав ни строчки кода.

Первые три раздела урока показывают, как создать простое приложение из графических бинов
(*beans*), входящих в состав платформы Java. Последний раздел демонстрирует, насколько легко
встроить в приложение сторонний бин.

## Создание проекта (Creating a Project)

Проще всего изучить JavaBeans, начав ими пользоваться. Для начала
[скачайте и установите последнюю версию NetBeans](http://netbeans.org/). В этом руководстве
описана работа с NetBeans версии 7.0.

NetBeans — это **инструмент сборки бинов** (*bean builder tool*): он распознаёт компоненты
JavaBeans (бины) и позволяет с лёгкостью соединять компоненты в приложение.

Запустите NetBeans. Выберите в меню **File > New Project...** (Файл > Новый проект).

> _Снимок экрана (диалог «New Project»):_ выберите **Java** в списке **Categories** (Категории)
> и **Java Application** в списке **Projects** (Проекты). Нажмите **Next >** (Далее).

> _Снимок экрана (ввод имени проекта):_ введите **SnapApp** в качестве имени приложения.
> Снимите флажок **Create Main Class** (Создать главный класс) и нажмите **Finish** (Готово).

NetBeans создаёт новый проект, и вы видите его в панели **Projects** (Проекты).

> _Снимок экрана:_ проект **SnapApp** в панели **Projects**.

Щёлкните по проекту **SnapApp** с зажатой клавишей Control и выберите во всплывающем меню
**New > JFrame Form...** (Создать > Форма JFrame).

> _Снимок экрана:_ создание новой формы — контекстное меню проекта с пунктом **New > JFrame Form...**.

Введите **SnapFrame** в качестве имени класса и **snapapp** в качестве пакета. Нажмите
**Finish** (Готово). NetBeans создаёт новый класс и показывает его визуальный конструктор:

> _Снимок экрана (визуальный конструктор NetBeans):_ слева в панели **Projects** виден только
> что созданный класс `SnapFrame`. В центре экрана — визуальный конструктор (*visual designer*)
> NetBeans. Справа — **Palette** (Палитра), которая содержит все компоненты, доступные для
> добавления на форму в визуальном конструкторе.

## Кнопка — это бин (A Button is a Bean)

Присмотритесь к **Palette** (Палитре). Все перечисленные в ней компоненты — это бины.
Компоненты сгруппированы по назначению. Прокрутите до группы **Swing Controls** (Элементы
управления Swing), затем щёлкните по **Button** (Кнопка) и перетащите её в визуальный
конструктор. Кнопка — это бин!

Под палитрой, в правой части NetBeans, расположена панель инспектора (*inspector pane*), с
помощью которой можно изучать кнопку и управлять ею. Чтобы освободить место для панели
инспектора, попробуйте закрыть окно вывода (*output window*) внизу экрана.

> _Снимок экрана:_ добавление кнопки на форму `SnapFrame` — компонент кнопки перетаскивается
> в визуальный конструктор.

### Свойства (Properties)

**Свойства** (*properties*) бина — это то, что вы можете изменять и что влияет на его внешний
вид или внутреннее состояние. Для кнопки из этого примера свойствами являются, например, цвет
переднего плана (*foreground color*), шрифт (*font*) и текст, отображаемый на кнопке. Свойства
показаны двумя группами. **Properties** (Свойства) перечисляет наиболее часто используемые
свойства, а **Other Properties** (Прочие свойства) — менее распространённые.

Попробуйте отредактировать свойства кнопки. Для одних свойств значения можно вводить прямо в
таблицу. Для других нужно нажать кнопку **...**, чтобы отредактировать значение. Например,
нажмите **...** справа от свойства **foreground** (передний план). Всплывает диалог выбора
цвета — и вы можете выбрать новый цвет текста на кнопке. Попробуйте другие свойства и
посмотрите, что получится. Обратите внимание: вы не пишете никакого кода.

> _Снимок экрана:_ панель свойств кнопки (**Button properties**) со списком свойств в инспекторе.

### События (Events)

Бины могут также **порождать события** (*fire events*). Нажмите кнопку **Events** (События) в
панели свойств бина. Вы увидите список всех событий, которые способна порождать кнопка.

> _Снимок экрана:_ список событий кнопки (**Button events**).

С помощью NetBeans можно связывать бины через их события и свойства. Чтобы увидеть, как это
работает, перетащите **Label** (Метку) из палитры в визуальный конструктор формы `SnapFrame`.

Отредактируйте свойства метки так, чтобы она выглядела как нужно.

> _Снимок экрана:_ добавление метки на форму в визуальный конструктор.

## Связывание приложения (Wiring the Application)

Чтобы связать кнопку и метку, нажмите кнопку **Connection Mode** (Режим связывания) на панели
инструментов визуального конструктора.

> _Снимок экрана:_ кнопка **Connection Mode** на панели инструментов визуального конструктора.

Щёлкните по кнопке на форме `SnapFrame`. NetBeans обводит кнопку красным, показывая, что именно
этот компонент будет порождать событие.

> _Снимок экрана:_ NetBeans обводит компонент-источник события красной рамкой.

Щёлкните по метке. Всплывает **Connection Wizard** (Мастер связывания) NetBeans. Сначала вы
выбираете событие, на которое хотите отреагировать. Для кнопки это событие **action**
(действие). Нажмите **+** рядом с **action** и выберите **actionPerformed**. Нажмите
**Next >** (Далее).

> _Снимок экрана:_ выбор события-источника — экран мастера связывания с событием **action** и
> **actionPerformed**.

Теперь нужно выбрать, что произойдёт, когда кнопка породит своё событие **action**. **Connection
Wizard** перечисляет все свойства бина-метки. Выберите в списке **text** (текст) и нажмите
**Next** (Далее).

> _Снимок экрана:_ выбор свойства-приёмника — выбор свойства **text**.

На последнем экране **Connection Wizard** введите значение, которое хотите установить для
свойства **text**. Нажмите **Value** (Значение), затем введите **You pressed the button!**
(«Вы нажали кнопку!») или что-нибудь столь же красноречивое. Нажмите **Finish** (Готово).

> _Снимок экрана:_ ввод значения — последний экран мастера связывания с текстом
> «You pressed the button!».

NetBeans связывает компоненты и показывает результат своей работы в редакторе исходного кода.

> _Снимок экрана:_ код связывания в редакторе исходного кода.

Нажмите кнопку **Design** (Конструктор) на панели инструментов редактора кода, чтобы вернуться к
конструктору интерфейса. Нажмите **Run Main Project** (Запустить главный проект) или клавишу
**F6**, чтобы собрать и запустить проект.

> _Снимок экрана:_ кнопка **Run Main Project** на панели инструментов.

NetBeans собирает и запускает проект. Он просит указать главный класс — это `SnapFrame`. Когда
появится окно приложения, нажмите на кнопку. Вы увидите свою бессмертную фразу в метке.

> _Снимок экрана:_ «Смотрите, мама, без кода!» — запущенное приложение с кнопкой и меткой.

Обратите внимание: вы не написали ни строчки кода. В этом и состоит настоящая мощь JavaBeans —
с хорошим инструментом сборки вроде NetBeans вы можете быстро связать компоненты и получить
работающее приложение.

## Использование стороннего бина (Using a Third-Party Bean)

Почти любой код можно упаковать как бин. Все бины, которые вы видели до сих пор, были
визуальными, но бин может предоставлять функциональность и без видимого компонента.

Сила JavaBeans в том, что вы можете использовать программные компоненты, не написав их и не
вникая в их реализацию.

Этот раздел описывает, как добавить JavaBean в своё приложение и воспользоваться его
функциональностью.

### Добавление бина в палитру NetBeans

Скачайте пример компонента JavaBean — `BumperSticker`. Бины распространяются в виде JAR-файлов.
Сохраните файл где-нибудь на компьютере. `BumperSticker` — это графический компонент, который
предоставляет один метод, `go()`, запускающий анимацию.

Чтобы добавить `BumperSticker` в палитру NetBeans, выберите в меню NetBeans
**Tools > Palette > Swing/AWT Components** (Инструменты > Палитра > Компоненты Swing/AWT).

> _Снимок экрана:_ менеджер палитры NetBeans (**Palette Manager**).

Нажмите кнопку **Add from JAR...** (Добавить из JAR...). NetBeans просит указать JAR-файл,
содержащий бины, которые вы хотите добавить в палитру. Найдите только что скачанный файл и
нажмите **Next** (Далее).

> _Снимок экрана:_ поиск JAR-файла — диалог выбора файла.

NetBeans показывает список классов из JAR-файла. Выберите те, которые хотите добавить в палитру.
В данном случае выберите **BumperSticker** и нажмите **Next** (Далее).

> _Снимок экрана:_ выбор бинов для добавления в палитру — список доступных классов бинов.

Наконец, NetBeans нужно знать, в какой раздел палитры поместить новые бины. Выберите **Beans**
(Бины) и нажмите **Finish** (Готово).

> _Снимок экрана:_ выбор раздела палитры.

Нажмите **Close** (Закрыть), чтобы закрыть окно **Palette Manager**. Теперь загляните в палитру.
`BumperSticker` находится в разделе **Beans**.

### Использование вашего нового JavaBean

Перетащите `BumperSticker` из палитры на свою форму.

> _Снимок экрана:_ «We all love the Java Tutorial» — компонент `BumperSticker` с изображением
> сердца на форме.

С экземпляром `BumperSticker` можно работать так же, как с любым другим бином. Чтобы убедиться в
этом, перетащите на форму ещё одну кнопку. Эта кнопка будет запускать анимацию `BumperSticker`.

> _Снимок экрана:_ добавление ещё одной кнопки на форму с компонентом `BumperSticker`.

Свяжите кнопку с бином `BumperSticker` так же, как вы уже связали первую кнопку с текстовым полем.

1. Начните с нажатия кнопки **Connection Mode** (Режим связывания).
2. Щёлкните по второй кнопке. NetBeans обведёт её красной рамкой.
3. Щёлкните по компоненту `BumperSticker`. Всплывает **Connection Wizard** (Мастер связывания).
4. Нажмите **+** рядом с **action** и выберите **actionPerformed**. Нажмите **Next >** (Далее).
5. Выберите **Method Call** (Вызов метода), затем выберите **go()** из списка. Нажмите
   **Finish** (Готово).

Если какой-то из шагов вызывает затруднения, перечитайте раздел «Связывание приложения
(Wiring the Application)». Процесс здесь очень похож.

Запустите приложение снова. Когда вы нажмёте на вторую кнопку, компонент `BumperSticker`
анимирует цвет сердца.

И снова обратите внимание, как вы получили работающее приложение, не написав никакого кода.

> _Снимок экрана:_ собранное из компонентов приложение — итоговая форма с компонентами
> `BumperSticker` и кнопкой.
