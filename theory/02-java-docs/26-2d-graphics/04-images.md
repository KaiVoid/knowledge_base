# Урок 4. Работа с изображениями

**Трейл:** 2D Graphics · **Оригинал:** [Working with Images](https://docs.oracle.com/javase/tutorial/2d/images/index.html)
**Связанные области:** [[01-core-java-syntax-oop]] · **Вопросы:** core-java

> Перевод официального руководства Oracle (The Java Tutorials, JDK 8). Объединяет
> страницы *Working with Images* (индекс), *Reading/Loading an Image*, *Drawing an Image*
> (включая раздел *Filtering Images*), *Creating and Drawing to an Image* и
> *Writing/Saving an Image*.

> Как вы уже узнали из урока *Images*, изображения (*image*) описываются шириной и
> высотой, измеряемыми в пикселях, и имеют собственную систему координат, не зависящую
> от поверхности рисования.

При работе с изображениями возникает ряд типичных задач:

- Загрузка внешнего файла изображения в формате GIF, PNG или JPEG во внутреннее
  представление изображения, используемое Java 2D.
- Непосредственное создание изображения Java 2D и рисование в нём.
- Отрисовка содержимого изображения Java 2D на поверхности рисования.
- Сохранение содержимого изображения Java 2D во внешний файл формата GIF, PNG или JPEG.

Этот урок обучает основам загрузки, отображения и сохранения изображений.

Чтобы работать с изображениями, нужно изучить два основных класса:

- Класс [`java.awt.Image`](https://docs.oracle.com/javase/8/docs/api/java/awt/Image.html) —
  суперкласс, представляющий графические изображения как прямоугольные массивы пикселей.
- Класс [`java.awt.image.BufferedImage`](https://docs.oracle.com/javase/8/docs/api/java/awt/image/BufferedImage.html),
  который расширяет класс `Image` и позволяет приложению работать с данными изображения
  напрямую (например, получать или устанавливать цвет пикселя). Приложения могут создавать
  экземпляры этого класса непосредственно.

Класс `BufferedImage` — краеугольный камень API изображений Java 2D, работающего в
непосредственном режиме (*immediate-mode*). Он управляет изображением в памяти и
предоставляет методы для хранения, интерпретации и получения данных о пикселях.
Поскольку `BufferedImage` — подкласс `Image`, его можно отрисовывать методами `Graphics`
и `Graphics2D`, принимающими параметр типа `Image`.

`BufferedImage` — это, по сути, `Image` с доступным буфером данных. Поэтому работать
напрямую с `BufferedImage` эффективнее. У `BufferedImage` есть *цветовая модель*
(*ColorModel*) и *растр* (*Raster*) данных изображения. Цветовая модель задаёт цветовую
интерпретацию данных о пикселях.

Растр (`Raster`) выполняет следующие функции:

- представляет прямоугольные координаты изображения;
- хранит данные изображения в памяти;
- предоставляет механизм создания нескольких подизображений из одного буфера данных
  изображения;
- предоставляет методы доступа к конкретным пикселям внутри изображения.

## Чтение и загрузка изображения

Когда речь заходит о цифровых изображениях, обычно вспоминают о сэмплированных форматах,
таких как JPEG, применяемый в цифровой фотографии, или GIF, распространённый на
веб-страницах. Все программы, которые работают с такими изображениями, должны сначала
преобразовать их из внешнего формата во внутренний.

Java 2D поддерживает загрузку этих внешних форматов изображений в свой формат
`BufferedImage` с помощью API ввода-вывода изображений (*Image I/O API*), находящегося
в пакете `javax.imageio`. Image I/O имеет встроенную поддержку форматов GIF, PNG, JPEG,
BMP и WBMP. Image I/O также расширяем: разработчики или администраторы могут «подключить»
(*plug-in*) поддержку дополнительных форматов. Например, отдельно доступны плагины для
TIFF и JPEG 2000.

Чтобы загрузить изображение из конкретного файла, используйте следующий код (из
`LoadImageApp.java`):

```java
BufferedImage img = null;
try {
    img = ImageIO.read(new File("strawberry.jpg"));
} catch (IOException e) {
}
```

Image I/O распознаёт содержимое файла как изображение в формате JPEG и декодирует его в
`BufferedImage`, который можно непосредственно использовать в Java 2D.

`LoadImageApp.java` показывает, как отобразить это изображение.

Если код выполняется в апплете, получить изображение из базового каталога апплета
(*codebase*) так же просто. Следующий фрагмент взят из `LoadImageApplet.java`:

```java
try {
    URL url = new URL(getCodeBase(), "examples/strawberry.jpg");
    img = ImageIO.read(url);
} catch (IOException e) {
}
```

Метод `getCodeBase`, использованный в этом примере, возвращает URL каталога, содержащего
данный апплет, когда апплет развёрнут на веб-сервере. Если апплет развёрнут локально,
`getCodeBase` возвращает `null`, и апплет не запустится.

Приведённый пример показывает, как с помощью метода `getCodeBase` загрузить файл
`strawberry.jpg`.

> **Примечание.** Если апплет не запускается, необходимо установить как минимум Java SE
> Development Kit (JDK) 7.

`LoadImageApplet.java` содержит полный код этого примера; апплету требуется файл
изображения `strawberry.jpg`.

Помимо чтения из файлов или URL, Image I/O может читать из других источников, например
из `InputStream`. `ImageIO.read()` — самый простой удобный API для большинства приложений,
но класс `javax.imageio.ImageIO` предоставляет множество других статических методов для
более продвинутого использования Image I/O API. Набор методов этого класса представляет
лишь подмножество богатого набора API для получения информации об изображениях и для
управления процессом декодирования (чтения).

Некоторые другие возможности Image I/O мы рассмотрим далее, в разделе
[«Запись и сохранение изображения»](#запись-и-сохранение-изображения).

## Отрисовка изображения

Как вы уже узнали, метод `Graphics.drawImage` рисует изображение в заданной позиции:

```java
boolean Graphics.drawImage(Image img,
                   int x, int y,
                   ImageObserver observer);
```

Позиция `x,y` задаёт положение верхнего левого угла изображения. Параметр `observer`
уведомляет приложение об обновлениях изображения, загружаемого асинхронно. Параметр
`observer` редко используется напрямую и не нужен для класса
[`BufferedImage`](https://docs.oracle.com/javase/8/docs/api/java/awt/image/BufferedImage.html),
поэтому обычно он равен `null`.

Описанный метод покрывает только случай, когда изображение рисуется целиком, отображая
пиксели изображения в координаты пользовательского пространства один к одному. Иногда
приложениям нужно нарисовать часть изображения (подизображение), масштабировать
изображение под определённую область поверхности рисования либо преобразовать или
отфильтровать изображение перед отрисовкой.

Эти операции выполняют перегрузки метода `drawImage()`. Например, следующая перегрузка
`drawImage()` позволяет отрисовать столько от указанной области указанного изображения,
сколько доступно в данный момент, масштабируя её так, чтобы вписать в заданную область
поверхности назначения:

```java
boolean Graphics.drawImage(Image img,
       int dstx1, int dsty1, int dstx2, int dsty2,
       int srcx1, int srcy1, int srcx2, int srcy2,
       ImageObserver observer);
```

Параметры `src` задают область изображения, которую нужно скопировать и нарисовать.
Параметры `dst` задают область назначения, которую нужно покрыть исходной областью.
Координаты `dstx1, dsty1` определяют место отрисовки изображения. Ширина и высота области
назначения вычисляются по выражениям: `(dstx2-dstx1)` и `(dsty2-dsty1)`. Если размеры
исходной области и области назначения различаются, Java 2D API масштабирует изображение
вверх или вниз по необходимости.

Следующий пример кода делит изображение на четыре квадранта и случайным образом рисует
каждый квадрант исходного изображения в другой квадрант области назначения.

> **Примечание.** Если апплет не запускается, необходимо установить как минимум
> Java SE Development Kit (JDK) 7.

Полный код этого апплета находится в `JumbledImageApplet.java`.

В этом примере для отрисовки перемешанного изображения `duke_skateboard.jpg` используется
следующий код. Он перебирает четыре подизображения источника, рисуя каждое по очереди в
случайно выбранный квадрант назначения.

```java
/* делим изображение 'bi' на четыре прямоугольные
 * области и рисуем каждую из них в другую часть
 * изображения, чтобы перемешать его. 'cells' —
 * массив, заполненный значениями, которые
 * перенаправляют отрисовку одной подобласти
 * в другую подобласть.
 */
int cellWidth = bi.getWidth(null)/2;
int cellHeight = bi.getHeight(null)/2;
for (int x=0; x<2; x++) {
    int sx = x*cellWidth;
    for (int y=0; y<2; y++) {
        int sy = y*cellHeight;
        int cell = cells[x*2+y];
        int dx = (cell / 2) * cellWidth;
        int dy = (cell % 2) * cellHeight;
        g.drawImage(bi,
                    dx, dy, 
                    x+cellWidth, dy+cellHeight,
                    sx, sy,
                    sx+cellWidth, sy+cellHeight,
                    null);
    }
}
```

### Фильтрация изображений

Помимо копирования и масштабирования изображений, Java 2D API также позволяет фильтровать
изображение. *Фильтрация* (*filtering*) — это рисование или порождение нового изображения
путём применения алгоритма к пикселям исходного изображения. Фильтры изображений можно
применять следующим методом:

```java
void Graphics2D.drawImage(BufferedImage img,
                          BufferedImageOp op,
                          int x, int y)
```

Параметр `BufferedImageOp` реализует фильтр. Следующий апплет показывает изображение,
нарисованное поверх текста. Перетаскивая ползунок, можно показать больше или меньше текста
сквозь изображение и сделать изображение более или менее прозрачным.

> **Примечание.** Если апплет не запускается, необходимо установить как минимум
> Java SE Development Kit (JDK) 7.

Следующий код показывает, как выполняется действие фильтра: операция применяется к объекту
`BufferedImage` с альфа-каналом (*alpha channel*), и этот альфа-канал масштабируется с
помощью объекта `RescaleOp`. Альфа-канал определяет полупрозрачность (*translucency*)
каждого пикселя. Он также задаёт степень, в которой это изображение перекрывает фон.

```java
/* Создаём BufferedImage в формате ARGB */
BufferedImage img = ImageIO.read(imageSrc);
int w = img.getWidth(null);
int h = img.getHeight(null);
BufferedImage bi = new
    BufferedImage(w, h, BufferedImage.TYPE_INT_ARGB);
Graphics g = bi.getGraphics();
g.drawImage(img, 0, 0, null);

/*
 * Создаём фильтр-операцию rescale, который делает
 * изображение непрозрачным на 50%.
 */
float[] scales = { 1f, 1f, 1f, 0.5f };
float[] offsets = new float[4];
RescaleOp rop = new RescaleOp(scales, offsets, null);

/* Рисуем изображение, применяя фильтр */
g2d.drawImage(bi, rop, 0, 0);
```

Полный пример, представленный в `SeeThroughImageApplet.java`, включает код, который с
помощью ползунка регулирует прозрачность, начиная с исходных 50%. Этому примеру также
требуется изображение `duke_skateboard.jpg`.

Объект `RescaleOp` — лишь один из множества фильтров, которые можно создать. В Java 2D API
есть несколько встроенных фильтров, включая следующие:

- `ConvolveOp` — каждый выходной пиксель вычисляется по окружающим пикселям исходного
  изображения. Может использоваться для размытия или повышения резкости изображений.
- `AffineTransformOp` — этот фильтр отображает пиксели источника в другую позицию
  назначения, применяя преобразование к местоположению пикселя.
- `LookupOp` — этот фильтр использует предоставляемую приложением таблицу подстановки
  (*lookup table*) для переотображения цветов пикселей.
- `RescaleOp` — этот фильтр умножает цвета на некоторый множитель. Может использоваться
  для осветления или затемнения изображения, увеличения или уменьшения его непрозрачности
  и т. п.

Полный код этого апплета находится в `ImageDrawingApplet.java`; апплету требуется

## Создание изображения и рисование в нём

Мы уже знаем, как загрузить существующее изображение, которое было создано и сохранено в
вашей системе или в каком-либо сетевом расположении. Но, вероятно, вам захочется также
создать новое изображение как буфер данных пикселей.

В этом случае можно создать объект `BufferedImage` вручную, используя три конструктора
этого класса:

- `new BufferedImage(width, height, type)` — конструирует `BufferedImage` одного из
  предопределённых типов изображения.
- `new BufferedImage(width, height, type, colorModel)` — конструирует `BufferedImage`
  одного из предопределённых типов изображения: `TYPE_BYTE_BINARY` или `TYPE_BYTE_INDEXED`.
- `new BufferedImage(colorModel, raster, premultiplied, properties)` — конструирует новый
  `BufferedImage` с заданными `ColorModel` и `Raster`.

С другой стороны, можно использовать методы класса `Component`. Эти методы способны
проанализировать разрешение экрана для данного `Component` или `GraphicsConfiguration` и
создать изображение подходящего типа.

- `Component.createImage(width, height)`
- `GraphicsConfiguration.createCompatibleImage(width, height)`
- `GraphicsConfiguration.createCompatibleImage(width, height, transparency)`

`GraphicsConfiguration` возвращает объект типа `BufferedImage`, тогда как `Component`
возвращает объект типа `Image`. Если вместо этого нужен объект `BufferedImage`, можно
выполнить проверку `instanceof` и привести (*cast*) объект к `BufferedImage` в своём коде.

Как уже упоминалось в предыдущих уроках, изображения можно отрисовывать не только на
экране. Само изображение можно рассматривать как поверхность рисования. Для этого можно
использовать метод `createGraphics()` класса `BufferedImage`:

```java
BufferedImage off_Image =
  new BufferedImage(100, 50,
                    BufferedImage.TYPE_INT_ARGB);

Graphics2D g2 = off_Image.createGraphics();
```

Ещё одно интересное применение внеэкранных (*off-screen*) изображений — автоматическая
*двойная буферизация* (*double buffering*). Эта возможность позволяет избежать мерцания в
анимированных изображениях за счёт того, что изображение рисуется в задний буфер
(*back buffer*), а затем этот буфер копируется на экран — вместо прямой отрисовки на
экране.

Java 2D также предоставляет доступ к аппаратному ускорению (*hardware acceleration*) для
внеэкранных изображений, что может обеспечить лучшую производительность отрисовки в такие
изображения и копирования из них. Воспользоваться этой функциональностью можно с помощью
следующих методов класса `Image`:

- метод `getCapabilities` позволяет определить, ускорено ли изображение в данный момент;
- метод `setAccelerationPriority` позволяет задать подсказку о том, насколько важно
  ускорение для данного изображения;
- метод `getAccelerationPriority` возвращает подсказку о важности ускорения.

## Запись и сохранение изображения

Этот урок начался с объяснения того, как использовать пакет `javax.imageio` для загрузки
изображений из внешнего формата во внутренний формат `BufferedImage`, используемый
Java 2D. Затем было показано, как с помощью `Graphics.drawImage()` отрисовать это
изображение с необязательной фильтрацией.

Заключительный этап — сохранение объекта `BufferedImage` во внешний формат изображения.
Это может быть изображение, изначально загруженное классом Image I/O из внешнего формата и,
возможно, изменённое средствами Java 2D API, либо изображение, созданное самим Java 2D.

Класс Image I/O предоставляет простой способ сохранять изображения в различных форматах,
как в следующем примере:

```java
static boolean ImageIO.write(RenderedImage im, 
                             String formatName,
                             File output)  throws IOException
```

> **Примечание.** Класс `BufferedImage` реализует интерфейс `RenderedImage`.

Параметр `formatName` выбирает формат изображения, в котором следует сохранить
`BufferedImage`.

```java
try {
    // получаем изображение
    BufferedImage bi = getMyImage();
    File outputfile = new File("saved.png");
    ImageIO.write(bi, "png", outputfile);
} catch (IOException e) {
    ...
}
```

Метод `ImageIO.write` вызывает код, реализующий запись PNG, — «плагин записи PNG»
(*PNG writer plug-in*). Термин *плагин* (*plug-in*) используется потому, что Image I/O
расширяем и может поддерживать широкий диапазон форматов.

Но следующие стандартные плагины форматов изображений — JPEG, PNG, GIF, BMP и WBMP —
присутствуют всегда.

У каждого формата изображения есть свои достоинства и недостатки:

| Формат | Плюсы | Минусы |
|--------|-------|--------|
| GIF | Поддерживает анимацию и прозрачные пиксели | Поддерживает только 256 цветов и не поддерживает полупрозрачность |
| PNG | Лучшая альтернатива GIF или JPG для полноцветных изображений без потерь, поддерживает полупрозрачность | Не поддерживает анимацию |
| JPG | Отлично подходит для фотографических изображений | Сжатие с потерями; не подходит для текста, скриншотов и любых приложений, где исходное изображение должно сохраняться в точности |

Для большинства приложений достаточно использовать один из этих стандартных плагинов. Их
преимущество в том, что они доступны без дополнительных усилий. Класс Image I/O
предоставляет способ подключить поддержку дополнительных форматов, и таких плагинов
существует множество. Если вас интересует, какие форматы файлов доступны для загрузки или
сохранения в вашей системе, можно использовать методы `getReaderFormatNames` и
`getWriterFormatNames` класса `ImageIO`. Эти методы возвращают массив строк со списком всех
форматов, поддерживаемых в данной JRE.

```java
String writerNames[] = ImageIO.getWriterFormatNames();
```

Возвращаемый массив имён будет включать любые дополнительно установленные плагины, и любое
из этих имён можно использовать как имя формата для выбора средства записи изображения
(*image writer*). Следующий пример кода — это упрощённая версия полноценной программы
редактирования / ретуши изображений, использующая переработанную версию примера
`ImageDrawingApplet.java`, которую можно использовать так:

- сначала изображение загружается через Image I/O;
- пользователь выбирает фильтр из выпадающего списка, и рисуется новое обновлённое
  изображение;
- пользователь выбирает формат сохранения из выпадающего списка;
- затем появляется диалог выбора файла, и пользователь указывает, куда сохранить
  изображение;
- изменённое изображение теперь можно просмотреть в других настольных приложениях.

Полный код этого примера представлен в `SaveImage.java`.

В этом уроке вы изучили лишь основы Image I/O, который предоставляет обширную поддержку
записи изображений, включая работу напрямую с плагином `ImageWriter` для более тонкого
контроля над процессом кодирования. Image I/O может записывать несколько изображений,
метаданные изображения и определять компромисс между качеством и размером. Подробнее см.
[Java Image I/O API Guide](https://docs.oracle.com/javase/8/docs/technotes/guides/imageio/spec/title.fm.html).
