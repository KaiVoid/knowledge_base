# Урок 5. Печать

**Трейл:** 2D Graphics · **Оригинал:** [Printing](https://docs.oracle.com/javase/tutorial/2d/printing/index.html)
**Связанные области:** [[01-core-java-syntax-oop]] · **Вопросы:** core-java

> Перевод официального руководства Oracle (The Java Tutorials, JDK 8). Урок объединяет
> индексную страницу *Printing* и подстраницы: *A Basic Printing Program*,
> *Using Print Setup Dialogs*, *Printing a Multiple Page Document*,
> *Working with Print Services and Attributes*, *Printing the Contents of a User Interface*
> и *Printing Support in Swing Components*.

> Поскольку API Java 2D позволяет рисовать на любой поверхности, естественным его
> расширением становится возможность печатать графику Java 2D. Принтер можно рассматривать
> как графическое устройство — точно так же, как дисплей.
>
> API печати Java 2D не ограничивается печатью графики. Он позволяет печатать и содержимое
> пользовательского интерфейса приложения. Содержимое можно напечатать, отправив на принтер
> «сырые» данные под управлением форматирования API печати Java 2D, либо воспользовавшись
> графическим API Java 2D.
>
> В этом уроке вы изучите функции управления принтером и заданием печати в API печати Java 2D,
> которые дополняют элементы отрисовки (rendering). Вы научитесь находить принтеры,
> настроенные в системе или сети, и узнавать о них сведения — например, какие размеры бумаги
> они поддерживают, — а также выбирать эти атрибуты для печати и пользоваться диалоговыми окнами.

Основные классы и интерфейсы, участвующие в печати, находятся в пакетах
[`java.awt.print`](https://docs.oracle.com/javase/8/docs/api/java/awt/print/package-summary.html)
и [`javax.print`](https://docs.oracle.com/javase/8/docs/api/javax/print/package-summary.html)
(последний пакет даёт доступ к службам печати).

Базовые операции печати рассмотрены в следующих разделах:

- **Базовая программа печати** — описывает интерфейс `Printable` и приводит простейшую
  программу печати.
- **Использование диалогов настройки печати** — объясняет, как показать диалог настройки печати.
- **Печать многостраничного документа** — объясняет, как применять разбиение на страницы
  (pagination) для печати многостраничного документа.
- **Работа со службами печати и атрибутами** — рассказывает о службах печати, о том, как
  задать формат данных печати и как создать задание печати с помощью пакета `javax.print`.
- **Печать содержимого пользовательского интерфейса** — объясняет, как напечатать содержимое
  окна или фрейма.
- **Поддержка печати в компонентах Swing** — кратко описывает связанную функциональность
  печати в `Swing` и отсылает к конкретным классам и интерфейсам `Swing`.

## Базовая программа печати

> В этом разделе объясняется, как создать простую программу печати, которая показывает диалог
> печати и печатает текст «Hello World» на выбранном принтере.

Задача печати обычно состоит из двух частей:

- **Управление заданием (job control)** — создание задания печати, привязка его к принтеру,
  указание числа копий и взаимодействие с пользователем через диалог печати.
- **Формирование страницы (page imaging)** — отрисовка содержимого на странице и управление
  содержимым, которое выходит за пределы одной страницы (разбиение на страницы, pagination).

Сначала создайте задание печати. Класс, представляющий задание печати, и большинство связанных
классов находятся в пакете
[`java.awt.print`](https://docs.oracle.com/javase/8/docs/api/java/awt/print/package-summary.html).

```java
import java.awt.print.*;

PrinterJob job = PrinterJob.getPrinterJob();
```

Далее напишите код, который отрисовывает содержимое на странице, реализовав интерфейс
[`Printable`](https://docs.oracle.com/javase/8/docs/api/java/awt/print/Printable.html).

```java
class HelloWorldPrinter
              implements Printable { ... }
...
job.setPrintable(new HelloWorldPrinter());
```

Приложение обычно показывает диалог печати, чтобы пользователь мог настроить различные
параметры — число копий, ориентацию страницы или принтер-получатель.

```java
boolean doPrint = job.printDialog();
```

Этот диалог отображается до тех пор, пока пользователь не подтвердит или не отменит печать.
Переменная `doPrint` будет равна `true`, если пользователь дал команду продолжить и печатать.
Если `doPrint` равна `false`, пользователь отменил задание печати. Поскольку отображение диалога
вообще не обязательно, возвращаемое значение носит чисто информационный характер.

Если переменная `doPrint` равна `true`, то приложение запрашивает печать задания, вызывая метод
`PrinterJob.print`.

```java
if (doPrint) {
    try {
        job.print();
    } catch (PrinterException e) {
        // Задание не завершилось
        // успешно
    }
}
```

Исключение `PrinterException` будет выброшено, если возникла проблема при отправке задания на
принтер. Однако, поскольку метод `PrinterJob.print` возвращает управление сразу после отправки
задания на принтер, приложение пользователя не может обнаружить замятие бумаги или её отсутствие.
Этого шаблона управления заданием достаточно для базовых задач печати.

Интерфейс `Printable` имеет всего один метод:

```java
public int print(Graphics graphics,
           PageFormat pf, int page)
           throws PrinterException;
```

Класс [`PageFormat`](https://docs.oracle.com/javase/8/docs/api/java/awt/print/PageFormat.html)
описывает ориентацию страницы (книжная — portrait, или альбомная — landscape), её размер и
область печати (imageable area) в единицах 1/72 дюйма. Область печати учитывает ограничения полей
большинства принтеров (аппаратные поля, hardware margin). Область печати — это пространство внутри
этих полей; на практике оно нередко дополнительно сужается, чтобы оставить место для колонтитулов.

Параметр `page` — это отсчитываемый от нуля номер страницы, которая будет отрисована.

Следующий код представляет полную реализацию `Printable`:

```java
import java.awt.print.*;
import java.awt.*;

public class HelloWorldPrinter
    implements Printable {

  public int print(Graphics g, PageFormat pf, int page)
      throws PrinterException {

    // У нас только одна страница, а 'page'
    // отсчитывается от нуля
    if (page > 0) {
         return NO_SUCH_PAGE;
    }

    // Пользовательская точка (0,0) обычно находится за
    // пределами области печати, поэтому мы должны сместить
    // её на значения X и Y из PageFormat,
    // чтобы избежать обрезки.
    Graphics2D g2d = (Graphics2D)g;
    g2d.translate(pf.getImageableX(), pf.getImageableY());

    // Теперь выполняем нашу отрисовку
    g.drawString("Hello world!", 100, 100);

    // сообщаем вызывающей стороне, что эта страница —
    // часть печатаемого документа
    return PAGE_EXISTS;
  }
}
```

Полный код этого примера находится в файле `HelloWorldPrinter.java`.

Передача экземпляра [`Graphics`](https://docs.oracle.com/javase/8/docs/api/java/awt/Graphics.html)
на принтер по сути ничем не отличается от отрисовки его на экране. В обоих случаях нужно выполнить
следующие шаги:

- Нарисовать тестовую строку так же просто, как и выполнить другие операции рисования в
  `Graphics2D`.
- Графика принтера имеет более высокое разрешение, что должно быть прозрачно для большей части кода.
- Метод `Printable.print()` вызывается системой печати — точно так же, как метод
  `Component.paint()` вызывается для отрисовки компонента (Component) на дисплее. Система печати
  будет вызывать метод `Printable.print()` для страниц 0, 1 и т. д., пока метод `print()` не
  вернёт `NO_SUCH_PAGE`.
- Метод `print()` может вызываться с одним и тем же индексом страницы несколько раз, пока документ
  не будет завершён. Это применяется, когда пользователь задаёт такие атрибуты, как несколько
  копий с опцией подбора (collate).
- Область печати (imageable area) объекта `PageFormat` определяет область отсечения (clip area).
  Область печати также важна при расчёте разбиения на страницы, то есть распределения содержимого
  по печатаемым страницам, поскольку разрывы страниц определяются тем, сколько помещается на каждой
  странице.

> **Примечание.** Вызов метода `print()` может быть пропущен для некоторых индексов страниц, если
> пользователь задал иной диапазон страниц, не включающий конкретный индекс.

## Использование диалогов настройки печати

> Традиционно пользователь хочет видеть окна настройки страницы и печати. В диалоге печати можно
> выбрать принтер, указать печатаемые страницы и задать число копий.

![Диалог печати (оригинал)](../../figures/2d/print-dialog.gif)

> *Подпись оригинала: рисунок представляет диалог печати.*

Приложение показывает диалог печати, когда пользователь нажимает кнопку, связанную с командой
печати, или выбирает пункт из меню печати. Чтобы показать этот диалог, вызовите метод
[`printDialog`](https://docs.oracle.com/javase/8/docs/api/java/awt/print/PrinterJob.html#printDialog--)
класса
[`PrinterJob`](https://docs.oracle.com/javase/8/docs/api/java/awt/print/PrinterJob.html):

```java
PrinterJob pj = PrinterJob.getPrinterJob();
...
    if (pj.printDialog()) {
        try {pj.print();}
        catch (PrinterException exc) {
            System.out.println(exc);
         }
     }
...
```

Этот метод возвращает `true`, если пользователь нажал «OK» и закрыл диалог, и `false` в противном
случае. Выбор пользователя в диалоге ограничивается числом и форматом страниц, которые были заданы
для `PrinterJob`.

Метод `printDialog` в приведённом фрагменте открывает нативный диалог печати. Пример кода
`PrintDialogExample.java` показывает, как отобразить кросс-платформенный диалог печати.

Изменить информацию о настройке страницы, содержащуюся в объекте
[`PageFormat`](https://docs.oracle.com/javase/8/docs/api/java/awt/print/PageFormat.html), можно
с помощью диалога настройки страницы.

![Диалог настройки страницы (оригинал)](../../figures/2d/page-setup-dialog.gif)

> *Подпись оригинала: рисунок представляет окно диалога настройки страницы.*

Чтобы показать диалог настройки страницы, вызовите метод `pageDialog` класса `PrinterJob`.

```java
PrinterJob pj = PrinterJob.getPrinterJob();
PageFormat pf = pj.pageDialog(pj.defaultPage());
```

Диалог настройки страницы инициализируется параметром, переданным в `pageDialog`. Если пользователь
нажимает кнопку «OK» в диалоге, экземпляр `PageFormat` будет создан в соответствии с выбором
пользователя и возвращён. Если пользователь отменяет диалог, `pageDialog` возвращает исходный,
неизменённый `PageFormat`.

Обычно API печати Java 2D требует, чтобы приложение показывало диалог печати, но иногда возможно
печатать вообще без отображения какого-либо диалога. Такой вид печати называется **«тихой печатью»
(silent printing)**. Он может быть полезен в особых случаях — например, когда нужно печатать
определённый еженедельный отчёт из базы данных. В остальных случаях всегда рекомендуется уведомлять
пользователя о начале процесса печати.

## Печать многостраничного документа

> Вы уже узнали, как использовать интерфейс
> [`Printable`](https://docs.oracle.com/javase/8/docs/api/java/awt/print/Printable.html) для печати
> одностраничного документа. Однако документы обычно занимают более одной физической страницы.
> **Разбиение на страницы (pagination)** — это процесс определения мест в документе, где
> происходят разрывы страниц, и печати соответствующим образом.

В случае печати нескольких графических изображений — по одному на страницу — используйте индекс
страницы, чтобы перебирать эти страницы и печатать по одному изображению на каждой. Например, если
несколько изображений представлены следующим массивом:

```java
BufferedImage[] images = new BufferedImage[10];
```

то используйте метод `print()`, как показано в следующем фрагменте кода:

```java
public int print(Graphics graphics,
           PageFormat pageFormat, int pageIndex)
           throws PrinterException {

    if (pageIndex < images.length) {
        graphics.drawImage(images[pageIndex], 100, 100, null);
        return PAGE_EXISTS;
    } else {
        return NO_SUCH_PAGE:
    }
}
```

Если документ непрерывен, приложение должно вычислить, сколько содержимого помещается на каждой
странице, и разорвать страницу в этой точке. Если текстовый документ состоит из множества строк,
приложение должно подсчитать, сколько из этих строк целиком помещается на странице. Класс
[`Point`](https://docs.oracle.com/javase/8/docs/api/java/awt/Point.html) создаёт точку,
представляющую местоположение в координатах (x, y).

Чтобы вычислить высоту одной строки текста, используйте класс
[`FontMetrics`](https://docs.oracle.com/javase/8/docs/api/java/awt/FontMetrics.html).

```java
Font font = new Font("Serif", Font.PLAIN, 10);
FontMetrics metrics = graphics.getFontMetrics(font);
int lineHeight = metrics.getHeight();
```

Параметр `PageFormat` описывает печатаемую область страницы. В частности, чтобы узнать вертикальную
протяжённость страницы, используйте следующий фрагмент кода:

```java
double pageHeight = pageFormat.getImageableHeight();
```

Используйте следующий фрагмент кода, чтобы вычислить число строк, помещающихся на странице, и число
разрывов страниц:

```java
int linesPerPage = ((int)pageHeight)/lineHeight);
int numBreaks = (textLines.length-1)/linesPerPage;
int[] pageBreaks = new int[numBreaks];
for (int b=0; b < numBreaks; b++) {
    pageBreaks[b] = (b+1)*linesPerPage;
}
```

Печатаемую область следует вычислять внутри метода `print()` по следующим причинам:

- Измерение текста зависит от `FontRenderContext`, а он неявно содержится в объекте `FontMetrics`,
  возвращаемом графикой принтера, который доступен только внутри метода `print()`.
- Формат страницы может быть неизвестен до момента печати. Если пользователь выбрал альбомный режим
  (landscape) в диалоге печати, эту настройку необходимо учесть. Объект `PageFormat`, переданный в
  метод `print()`, предоставляет эту информацию.

Позиции разрывов страниц используются так, как показано в следующем фрагменте кода:

```java
/* Рисуем каждую строку, которая находится на этой странице.
 * Увеличиваем позицию 'y' на lineHeight
 * для каждой строки.
 */
int y = 0;
int start = (pageIndex == 0) ? 0 : pageBreaks[pageIndex-1];
int end   = (pageIndex == pageBreaks.length)
                 ? textLines.length : pageBreaks[pageIndex];
for (int line=start; line<end; line++) {
    y += lineHeight;
    g.drawString(textLines[line], 0, y);
}
```

Если документ содержит 100 строк, а на странице помещается лишь 48 строк, то приложение печатает
3 страницы с разрывами после 48-й и 96-й строк текста. Оставшиеся 4 строки печатаются на последней
странице. Полный код этого примера находится в файле `PaginationExample.java`.

В коде `PaginationExample` использованы следующие упрощающие допущения:

- Каждая страница имеет одинаковую высоту.
- Используется один и тот же шрифт.

## Работа со службами печати и атрибутами

> Из предыдущих разделов вы узнали, что API печати Java 2D поддерживает формирование страницы,
> отображает диалоги печати и настройки страницы, а также задаёт атрибуты печати. Службы печати —
> ещё один ключевой компонент любой подсистемы печати.

**API службы печати Java (Java Print Service, JPS)** расширяет имеющиеся возможности печати Java 2D,
предоставляя следующую функциональность:

- Приложение обнаруживает принтеры, отвечающие его потребностям, динамически запрашивая возможности
  принтера.
- Приложение расширяет набор атрибутов, входящих в API JPS.
- Сторонние разработчики могут подключать собственные службы печати через интерфейс поставщика
  служб (Service Provider Interface), которые печатают в различных форматах, включая PostScript,
  PDF и SVG.

API службы печати Java состоит из четырёх пакетов:

![Четыре пакета, необходимых для печати (оригинал)](../../figures/2d/JPS.gif)

> *Подпись оригинала: рисунок представляет четыре пакета, необходимых для печати.*

Пакет
[`javax.print`](https://docs.oracle.com/javase/8/docs/api/javax/print/package-summary.html)
предоставляет основные классы и интерфейсы API службы печати Java. Он позволяет клиентским и
серверным приложениям:

- Обнаруживать и выбирать службы печати на основе их возможностей.
- Задавать формат данных печати.
- Передавать задания печати службам, поддерживающим печатаемый тип документа.

### Спецификация типа документа

Класс
[`DocFlavor`](https://docs.oracle.com/javase/8/docs/api/javax/print/DocFlavor.html)
представляет формат данных печати, например JPEG или PostScript. Формат `DocFlavor` состоит из двух
частей: MIME-типа и имени класса-представления (representation class name). MIME-тип описывает
формат, а имя класса-представления документа указывает, как документ доставляется на принтер или в
выходной поток. Приложение использует `DocFlavor` и набор атрибутов, чтобы найти принтеры с
возможностями, заданными этим набором атрибутов. Этот пример кода демонстрирует получение массива
объектов `StreamPrintServiceFactory`, которые могут возвращать объекты `StreamPrintService`,
способные преобразовать изображение GIF в PostScript:

```java
DocFlavor flavor  = DocFlavor.INPUT_STREAM.GIF;
String psMimeType = DocFlavor.BYTE_ARRAY.
                    POSTSCRIPT.getMimeType();
StreamPrintServiceFactory[] psfactories =
              StreamPrintServiceFactory.
              lookupStreamPrintServiceFactories(
              flavor, psMimeType);
```

### Определения атрибутов

Пакеты
[`javax.print.attribute`](https://docs.oracle.com/javase/8/docs/api/javax/print/attribute/package-summary.html)
и
[`javax.print.attribute.standard`](https://docs.oracle.com/javase/8/docs/api/javax/print/attribute/standard/package-summary.html)
определяют атрибуты печати, которые описывают возможности службы печати, задают требования к заданию
печати и отслеживают ход выполнения задания печати.

Например, если вы хотите использовать формат бумаги A4 и напечатать три копии документа, вам нужно
создать набор следующих атрибутов, реализующий интерфейс `PrintRequestAttributeSet`:

```java
PrintRequestAttributeSet attr_set =
    new HashPrintRequestAttributeSet();
attr_set.add(MediaSize.ISO_A4);
attr_set.add(new Copies(3));
```

Затем вы должны передать набор атрибутов в метод `print` задания печати вместе с `DocFlavor`.

### Обнаружение служб печати

Приложение вызывает статические методы абстрактного класса `PrintServiceLookup`, чтобы найти службы
печати, способные удовлетворить запрос печати приложения. Например, чтобы напечатать две копии
двустороннего документа, приложению сначала нужно найти принтеры с возможностью двусторонней печати:

```java
DocFlavor doc_flavor = DocFlavor.INPUT_STREAM.PDF;
PrintRequestAttributeSet attr_set =
    new HashPrintRequestAttributeSet();
attr_set.add(new Copies(2));
attr_set.add(Sides.DUPLEX);
PrintService[] service = PrintServiceLookup.
              lookupPrintServices(doc_flavor,
              attr_set);
```

### Типичное использование API

В заключение: API службы печати Java выполняет следующие шаги для обработки запроса печати:

1. Выбирает `DocFlavor`.
2. Создаёт набор атрибутов.
3. Находит службу печати, способную обработать запрос печати, заданный `DocFlavor` и набором
   атрибутов.
4. Создаёт объект `Doc`, инкапсулирующий `DocFlavor` и фактические данные печати.
5. Получает задание печати, представленное `DocPrintJob`, от службы печати.
6. Вызывает метод `print` задания печати.

Дополнительные сведения об API службы печати Java см. в
[руководстве пользователя Java 2D Print Service API](https://docs.oracle.com/javase/8/docs/technotes/guides/jps/spec/JPSTOC.fm.html).

## Печать содержимого пользовательского интерфейса

> Ещё одна распространённая задача печати — напечатать содержимое окна или фрейма, целиком или
> частично. Окно может содержать следующие компоненты: панели инструментов, кнопки, ползунки,
> текстовые метки, прокручиваемые текстовые области, изображения и другое графическое содержимое.
> Все эти компоненты печатаются с помощью следующих методов API печати Java 2D:

```java
java.awt.Component.print(Graphics g);
java.awt.Component.printAll(Graphics g);
```

Следующий рисунок представляет простой пользовательский интерфейс.

![Печать 12 строк в окне (оригинал)](../../figures/2d/printui.png)

> *Подпись оригинала: печать 12 строк в окне.*

Код для создания этого интерфейса находится в примере программы `PrintUIWindow.java`.

Чтобы напечатать это окно, измените код из предыдущих примеров, в которых печатались текст или
изображения. Получившийся код должен выглядеть так:

```java
public int print(Graphics g, PageFormat pf, int page)
    throws PrinterException {
    if (page > 0) {
        return NO_SUCH_PAGE;
    }

    Graphics2D g2d = (Graphics2D)g;
    g2d.translate(pf.getImageableX(), pf.getImageableY());

    // Печатаем всё видимое содержимое
    // java.awt.Frame.
    frame.printAll(g);

    return PAGE_EXISTS;
}
```

> **Примечание.** Вызов метода `printAll` — единственное отличие этого примера от примеров печати
> текста или изображения. Метод `print(Graphics g)` повторяет (mirrors) метод
> `java.awt.Component.paint(Graphics g)`, используемый для экранной отрисовки. Используйте метод
> `print()`, а не `paint()`, поскольку класс `Components` мог переопределить метод `print()`, чтобы
> обрабатывать случай печати по-другому.

Метод `printAll(Graphics g)` печатает компонент и все его подкомпоненты. Этот метод обычно
используется для печати объекта вроде целого окна, а не одного компонента.

## Поддержка печати в компонентах Swing

> Пример `PrintUIWindow.java`, представленный в предыдущем разделе, демонстрирует, что распечатка
> в точности совпадает с тем, что вы видели на экране. Такое поведение выглядит разумным. Однако
> если окно прокручиваемое, то содержимое, в данный момент прокрученное за пределы видимости, не
> попадает в распечатку. Это создаёт эффект «обрезки» на принтере. Это становится особенно серьёзной
> проблемой при печати крупных компонентов, таких как таблица Swing или текстовые компоненты.
> Компоненты могут содержать много строк текста, которые не могут целиком поместиться на экране. В
> этом случае печатайте содержимое, отображаемое компонентом, способом, согласованным с экранным
> отображением.

Чтобы решить эту проблему, таблица Swing и все текстовые компоненты «осведомлены о печати» (printing
aware). Следующие методы напрямую обеспечивают использование печати Java 2D:

- [`javax.swing.JTable.print();`](https://docs.oracle.com/javase/8/docs/api/javax/swing/JTable.html#print--)
- [`javax.swing.text.JTextComponent.print();`](https://docs.oracle.com/javase/8/docs/api/javax/swing/text/JTextComponent.html#print--)

Эти методы предоставляют полную реализацию печати своего содержимого. Приложению не нужно
непосредственно создавать объект `PrinterJob` и реализовывать интерфейс `Printable`. Вызов этих
методов показывает диалог печати и печатает данные компонента в соответствии с выбором пользователя.
Существуют также дополнительные методы, предоставляющие больше параметров.
